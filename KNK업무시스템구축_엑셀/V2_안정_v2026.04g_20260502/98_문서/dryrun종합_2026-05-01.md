## 🟡 빅터 권고 사항

### 권고 1. 1_프로젝트등록에서도 직접 매출 입력 가능하게 개선?
현 로직: 4_완료이력 경유 필수 → 직원이 헷갈릴 가능성
**대안 A**: `_archive_settled`를 1_프로젝트등록에도 적용 (납품완료 처리 + 매출 입력 동시 시 → 4_완료이력 + 8_매출마감 양쪽으로 분배)
**대안 B**: 현 흐름 유지하되 사용법 명문화 (현재 적용)

→ A는 코드 변경 필요. B는 즉시 가능. 운영 시작 후 직원 혼선 발견되면 A로 전환 검토.

### 권고 2. 1_프로젝트등록의 계산서·입금 컬럼 R3 가이드 강화
현재 R3가 단순 `input`. **"4_완료이력에서 입력하세요"** 같은 안내 메모 추가하면 혼선 감소.

### 권고 3. 매출_미수금 시트 정렬
현재는 입력 순. **계산서 발행일 오래된 순(장기 미수 우선)** 정렬이 회계 관점 더 유용. 코드 1줄 추가.
