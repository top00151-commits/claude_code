## 2. 시트별 사용법

### 2.1 오늘의현황 (일일 KPI, 27행 × 4열)
**누가**: 매일 아침 대표·임원
**용도**:
- 오늘 출하 예정 건수
- 오늘 마감 예정 (D-day = 0)
- 미수금 합계 (KRW/USD)
- 임박한 D-day TOP 5

### 2.2 매출총괄 (8행 × 10열)
| 컬럼 | 의미 |
|---|---|
| 유형 | 검사기 / 자동화 / 소모품 |
| 진행 | 활성 프로젝트 수 (수주예정·진행중·보류) |
| 완료 | 납품완료 건수 |
| 총 | 진행 + 완료 합계 |
| KRW | 원화 수주 합계 |
| USD | 달러 수주 합계 |
| INV$ | 인보이스 USD 환산 합계 |
| 진척률 | 활성 프로젝트 평균 진척률 |
| 지연 | 납기 지난 활성 프로젝트 수 (D-day > 0) |

**합계 행**: 모든 모듈을 통합한 전사 수치

### 2.3 이번주_출하 (★ v2026.04e — Gantt 시각화, 66행 × 15열)
**누가**: 영업·PM·생산
**용도**: 검사기 모듈 D-day [-7, +3] 범위 프로젝트를 **이번 주 월~금 일자별 컬럼**으로 펼쳐 색상 표시
- 출하 예정 = 노란색 셀
- 출하 완료 = 녹색 셀
- 지연 = 빨강 셀
- 한눈에 이번 주 출하 부하 파악

### 2.4 진행률_매트릭스 (★ v2026.04e, 132행 × 20열)
**누가**: 부서장·PM
**용도**: 행=프로젝트, 열=8개 부서, 셀=해당 부서의 해당 프로젝트 진척률
- 색상 5단계: 0% (회색) / 1-30% (빨강) / 31-70% (주황) / 71-99% (노랑) / 100% (녹색)
- N/A (해당 부서 미참여) = 흰 칸
- 부서별 부하·병목 식별

### 2.5 프로젝트목록 (2000행 × 12열)
| 자동 표시 |
|---|
| 유형, 관리코드, 수주번호, 고객사, 품명, 통화, 금액, 진척률, 납기일, D-day, 진행상태, 등록일자 |

**활용 방법**:
- Excel 필터 기능으로 고객사별·상태별 검색
- D-day > 0 인 항목만 추출 → 즉시 대응 회의 자료
- 통화 = USD인 항목만 추출 → 환율 영향 모니터링

### 2.6 매출_발행예정 (★ v2026.04e, 81행 × 11열)
**누가**: 기술영업팀(안지연·이새롬)
**용도**: 납품완료 + 계산서 미발행 프로젝트 리스트
- 발행해야 할 세금계산서 우선순위
- 발행 후 PMS C32 `계산서발행일` 입력 → 다음 시트로 자동 이동

### 2.7 매출_미수금 (★ v2026.04e, 5행 × 12열)
**누가**: 대표·기술영업·회계
**용도**: 계산서 발행 후 입금 대기 항목
- 미수금 합계
- 계산서 발행일 ~ 오늘 경과 일수 (장기 미수 식별)
- 입금 확인 시 PMS C33 `입금일` 입력 → 8_매출마감 자동 이관

### 2.8 매출_완납YTD (★ v2026.04e, 17행 × 8열)
**누가**: 대표·회계
**용도**: 연중 누적 입금 완료 (월별 집계)
- 1월·2월·…·12월 매출 (KRW/USD)
- 누적 합계
- 전년 동월 비교 (향후 추가)

### 2.9 매출_예측캘린더 (★ v2026.04e, 9행 × 9열)
**누가**: 대표·기술영업
**용도**: 향후 3개월 매출 예측
- 납기일 기반 예상 발행 일정
- 예상 입금 일정 (납기 + 평균 회수 기간)
- 캐시플로우 계획

```cmd
cd 10_대시보드
python -X utf8 scripts\gen_dashboard.py
```

자동 처리:

**실행 시점**:

## 5. 매출 사이클 4시트 흐름 (★ v2026.04e — 2026-05-01 dry-run 검증 완료)

```
[1_프로젝트등록 진행중]
   ↓ (PM이 진행상태 = 납품완료, sync)
[4_완료이력으로 자동 이동]
   ↓
"매출_발행예정" 시트에 표시 (계산서 미발행)
   ↓ (기술영업이 4_완료이력 C32 계산서발행일 입력, sync)
"매출_발행예정"에서 사라짐 → "매출_미수금" 시트 표시
   ↓ (회계가 4_완료이력 C33 입금일 입력, sync)
"매출_미수금"에서 사라짐
   ↓ (자동 이관)
[8_매출마감으로 영구 이관] (4_완료이력에서 사라짐)
   ↓
"매출_완납YTD" 시트 누적 (월별 집계)
```

**향후 3개월 예측**은 "매출_예측캘린더"에서 1_프로젝트등록의 진행중 항목 + 납기일을 기반으로 별도 자동 계산.

### 5.1 입력 위치 핵심 (자주 헷갈리는 부분)
- 계산서·입금일은 **4_완료이력** 시트의 우측 끝(C32·C33)에 입력
- 1_프로젝트등록의 동일 컬럼에 입력해도 8_매출마감 자동 이관은 일어나지 않음
- **흐름 순서를 반드시 지킬 것**: 진행상태 = 납품완료 → sync → 4_완료이력에서 매출 정보 입력 → sync → 8_매출마감 이관

## 7. 자주 묻는 질문

**Q. 9개 시트가 너무 많아요. 매일 봐야 하나요?**
A. 매일 보는 건 **오늘의현황 + 매출_미수금** 2개면 충분합니다. 나머지는 주간(이번주_출하·진행률_매트릭스), 월간(매출_완납YTD·매출_예측캘린더) 단위로 보세요.

**Q. 미수금 합계가 매출총괄과 일치하지 않아요.**
A. 매출총괄은 **수주 합계**(계약 기준), 매출_미수금은 **계산서 발행 후 미입금**(현금 흐름 기준)입니다. 두 시점 차이 때문.

**Q. 가공팀이 빈 칸인 프로젝트가 진행률_매트릭스에서 어떻게 표시되나요?**
A. N/A로 표시 (흰 칸). 평균 진척률 계산에서도 자동 제외됩니다.

**Q. 대시보드 자동 갱신 가능한가요?**
A. 현재는 수동(`gen_dashboard.py` 또는 .bat 실행). Windows 작업 스케줄러로 매일 아침 자동 실행 설정 가능 — 빅터에게 요청.
