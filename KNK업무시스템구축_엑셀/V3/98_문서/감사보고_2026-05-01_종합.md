## 🔴 A. 도입 차단 결함 (즉시 수정)

### A-1. **.bat 18개 전수 `\a` 버그 (CRITICAL)**

**현상**: 18개 .bat 파일 모두 `apply_standard.py` 호출 라인에 ASCII 7(BEL) 문자 삽입됨

```
실제 저장된 바이트: scripts\x07pply_standard.py
의도한 라인:        scripts\apply_standard.py
```

**원인**: BAT 작성 Python 코드에서 일반 문자열로 `"scripts\apply_standard.py"`를 사용 → Python이 `\a`를 BEL(0x07)로 해석 → BAT 파일에 그대로 기록

**영향**:
- sync.py까지는 정상 실행
- apply_standard.py 단계에서 cmd가 `scriptspply_standard.py`(보이지 않는 BEL 포함) 경로를 못 찾아 실패
- 결과: 컴팩트 직전 사용자가 캡처한 *"M is not a command"* 류 에러의 진짜 원인일 가능성 높음

**영향 범위**: 18/18개 .bat (01 검사기 9개 + 02 자동화 8개 + 05 소모품 1개)

**수정 방안**:
```python
# Python BAT 생성 시
content = r"python -X utf8 scripts\apply_standard.py"   # raw string (r-prefix)
# 또는
content = "python -X utf8 scripts/apply_standard.py"    # 슬래시
# 또는
content = "python -X utf8 scripts\\apply_standard.py"   # 더블백슬래시
```

→ 일괄 패치 후 18개 모두 재발행 필요
