## 2. 입력 순서 (2단계)

### 단계 1 — 품목 등록 (소모품목록 시트, 10열)

신규 소모품이 들어오면 먼저 등록합니다.

| # | 입력 컬럼 | 예시 | 비고 |
|---|---|---|---|
| 1 | NO | 1, 2, 3... | 자동 채번 |
| 2 | 품목코드 | CON-001, FIL-001 | 자체 코드 부여 |
| 3 | 품명 | 연결 케이블 2m | |
| 4 | 규격 | M12 / 2m / 24V | 사양 |
| 5 | 단위 | EA / SET / KG / L / BOX | 드롭다운 |
| 6 | 메이커 | SDI / 한일필터 / 미쓰보시 | 제조사 |
| 7 | 기본단가(KRW) | 8500 | 표준 단가 |
| 8 | 카테고리 | 케이블류 / 소모성자재 / 구동부품 / 유지관리 / 센서류 | |
| 9 | 비고 | 메모 | |
| 10 | 등록일 | YYYY-MM-DD | 자동 |

---

### 단계 2 — 출하 기록 (출하관리 시트, 40열)

자재 출하 시마다 한 줄씩 기록. 컬럼이 많지만 **국내 출하는 1~16열만 입력하면 충분**, 해외(베트남) 출하 시 나머지 통관 정보 추가.

#### 기본 영역 (국내 출하 필수, C1~C16)
| # | 컬럼 | 자동/수동 |
|---|---|---|
| 1-2 | 관리코드·수주번호 | (선택) — 특정 프로젝트 출하 시 PMS에서 가져옴 |
| 3-4 | 품명·규격 | 소모품목록에서 자동 또는 수동 |
| 5-6 | 메이커·업체명 | 입력 |
| 7-8 | 단위·원산지 | 드롭다운 / 입력 |
| 9-12 | 수량·통화·단가·금액 | 금액 = 수량 × 단가 (자동) |
| 13 | HS CODE | 해외 통관용 (국내는 빈 칸) |
| 14-16 | 발주일·입고상태·입고일 | 드롭다운(입고완료/입고대기) |

#### 출하·인보이스 영역 (C17~C32)
| # | 컬럼 | 용도 |
|---|---|---|
| 17 | INV번호(출하코드) | 인보이스 번호 (해외 출하) |
| 18 | 출하차수 | 동일 INV 내 분할 출하 시 차수 |
| 19-21 | 거래유형·통관구분·통관분류 | 해외 통관 분류 |
| 22-23 | 환율·상승률(%) | 환차손익 추적 |
| 24-25 | 출하일·출하상태 | 출하완료 / 진행중 / 미출하 |
| 26-27 | DUTY(%)·VAT(%) | 관세율·부가세율 |
| 28-31 | 인보이스단가/금액 (KRW·USD 양쪽) | 환산값 |
| 32-33 | 관세 (KRW·USD) | 통관세 |

#### 박스·통관 영역 (C34~C40)
| # | 컬럼 | 용도 |
|---|---|---|
| 34-37 | BOX번호·BOX수량·무게(Kg)·PALLET SIZE | 물류 정보 |
| 38 | 통관상태 | 통관완료 / 진행 / 대기 |
| 39 | VN입고일 | 베트남 법인 입고 확인일 |
| 40 | 비고 | 메모 |

## 6. 자주 묻는 질문

**Q. 출하했는데 대시보드에 안 보여요.**
A. 출하관리 시트의 `품명` 또는 `금액` 컬럼이 비어있을 가능성. 둘 다 채우면 대시보드 매출총괄 시트의 "소모품" 행에 자동 집계됩니다.

**Q. 같은 품목을 여러 번 출하하면 매번 등록해야 하나요?**
A. 네. 출하관리 시트는 출하 이력을 누적합니다. 품목 등록(소모품목록)은 한 번만.

**Q. 베트남 출하인데 통관 컬럼이 너무 많아요.**
A. 우선 INV번호·환율·HS CODE·DUTY·VAT 5개만 채우면 됩니다. 나머지는 통관 완료 후 단계적으로 채워도 됩니다.

**Q. 현재 데이터가 0건이에요.**
A. 2026-05-01 감사 시점 빈 양식 상태. 시뮬 데이터 / 실데이터 / 빈 양식 유지 중 어느 것으로 운영할지는 대표 결정 사항.

**Q. 향후 자재마스터 ERP가 도입되면?**
A. 현재 소모품 모듈은 그대로 유지하면서, 자재마스터에서 가져오는 구조로 발전 예정. (보류 폴더 _03/_04 자재 ERP 설계 후 재개)
