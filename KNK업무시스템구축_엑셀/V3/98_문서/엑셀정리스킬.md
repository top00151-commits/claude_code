# KNK 업무일정표 엑셀 정리 스킬 (v2026.04e — 매출 사이클 + 부서 통합 + 대시보드 9시트 반영)

> **2026-05-01 갱신** (100_HAIST_WORKS_엑셀 빅터)
> - PMS 23/22열 → **33/31열** (부서 8/7 + 매출 2 추가)
> - freeze F5 → **자동검출 (I5/H5 등 품명 뒷쪽까지)**

## 목적
㈜케이엔케이의 업무용 엑셀 파일을 **일관된 KNK 표준 양식**으로 정리·생성·유지보수한다. KNK PMS V2~V4 개발 과정에서 시행착오로 정립된 모든 규칙을 이 한 문서에 집약한다. 이 문서의 규칙을 벗어나는 개별 조작을 금지한다.

---

## ★ 운영 원칙 (반드시 준수 — 재작업 방지용)

### 원칙 1. 단일 통합 스크립트 원칙
- 엑셀을 수정·재빌드한 뒤에는 **반드시 `apply_knk_standard.py` 1개만 실행**한다.
- 여러 개의 보정 스크립트를 쪼개서 순차 실행하지 않는다(서식 파괴 위험).
- `apply_knk_standard.py`는 이 문서의 모든 규칙(1~20번)을 한 번에 적용한다.

### 원칙 2. build.py 완성형 원칙
- 엑셀을 새로 빌드할 때도 이 문서의 모든 규칙이 이미 반영된 상태로 생성되어야 한다.
- build.py 수정 후에는 반드시 `apply_knk_standard.py` 재실행으로 확인.

### 원칙 3. sync.py 서식 보호 원칙
- sync는 **데이터(값)만 변경**하고 서식(폰트/정렬/색상/너비/freeze)을 파괴하지 않는다.
- sync 종료 시점에 _sort_and_pad_proj 등 정렬 함수는 있으나, 이들도 value + number_format만 복원하고 나머지는 건드리지 않는다.

### 원칙 4. 자동 검증 원칙
- 작업 완료 후 `diag.py` 자동 진단 스크립트로 체크리스트 검증한다.
- 진단 실패 항목이 있으면 `apply_knk_standard.py` 재실행.

## 7. 금액 셀 서식 (★ 중요)

### 7.1 통화별 포맷
```python
ACCT_KRW = '₩#,##0;(₩#,##0);"-"'
ACCT_USD = '$#,##0.00;($#,##0.00);"-"'
```

### 7.2 적용 규칙
- **통화 컬럼(C13 통화)의 값(KRW/USD)에 따라 금액·단가 셀의 number_format을 동적으로 지정**
- 우측 정렬
- 음수 괄호, 0은 "-"

### 7.3 환율 자동 채움 금지 (★ v2026.04)
- **KRW든 USD든 환율 값은 자동으로 넣지 않는다**
- KRW는 환율이 불필요하므로 빈 칸
- USD는 사용자가 직접 환율 입력 (예: 1320)
- **임포트·sync 어느 경우에도 환율 1, 1320을 자동 채우지 않는다**

### 7.4 수량 셀 서식 (★ v2026.04)
- number_format: `#,##0`
- 정렬: **중앙** (숫자라도 수량은 가운데)
- BOX수량도 동일

### 8.2 데이터 기반 자동조정 (필수 실행)
**데이터 입력 후** 반드시 재계산. 한글은 2폭 환산.

```python
def auto_width(ws, c, min_w=5, max_w=40):
    header = ws.cell(4, c).value
    h_max = 0
    if header:
        for line in str(header).split('\n'):
            h_max = max(h_max, sum(2 if ord(ch) > 127 else 1 for ch in line.strip()))
    data_max = 0
    for r in range(5, min(ws.max_row + 1, 300)):
        v = ws.cell(r, c).value
        if v is not None:
            data_max = max(data_max, sum(2 if ord(ch) > 127 else 1 for ch in str(v)))
    std = header_std_width(header)  # 표준 룩업
    base = max(h_max, data_max)
    if std:
        return max(min_w, min(max_w, max(std, base + 1)))
    return max(min_w, min(max_w, base + 2))
```

### 8.3 원칙
- min 5, max 40
- 데이터 max > 표준 → 데이터 max 채택
- sync/apply 종료 시점에 반드시 재호출

## 11. 관리코드(관리번호) 채번 방식

### 11.1 구조
```
[일련번호 3자리] + [사업부코드 1자리] + [연월 4자리]   = 8자리
```
- 사업부: `T`=검사기, `M`=자동화
- 예: `001T2604`, `015T2605`, `001M2604`

### 11.2 채번 규칙
| 영업단계 | 채번 |
|---|---|
| 제안작성/제안제출 | 채번 안 함 |
| 수주확정 이상 | **자동 채번** |
| 추가PO | 기존 코드 유지 |

| **고객사담당자(E4)** | **"고객사담당자 이름 작성"** ★ 하드코딩 |
| 수주일/발주일 | "수주일\n입력 YYYY-MM-DD (자동 변환)" |
| D-day | "D-day\n자동 — (오늘 - 납기일), 역카운터" |
| 진행상태 | "진행상태\n선택 — 수주예정/진행중/납품완료/취소/보류" |

### 14.4 VML 강제 적용 (openpyxl 버그 우회)
openpyxl은 저장 시 메모 박스 크기를 144x79 px로 reset함. xlsx 내부 `xl/drawings/commentsDrawingN.vml`의 width/height를 **380/110 px로 강제 재작성**해야 한다.

## 15. D-day 역카운터 (★ v2026.04)

### 15.1 계산 방식
```python
dday = (today - due).days
```
- **오늘 납품 = 0**
- **내일 납품 = -1** (아직 남음, 마이너스)
- **모레 납품 = -2**
- **어제 지남 = +1** (지연, 플러스)
- **3일 전 지남 = +3**

### 15.2 number_format
```
+0"일";-0"일";"0일"
```
- 양수 → `+3일` (지연)
- 음수 → `-1일` (남음)
- 0 → `0일` (당일)

### 15.3 납품완료·취소·보류 → 빈 칸
```python
if status in {"납품완료", "완료", "취소", "보류"}:
    dday_cell.value = None
```
