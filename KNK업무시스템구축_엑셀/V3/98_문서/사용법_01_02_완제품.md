## 2. 입력 순서 (3단계)

### 단계 1 — 영업·PM: 신규 수주 등록 (PMS 파일 33/31열)

`KNK_*_완제품_PMS_2026.xlsx` 열기 → `1_프로젝트등록` 시트

신규 수주는 **R5~R14 빈 공간** 또는 **Ctrl+End 후 다음 행**에 입력:

#### 영업·수주 영역 (필수)
| 컬럼 | 내용 | 자동화 |
|---|---|---|
| 고객사 | 삼성전자·삼성전기·드림텍 등 | NO·관리코드·수주번호 자동 |
| 고객사담당자 | 거래처 측 담당자명 | |
| **제품구분** | PBA / TSP / SENSOR / 기타 (드롭다운, **01만**) | |
| 모델 | 장비 모델명 | |
| 품명 | 장비 품명 | |
| PO유형 | 신규 / 추가 (드롭다운) | |
| 영업단계 | 제안작성→제안제출→수주확정→납품→A/S | 수주확정 시 코드 발번 |
| 담당영업·PM | 직원명 | |
| 수량·통화·단가 | 수량×단가 → 금액 자동 계산 | 금액 (자동) |
| 수주일·납기일 | YYYY-MM-DD | D-day (자동) |
| 출하경로 | K→고객사 / K→V→K→고객사 / K→V→고객사 | |
| 진행상태 | 수주예정 / 진행중 / 납품완료 / 취소 / 보류 | |

#### 부서 영역 (★ v2026.04e — PMS에 통합)
PMS 1_프로젝트등록 우측에 **부서 컬럼 8개(01) / 7개(02)** 가 있습니다. 각 부서 담당자명을 직접 입력:

| 컬럼 | 입력 | 의미 |
|---|---|---|
| 설계팀 | "정민규" | 설계팀 참여, 담당자 정민규 |
| 검사기팀 | "한재운" | 검사기팀 참여, 담당자 한재운 |
| 가공팀 | (빈 칸) | **가공팀 미참여 → 진척률 계산에서 자동 제외** |
| 가공팀 | "제외" | 동일 효과 (호환성, 명시적 제외) |

부서 컬럼은 영업·PM이 PMS에서 직접 부서 배정을 보고 의사결정합니다.

#### 매출 영역 (★ v2026.04e — 신규)
| 컬럼 | 입력 시점 | 후속 동작 |
|---|---|---|
| 계산서발행일 | 납품 완료 후 기술영업팀(안지연·이새롬)이 발행 시 | 매출_미수금 시트로 이동 |
| 입금일 | 고객사 입금 확인 시 | 8_매출마감으로 자동 이관 + 매출_완납YTD 누적 |

#### 자동 채번 규칙
- 관리코드: `T0012605` (T=검사기 / M=자동화 + 일련번호 + YYMM)
- 수주번호: `T-260501` (사업부-YYMMDD, 동일일자 N건은 `-2`, `-3`)
- `영업단계 = 제안작성/제안제출`인 경우 채번 안 함 (수주확정 시점에 발번)

#### 시나리오별 입력 흐름 (★ v2026.04f 정리)

**🟢 시나리오 A — 신규 프로젝트**
| 컬럼 | 입력 |
|---|---|
| C2 관리코드 | **비워둠** → .bat 실행 시 자동 채번 |
| C3 수주번호 | 비워둠 → 자동 채번 |
| C4·C7·C8 등 | 정상 입력 |
| **C10 영업단계** | **`수주확정` 이상** ← 채번 트리거 |
| C17 수주일 | 정확한 일자 입력 (수주번호 base) |

→ .bat 실행 시 관리코드(예: `001T2605`) + 수주번호(예: `T-260501`) **모두 자동 발번**.

**🟠 시나리오 B — 추가 PO (기존 검사기 추가 발주)**

방법 ①: **같은 행에 누적 (권장 — 부서 진척률 통합 관리)**
- 기존 행의 C13 수량 / C15 단가만 갱신
- 비고에 추가 PO 이력 메모

방법 ②: **별도 행 + 기존 관리코드 직접 입력 (이력 추적 필요 시)**
- C2: 기존 코드 직접 입력 (예: `005T2604`)
- C3: 비워둠 (자동 채번)
- C9 PO유형: `추가`
- ⚠️ **주의**: 같은 관리코드 여러 행이 있으면 부서 입력 파일에서는 **마지막 행의 진척률만 살아남음**. PMS 진척률 write-back은 **첫 번째 행에만** 기록.

**📋 영업단계별 자동 정렬 (★ v2026.04f 명시)**

sync 실행 시 1_프로젝트등록 시트는 다음 순서로 자동 정렬:

| 우선 | 그룹 | 내부 정렬 | 비고 |
|---|---|---|---|
| 1 | 진행중·수주예정 | 납기일 ASC (가까운 순) | 최상단 |
| 2 | 제안작성·제안제출 | 수주일 ASC | 관리코드 없어도 보존 |
| 3 | 보류 | 납기일 ASC | |
| 4 | 취소 | 납기일 ASC | 최하단 |
| — | 납품완료·완료 | (제외) | **4_완료이력으로 자동 이동** |

→ 매일 아침 PMS 열면 가장 위에 **납기 임박한 진행중 건**이 정렬되어 있음. 즉시 의사결정 자료.

### 단계 2 — 부서별 담당자: 진척 입력

각 부서 파일 열기 → `1_부서진척입력` 시트

| 자동 표시 (PMS에서 sync) | 입력 항목 |
|---|---|
| 관리코드·수주번호·고객사·품명·납기일·진행상태 | 세부 항목 1~N별 진척률 (0~1 소수) |

**입력 비율은 0~1 사이 소수**로 입력 (예: 0.8 = 80%)

부서별 세부항목 수:
- 01 검사기: 설계(4)·검사기(5)·개발혁신(3)·품질(3)·제조기술1(2)·구매(4)·가공(2)·베트남(5)
- 02 자동화: 설계(5)·전장설계(4)·소프트웨어(4)·제조기술2(4)·구매(5)·가공(3)·베트남(7)

**중요**: PMS 1_프로젝트등록의 부서 컬럼이 빈 칸이면 해당 부서 입력 파일에 그 프로젝트가 자동 표시되지 않습니다. (담당자명 입력 = 참여 의사)

---

### 단계 3 — sync 실행 (★ 더블클릭 권장)

각 폴더의 **.bat 파일을 더블클릭**하면 sync + 표준화가 자동 실행됩니다.

| .bat 파일 | 누가 |
|---|---|
| 실행_PMS(영업·PM).bat | 영업·PM이 자기 입력 후 |
| 실행_설계팀.bat | 설계팀장이 자기 입력 후 |
| 실행_검사기팀.bat | 검사기팀장 |
| ... | 각 부서 |

자동 처리:
1. 신규 수주에 관리코드·수주번호 발번
2. 부서진척 → PMS 전체 진척률 자동 집계
3. D-day 갱신
4. 입금일 채워진 항목 → `8_매출마감` 자동 이관 (★ v2026.04e)
5. 진행상태 = 납품완료/취소/보류 → `4_완료이력` 자동 이동
6. `2_진행현황` 시트 갱신
7. `3_출하현황` 시트 갱신

## 5. 시트 보호 + ⚠️ 데이터 초기화

### 5.1 시트 보호
- 비밀번호: `knk2026`
- 자동 입력 셀(회색)은 잠금 상태

### 5.2 ⚠️ 데이터 초기화 (검증/실데이터 전환 전용)

각 모듈 폴더의 **`데이터초기화.bat`** 더블클릭

| 위치 | 파일 | 비밀번호 |
|---|---|---|
| 01 검사기 | `01_검사기_완제품/데이터초기화.bat` | **knk1234** |
| 02 자동화 | `02_자동화_완제품/데이터초기화.bat` | **knk1234** |
| 05 소모품 | `05_소모품/데이터초기화.bat` | **knk1234** |

**동작 흐름**:
1. 비밀번호 입력 (`knk1234`) — 틀리면 즉시 종료
2. 마지막 확인 `Y` 입력 — 다른 키면 취소
3. `_참고용_원본데이터/{타임스탬프}/` 에 **현 xlsx 자동 백업**
4. R5+ 모든 데이터 삭제 (R1~R4 헤더·서식·드롭다운·메모는 보존)
5. sync + apply_standard 자동 실행 → 빈 양식 정규화 완료

**중요**:
- **부서 입력 파일도 같이 초기화됨** (모듈 폴더 직속 모든 xlsx)
- 백업이 자동이므로 실수 시 `_참고용_원본데이터/` 에서 복원 가능
- 비밀번호 변경 필요 시 빅터에게 요청
- ⚠️ **실 운영 데이터 입력 후에는 사용 금지** — 검증 단계나 시즌 전환 등 의도된 초기화에만 사용
- 입력 셀(녹색·주황·보라)만 수정 가능

## 6. 매출 사이클 (★ v2026.04e — KNK 영업 흐름 반영, 2026-05-01 dry-run 검증 완료)

### 6.1 정확한 흐름 (sync.py `_archive_settled` 로직 기반)

```
[1_프로젝트등록]
   │ 영업단계 = 수주확정 → 관리코드·수주번호 자동 발번
   │ 부서별 담당자명 입력 → 부서 진척 자동 집계
   │ 진행상태 = 진행중
   ↓
[진행상태 = 납품완료] → sync 실행
   ↓
[4_완료이력 시트로 자동 이동] (1_프로젝트등록에서 사라짐)
   │
   │ ★ 4_완료이력 시트의 우측 끝 C32·C33에서 매출 추적
   ↓
[기술영업팀(안지연·이새롬) 세금계산서 발행]
   │ 4_완료이력의 해당 행 → 계산서발행일(C32) 입력
   │ → sync 실행
   ↓
[대시보드 매출_미수금 시트에 표시] (계산서 발행 후 입금 대기)
   ↓
[고객사 입금 확인]
   │ 4_완료이력의 해당 행 → 입금일(C33) 입력
   │ → sync 실행
   ↓
[8_매출마감 시트로 자동 이관] (4_완료이력에서 사라짐, 영구 보존)
   │
   ↓
[대시보드 매출_완납YTD 시트 누적] (월별 집계)
```

### 6.2 단계별 입력 위치 정리

| 단계 | 입력 위치 (시트·컬럼) | 입력자 |
|---|---|---|
| ① 수주 등록 | 1_프로젝트등록 C4~C23 + C24~C31 부서 | 영업·PM |
| ② 부서 진척 | 부서별 입력 파일 → sync로 PMS 집계 | 부서장 |
| ③ 납품완료 처리 | 1_프로젝트등록 C22 진행상태 = `납품완료` | PM |
| ④ sync → **자동으로 4_완료이력으로 이동** | (자동) | 시스템 |
| ⑤ 계산서 발행 | **4_완료이력** C32 계산서발행일 | 기술영업(안·이) |
| ⑥ 입금 확인 | **4_완료이력** C33 입금일 | 회계 |
| ⑦ sync → **자동으로 8_매출마감으로 이관** | (자동) | 시스템 |

### 6.3 주의 사항
- **계산서·입금일 입력 위치는 4_완료이력 시트** (1_프로젝트등록 아님)
- 1_프로젝트등록에도 동일한 C32·C33 컬럼이 있으나, 진행중 상태에서 입력해도 8_매출마감 이관은 일어나지 않음 (납품완료 → 4_완료이력 경유 필수)
- 선결제·중도금이 발생해도 일단 1_프로젝트등록의 진행상태가 `진행중`이면 미수금/완납 시트에 반영되지 않음. 납품완료 처리 후 4_완료이력에서 매출 사이클 진행해야 정상 집계.

## 7. 자주 묻는 질문

**Q. 수주가 취소됐어요. 어떻게 하나요?**
A. 진행상태를 `취소`로 변경 → sync 실행. 4_완료이력으로 자동 이동. 대시보드에서 자동 제외됩니다.

**Q. 납품 완료된 프로젝트가 PMS에서 안 보여요.**
A. 진행상태가 `납품완료`면 `4_완료이력` 시트로 자동 이동했습니다. 입금까지 완료되면 `8_매출마감`으로 영구 이관됩니다.

**Q. 진척률이 0%로만 표시돼요.**
A. (1) 부서별 입력 파일에서 진척률을 입력하지 않았거나 (2) sync를 실행하지 않은 상태입니다. (3) PMS 부서 컬럼이 빈 칸이면 해당 부서가 진척률 계산에서 제외됩니다.

**Q. 관리코드가 발번되지 않았어요.**
A. 영업단계가 `제안제출`이거나 비어있습니다. `수주확정` 이후 sync 실행하면 발번됩니다. (제안 단계는 의도적으로 채번 X — 수주 전 추적용)

**Q. 추가 PO가 들어왔는데 별도 행으로 넣을지 같은 행에 누적할지?**
A. 부서별 진척률 추적이 별도 필요 없으면 **같은 행에 수량·금액 누적이 단순**. 별도 추적 필요 시 새 행 + 기존 관리코드 직접 입력하되, **부서 입력 파일에서는 마지막 행 진척률만 살아남음**에 주의.

**Q. 제안 단계 프로젝트는 어떻게 정렬되나요?**
A. 진행중·수주예정 다음 그룹(2번째)에 표시됩니다. 같은 그룹 내에서는 **수주일이 가까운 순**. 관리코드 없어도 행 보존되어 추적 가능.

**Q. 같은 관리코드를 여러 행에 두면 진척률은 어떻게 집계되나요?**
A. 부서 입력 파일에서 같은 관리코드가 여러 행이면 **마지막 행 값만 progress dict에 저장**(덮어쓰기). PMS 1_프로젝트등록 진척률 write-back은 **첫 번째 행에만** 기록. → 운영상 같은 관리코드는 한 행으로 유지 권장.

**Q. 같은 관리코드인데 모델·품명을 잘못 다르게 입력하면?**
A. v2026.04g부터 **자동 검증 + 보정·경고**가 작동합니다:
- **빈 칸**인 모델·품명 → 가장 위(첫 발견) 행의 값으로 **자동 채움** (sync 로그에 기록)
- **다른 값** 입력 → **노란색 셀(#FFEB9C)** 표시 + sync 로그 경고
- 노란색 셀 발견 시 사용자가 직접 검토:
  - 같은 검사기인데 잘못 입력 → 권위 행 값으로 수정
  - 진짜 다른 검사기 → 별개 관리코드 부여 (현 코드 지우고 sync 재실행)
- 권위 결정: 같은 관리코드 행 중 **PMS 시트 위쪽(R 작은) 행** = 보통 정렬 상 가장 최근 진행건

**Q. 가공팀은 이번 프로젝트에 안 들어가는데?**
A. PMS 1_프로젝트등록의 가공팀 컬럼을 **빈 칸으로 두면 자동 제외**됩니다. 또는 `제외`라고 명시 입력해도 동일.

**Q. 계산서 발행했는데 어디에 기록하나요?**
A. PMS 1_프로젝트등록 시트의 `계산서 발행일` 컬럼(C32/C30)에 발행 일자 입력. sync 시 대시보드 `매출_미수금`으로 자동 이동.

**Q. 입금까지 완료됐는데도 PMS에 보여요.**
A. `입금일` 컬럼(C33/C31)을 입력하지 않았거나 sync를 안 돌렸습니다. 입금일 입력 + sync = 8_매출마감으로 이관.

**Q. .bat 더블클릭하니 한글이 깨져요.**
A. 2026-05-01 BEL 0x07 버그 패치 완료 + UTF-8 BOM 적용. 현 시점은 정상이어야 합니다. 만약 여전히 깨지면 빅터에게 알려주세요.
