## 1. 대표 직접 발견 10건 분석 (2026-04-28)

### A. 시각 fidelity (회사 공식 자료와 비교)
- 로그인 페이지 로고가 본사 공식 KNK 3글자 로고 ≠ 인라인 SVG K-N-‖‖‖-K (4 막대)
- 헤더 로고 흰 배경·작은 사이즈·칙칙한 회색 → 투명·확대·brightness 1.2 필요

### B. 로그인 BEFORE 화면
- `?lang=ko/vi/en` 작동 안 함 (GET /login 핸들러가 lang 파라미터 무시)
- 로그인 페이지에서 인사말·로고·언어 선택 등 **첫인상 영역** 검증 부재

### C. 서브 도메인 Hub 누락
- `base.html` (메인) 검증은 04 가 했으나
- `base_sales.html` (/sales)·`base_logi.html` (/logistics) 까지는 안 들어감
- 같은 결함이 3개 파일에 동시 존재해도 1개만 발견

### D. UI 잔존물
- 사이드바 ◀ 토글이 기능 제거됐지만 DOM 에 남아있음
- 사용 안 하는 버튼·메뉴·아이콘이 화면에 노이즈

### E. 시각 겹침/가림
- 빅터 도크가 헤더 위로 떠서 사용자 칩·로그아웃·언어 셀렉터 가림
- HTTP 본문만 봐서는 절대 안 보임 — 시각으로만 발견 가능

### F. i18n 침투 깊이
- `/api/set-lang` POST 작동 ✅ (04가 검증한 부분)
- 그러나 사이드바 메뉴 라벨이 하드코딩 한국어 → 번역 안 됨 (04가 못 본 부분)

## 2. v3 신규 Level 추가

기존 Level 1~4 에 더해:

### Level-5 — **회사 공식 자료 fidelity** (신규)

**대상**:
- 로고 (`회사 로고.JPG`, `260330_KNK 로고 메뉴얼 (1).pdf`)
- 회사명 표기 (KNK·㈜케이엔케이·HAIST Innovation 일관성)
- 색상 팔레트 (KNK RED `#A5282C` 등 브랜드 토큰)
- 직급 호칭 (프로/매니저/사원/대리 등 인사 데이터 일관성)

**검증 방법**:
- 회사 공식 자료 경로 보유: `C:\Users\top00\JR\01_업무관련\98_문서양식\공용양식\본사\회사기본정보\`
- 매 라운드마다 로그인 페이지·홈·로고 등 회사 정체성 표시 영역 = 회사 공식 자료와 시각 비교
- 불일치 시 즉시 보고

### Level-6 — **로그인 BEFORE 검증** (신규)

**대상** (모두 비로그인 상태):
- `/login` 페이지 자체 (로고·인사말·언어 셀렉터·placeholder·도움말)
- `?lang=ko`, `?lang=vi`, `?lang=en` 쿼리 파라미터 작동 여부
- `/login?error=...` 에러 표시 일관성
- 회사 메인 도메인 인입 시 첫 화면 (404·500·robots.txt 등)

**검증 방법**:
- 매 라운드 시작 시 비로그인 상태에서 `/login` 진입 → 모든 인터랙션 클릭 시도
- 언어 토글 3종 시도 → 페이지 텍스트 vi/en 변환 확인

### Level-7 — **서브 도메인 Hub 회귀 매트릭스** (신규)

**대상 (3종)**:
- `/home` (메인 통합) — `base.html`
- `/sales` (매출·영업 Hub) — `base_sales.html`
- `/logistics` (자재·구매 Hub) — `base_logi.html`

**검증 방법**:
- 매 결함 발견 시 동일 결함이 3종 모두에 있는지 자동 검증
- "메인에서만 고치고 Hub 빠뜨림" 패턴 = 대표가 자주 발견하는 케이스
- grep 매트릭스: `grep -c "결함패턴" app/templates/base*.html` (3종 동시)

### Level-8 — **UI 잔존물·dead code 검증** (신규)

**대상**:
- 클릭해도 동작 안 하는 버튼
- 기능 제거됐는데 DOM 에 남은 요소 (`.sb-toggle` 같은 흔적)
- href="#" dead link
- 사용 안 하는 사이드바 메뉴 항목

**검증 방법**:
- 페르소나 시뮬 중 "이 버튼 누르면 뭐 나올까?" 호기심 클릭 매번 시도
- 사이드바·헤더·도크의 모든 클릭 가능 요소 1회 클릭

### Level-9 — **시각 겹침·가림** (신규)

**대상**:
- 도크·모달·팝오버가 헤더·사이드바·메인 컨텐츠를 가리는지
- 모바일 뷰포트에서 햄버거 메뉴가 컨텐츠 위에 뜨는지
- 알림 토스트가 액션 버튼을 가리는지

**검증 방법**:
- 스크린샷 영역 캡처 (1024×500 등) 로 시각 검증
- 도크 열림 상태에서 헤더 사용자 정보·로그아웃 버튼 표시 확인
- z-index 충돌 패턴 시각 검증

### Level-10 — **i18n 침투 깊이** (신규)

**대상**:
- 사이드바 메뉴 라벨 (sb_my_work, sb_home, sb_daily, sb_calendar 등)
- 헤더 버튼·플레이스홀더
- 모달 제목·버튼
- 에러 메시지·flash 배너
- 일일카드 카테고리 select 옵션

**검증 방법**:
- `vi`·`en` 언어 적용 후 페이지 본문 grep 으로 한국어 키워드 잔존 검증
- 잔존 한국어 ≥ 1 → i18n 누락 보고
- i18n.py 의 키 수 (현재 472+) 와 실제 템플릿 사용 키 수 매칭
