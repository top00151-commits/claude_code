## 🟠 S2 (대체수단 필요) — 완료는 되지만 업무가 더 커짐

### [OPS-002] `/api/task` 입력검증 전무 → 쓰레기 데이터 허용

**재현** (엄주영 로그인 후):
```
POST /api/task {"title":"", "hours":0}           → 200 OK (빈 제목 저장됨)
POST /api/task {"title":"음수 테스트", "hours":-5} → 200 OK (-5시간 저장)
POST /api/task {"title":"40시간 하루", "hours":40} → 200 OK (하루 40h 저장)
```

**영향**:
- 전사 대시보드·팀장 집계가 비정상값에 오염됨 ("오늘 팀 업무 40시간 초과" 같은 허위 표시 가능).
- 신입이 **아무 생각 없이** 테스트 카드를 만들다가 대시보드가 더럽혀짐.
- "자료를 정확하게 표현하는가" 원칙 위배.

**담당 세션**: 01 메인

**수정 요청**:
1. `app/main.py:441 api_create_task` 에 서버측 검증 추가:
   - `title` 공백이면 400
   - `hours` 범위 `[0, 24]` 외면 400
   - `status` 가 `STATUSES` 목록(`["진행중","완료","지연","대기","보류"]`)에 없으면 400
2. PUT `/api/task/{tid}` 에도 동일 검증.
3. 프런트 버튼 비활성화는 **부가** 장치일 뿐. 서버 검증이 1차 방어선.

### [OPS-003] 베트남 직원 언어 변경이 "겉에만" 작동

**재현** (쑤아잉 로그인):
- `POST /api/set-lang {"lang":"vi"}` → 200, DB `users.lang = 'vi'` 업데이트 ✅
- `GET /home` 재방문 → "Công việc" 4회, "Hôm nay" 1회 등장 (부분 번역 적용)
- 그러나 `GET /api/set-lang?lang=vi` (쿼리스트링 방식) → **405 Method Not Allowed**

**영향**:
- 베트남 직원이 언어를 바꾸려면 **서버 API를 직접 호출하는 수준의 이해가 필요**.
- 실제로 UI 메뉴에 언어 드롭다운이 구현돼 있어야 쑤아잉·땀·탕이 쓸 수 있음. 현재는 `/profile` 에도 언어 변경 UI가 노출되어 있는지 불투명.

**담당 세션**: 01 메인

**수정 요청**:
1. `/profile` 템플릿에 언어 드롭다운 (ko/vi/en) 명시적 배치 — 저장 버튼 눌렀을 때 JSON fetch 호출.
2. topbar 에도 베트남 국기 토글 추가 고려 (베트남 법인 5명이 매번 프로필까지 들어가지 않도록).
3. i18n 번역 적용이 제한적이므로 커버리지 체크: 사이드바·탑바·버튼의 키 누락 추적.

### [OPS-004] cockpit 페이지 100KB / onclick 226회 — 모바일·출장 사용 불가 위험

**재현** (kjr 로그인):

- `/admin` 235 KB, `/progress` 230 KB.

**영향**:
- 현장·출장·해외 셋업 중인 P8/P9/P10 페르소나는 모바일·약한 회선 기준. 100KB HTML+onclick 수백 개는 초기 렌더 3초 이상 예상.
- CEO 가 "3분 내 전사 파악"을 목표로 하는데, 페이지 로딩만 2~5초 걸리면 의미 훼손.

**담당 세션**: 01 메인

**수정 요청** (우선순위 낮음, 관찰 사항):
1. cockpit 의 중요 KPI 를 먼저 렌더하고 하위 표는 지연 로딩.
2. inline onclick → event delegation 으로 축소.
