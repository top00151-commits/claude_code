## 2. 의뢰 사항

### 2-1. 즉시 (오늘~내일)

#### A. 표준 페르소나 매트릭스 갱신
`01_시나리오_페르소나별.md` (또는 round3 후속) 에 다음 시나리오 **신규 추가**:

```
[T-MTX-1] 매트릭스 가로 스크롤 + 좌측 핀 컬럼 검증
  - P1 영업: /sales-outstanding 진입 → 우측 끝까지 가로 스크롤
    → 좌측 [거래처/관리코드] 컬럼 화면에 보이는가
  - P3 PM: /progress 진입 → 우측 끝까지 가로 스크롤
    → 좌측 [관리코드/프로젝트] 컬럼 화면에 보이는가
  - P6 자재: /stock-safety 진입 → 우측 끝 스크롤
    → 좌측 [부품번호/부품명] 화면에 보이는가
  PASS 조건: 좌측 2 컬럼이 가로 스크롤과 무관하게 항상 화면에 보임

[T-MTX-2] 매트릭스 세로 스크롤 + 컬럼 헤더 sticky 검증
  - 동일 페르소나·페이지에서 50행+ 까지 스크롤
  - 컬럼 헤더(thead) 가 화면 상단에 sticky 되어 있는가
  PASS 조건: 모든 페이지에서 thead sticky 동작

[T-MTX-3] 매트릭스 컨테이너 max-height 동작 검증
  - 페이지 로딩 후 가로 스크롤바 위치 확인
  - 페이지 맨 아래까지 스크롤하지 않고도 매트릭스 우측 컬럼 보기 가능한가
  PASS 조건: 가로 스크롤바가 컨테이너 (max-height: calc(100vh-320px)) 안에 위치

[T-FILT-1] 필터 버튼 라벨 일관성 검증
  - 시스템 12개 페이지 (progress/sales/stock/changes/issues/tickets/parts/po/projects/movements/suppliers/admin) 의 필터 버튼 라벨 grep
  - 동일 의도(폼 submit)의 버튼이 동일 라벨인가
  PASS 조건: 일반 사용자 페이지 모두 "검색" 통일

[T-FILT-2] 필터 적용 후 chip 표시 검증 (A 등급 — 1차 마감 후)
  - 필터 적용 후 상단에 적용된 조건 chip 표시되는가
  PASS 조건: chip 또는 같은 가독성의 표시 존재
```

#### B. DOM/CSS 정적 검증 자동화 추가
`scan_pages.py` 또는 신규 `mtx_layout_check.py` 작성:

```python
# 매트릭스 페이지 5종에 대해 정적 grep 검증
PAGES = ['progress_matrix.html', 'sales_outstanding.html', 'sales_aging.html',
         'stock_safety.html', 'stock_reorder.html']

REQUIRED = [
    ('class="mtx-scroll"', '컨테이너 표준 클래스'),
    ('col-pin-1', '1번 핀 컬럼'),
    ('col-pin-2', '2번 핀 컬럼'),
]
FORBIDDEN = [
    (r'>필터</button>', '비표준 필터 라벨'),
    (r'overflow-x:\s*auto.*min-width:\s*1[0-9]{3}px(?!.*max-height)', 
     'max-height 없는 가로 스크롤 컨테이너'),
]
# 5 × 3 PASS + 5 × 2 FAIL = 25개 grep 매트릭스
```

→ HTTP 시뮬에 더해 정적 분석 추가하면 본 결함류 즉시 잡을 수 있음.

### 2-3. 중장기 (Round 4+)

#### D. 페르소나 walkthrough 표준화
HTTP 시뮬 + DOM 정적 + 시각 인지 walkthrough = 3중 검증 매트릭스 구축.

매트릭스류 페이지 (sticky·스크롤·페이지네이션·필터)는 시각 인지 walkthrough 없이는 HTTP만으론 잡히지 않음. 04의 가용 도구 범위에서 정적 grep + Playwright headless screenshot diff 등을 검토해 주세요.
