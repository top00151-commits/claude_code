## 대표 관찰 결함 4건 (재확인)
1. 사이드바 하단 모니터에 안 보임
2. 해상도 변화 시 텍스트 크기 부적응
3. 메인 영역 글씨 작음
4. 메인 키우면 메뉴 글씨 더 커짐

## 즉시 측정할 것 — 해상도 3종 화면 캡처

### 측정 환경
- **데스크톱 1920×1080** (FHD 표준)
- **데스크톱 1440×900** (노트북 표준)
- **데스크톱 2560×1440** (QHD, 대표 환경 추정)
- **모바일 375×812** (iPhone 13 기준)

### 페르소나 1명 고정 (정성진 구매팀장)
- 다른 페르소나는 다음 라운드. 결함 패턴은 동일할 것.

### 측정 항목 (해상도당)
1. **사이드바 가시 영역** — 화면 높이·사이드바 콘텐츠 높이·잘리는 픽셀 수
2. **사이드바 메뉴 폰트** — px 실측 (개발자도구 computed)
3. **메인 본문 폰트** — px 실측 (대표가 "작다"고 한 영역 특정)
4. **브라우저 줌 100% / 110% / 125% / 150%** — 각각 메뉴 vs 본문 폰트 비율
5. **사이드바 스크롤 가능 여부** — 잘리면 스크롤로 보이는가?

### 산출 형식
```
| 해상도 | 사이드바 표시 % | 메뉴 폰트(px) | 본문 폰트(px) | 줌 100→125 시 메뉴 증가율 | 줌 100→125 시 본문 증가율 |
|--------|---------------|------------|------------|--------------------|--------------------|
| 1920×1080 | 95% (하단 28px 잘림) | 13 | 12 | +25% | +25% |
| ... |
```

## 회귀 시나리오 (대표 환경 재현)
1. 정성진 로그인 → 홈 도착
2. 사이드바 끝까지 스크롤 시도 → 가능 여부
3. 메인 본문 글씨 크기 측정 (개발자도구 Computed → font-size)
4. 브라우저 Ctrl+ 로 1단계 줌 → 메뉴와 본문 어느 쪽이 더 커지는지 측정
