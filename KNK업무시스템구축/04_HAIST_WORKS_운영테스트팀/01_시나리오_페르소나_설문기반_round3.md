## 시나리오 (하루 업무 흐름 기반)

### PS-01 정성진 — 구매팀 "일 50~70건 발주일"

**배경**: 설문 §10팀 공통 고통 1위 (카톡 출고 요청 누락). 본인은 ERP·엑셀 이중 입력으로 시간 소모. 1순위 기능 중 **요청 티켓 시스템**이 이 업무를 해소해야 함.

**시나리오 (실측 타깃 지표 포함)**:
1. 로그인 → `/home` — **오늘 내 PO 건수 / 요청 티켓 신규 / 지연 알림** 이 홈 KPI 에 보이는가?
2. `/tickets` — 자재요청 티켓이 카톡 없이 여기 들어오는가? 수신 목록 시각화.
3. `/tickets/new` — 구매팀이 **역으로** 현업에 "견적 회신 필요" 티켓 발행 가능?
4. `/logistics` → `/po/new` — 관리자가 `can_use_logistics` 플래그를 켜준 상태를 가정. (2라운드 확인한 OPS-011 해소 조건)
5. `/parts` → 부품 3건 조회 후 PO 반영.
6. 10건 연속 PO 등록 시간 측정 → 50건/일 환산 예상치 계산.

**통과 기준**:
- 티켓 리스트에 "카톡 치기 전에 먼저 봐야 할" 요청이 실제 들어있는가 (시드 데이터 풍부도)
- PO 1건당 입력 필드 수 ≤ 8개 (설문 §2.7 "이중 입력 금지")
- 홈에서 `/tickets` 까지 클릭 ≤ 2회

### PS-02 박지은 — 관리팀 "월말 결산일"

**배경**: 설문 §5 "통합 시스템 → 시간 절감 기대" 단, "데이터 정리 후 단계적 자동화" 강조. 증빙 제출 포털은 로드맵 2순위.

**시나리오**:
1. `/home` — 미처리 결재·증빙 현황 KPI 보이는가?
2. `/dashboard` — **전사 대시보드 접근 권한** (leader 로 2라운드 P5 는 /login 폴백 발생. 수정 확인)
3. `/board/company` — 공지 분류 (결제·일정·안내) 구분 되는가?
4. `/admin` - 관리팀 leader 면 admin 권한 필요한가? (role=leader 인데 admin 필요 업무)

**통과 기준**:
- leader 로 /dashboard 접근 시 200 OK 또는 명확한 권한 안내 메시지
- 증빙·결재 관련 라우트 존재 (또는 부재 시 "미구현" 투명성)

### PS-03 이해림 — 영업 이사 "모델별 진척율 확인"

**배경**: 설문 §3 매일 아침 원하는 정보 1위 = **모델별 진행률**. 1순위 ① 진행률 대시보드가 이 요구 직격.

**시나리오**:
1. `/home` — "우리 프로젝트 진척 요약" 섹션 존재?
2. `/progress` — 진행률 대시보드
3. 개별 프로젝트 `/progress/{project_id}` — 12공정 (수주→컨셉→설계→전장→SW→가공→구매→조립→검수→출하→Set-up→KNKVN) 모두 렌더?
4. 지연 표시: 빨간색 / 기준일 초과 / 담당자 표기 정확?
5. `/api/progress/summary` — JSON으로 자동 요약 데이터?
6. KNKVN(베트남) 이관 추적 가능?

**통과 기준**:
- 개별 프로젝트 상세에 공정 ≥ 10개 렌더
- "담당자 + 예정일 + 실제일" 3 필드가 각 공정에 표시
- 지연 시 시각적 구분 존재

### PS-04 김형렬 — 전장설계팀 "기구→전장 변경 inform 수신"

**배경**: 설문 §2 이슈 3위 (6팀 공통). 기구팀이 Inform 보내면 전장이 1~3일 내 반영. 현재는 카톡 or 메일 의존. **변경 Inform 시스템이 이 업무 직격**.

**시나리오**:
1. 로그인 → `/home` — **변경 알림 미확인 건수** KPI 노출?
2. `/changes` — 영향 부서 = 전장설계팀 인 변경 리스트 필터링 가능?
3. `/changes/{cid}` 상세 — 전/후 값 비교·urgency·담당자 표시?
4. `/changes/{cid}/ack` — **Ack 처리** (수신 확인) 원클릭?
5. Ack 하면 다시 /changes 에서 해당 항목이 "처리 완료" 상태로 표시?
6. 기구팀이 Ack 현황을 볼 수 있는가? (변경자 입장)

**통과 기준**:
- 영향 부서 필터 동작 (URL: `/changes?team=6` 등)
- Ack 버튼 1 클릭 + 즉시 반영
- 미ack 잔존 건수가 홈 KPI 에서 감소 반영

### PS-05 임택훈 — 제조기술2팀 "도면 + 변경 통보 사고 방지"

**배경**: 설문 §2 이슈 1위+3위+6위 교집합. "30일 link 만료로 구버전 작업", "변경 통보 누락 사고" 모두 본인 팀 실 사례. **도면 버전 관리는 3순위이지만 변경 inform 은 1순위**.

**시나리오**:
1. `/changes` — 전·후 값 대조 UI 있는가? (설문에서 "변경 전/후 part 구분 표시" 명시)
2. 변경 상세에서 **영향받는 부품·도면** 링크가 걸리는가?
3. 카톡방 다수 혼동 → 시스템이 단일 출처 역할 하는가? (알림 중복·피로 점검)
4. `/board/team` 팀 게시판 — 팀장 개인PC 파일 공유 대체 가능?
5. 게시판 첨부 기능 존재 여부?

**통과 기준**:
- 변경 상세에 `before_value`/`after_value` 필드 렌더
- 영향 강도 분류 (high/medium/low) 시각적으로 구분
- 게시판 파일 첨부 UI 존재 (없으면 S2 기록)

### PS-06 윤영조 — 가공팀 "긴급 삽입·우선순위"

**배경**: 설문 §2.8 "긴급/MODIFY 요청 삽입 — 우선순위 기준 없음". 주 15모델에 긴급이 붙으면 일정 충돌.

**시나리오**:
1. `/tickets` — 긴급도 정렬 가능?
2. `/tickets/new` — 긴급도 필드 존재? (urgency)
3. 긴급 티켓 수신 시 알림 — 모바일 UA 에서도 제대로 보이는가?
4. `/progress` — 가공 공정에서 지연 표시가 담당자(윤영조)에게 집중 노출되는가?

**통과 기준**:
- 티켓/변경 모두 urgency 필드 렌더링
- 긴급 티켓을 먼저 보여주는 정렬·강조 존재

### PS-07 김정록 — 품질팀 "이슈 전산 기록"

**배경**: 설문 §6 로드맵 ⑦ "이슈·AS 이력 DB" — 현재 종이 기록·개인 엑셀. 1순위 아니지만 설문 3팀 공통 요구.

**시나리오**:
1. `/issues` — 이슈·AS DB 접근. 현재 테이블 구조·검색 UI.
2. `/issues/new` — 필수 필드(고객사·모델·증상·처리·재발방지) 충족?
3. 재발 방지 검색 — `/search?q=증상키워드` 가능?
4. `/dashboard` — 2라운드 OPS-012 발견 이슈. 수정 확인.

**통과 기준**:
- issue 생성 폼에 최소 5개 필드 (customer/project/category/severity/note)
- 검색 결과에 issue 가 노출
- leader 로 /dashboard 200 응답 (OPS-012 회귀 체크)

### PS-08 이한중 — SW팀 "일정 변경 통보 + 라이브러리"

**배경**: 설문 §4 직원 제안 1위 "일을 위한 일 최소화". SW는 일정 통보가 카톡으로 와 놓치는 패턴.

**시나리오**:
1. `/changes` — 일정·스케줄 종류의 변경이 분류되는가? (change_type)
2. `/home` — SW 팀원에게 "오늘 주목해야 할 변경" 상단 표시?
3. 반복 로직 재사용 → 현재 시스템에는 해당 기능 없음. "없음"을 명시.

**통과 기준**:
- change_type 목록 확인 (기구·전기·SW·BOM·도면 등 카테고리가 설문과 매칭?)
- /home 에 변경 관련 KPI 노출
