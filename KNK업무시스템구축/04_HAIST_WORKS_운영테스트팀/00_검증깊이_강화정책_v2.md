## 1. 검증 4 단계 모델 (모든 페르소나 적용)

### Level-1 도달성 (~지금까지 04 가 한 수준)
- 로그인 가능?
- 페이지가 200 OK?
- title·heading 정상?

### Level-2 입력·저장 (이번부터 추가)
- 필수 필드 비워두고 저장 → **정확한 에러 메시지** 표시?
- 음수·0·매우 큰 값·이상한 날짜 입력 → **검증 실패 메시지** 표시?
- 한글·영문·이모지·특수문자 (`<script>`, `'`, `"`, `&`) 입력 → **인젝션·인코딩** 안전?
- 저장 후 **즉시 KPI/리스트에 반영** 되는가?
- 저장 후 새로고침 → **데이터 잔존**?

### Level-3 업무 완주 시나리오 (이번부터 추가)
- 페르소나가 "오늘 해야 하는 일"을 **시작부터 끝까지** 시뮬
- 예 (영업 안지연):
  1. 새 프로젝트 등록
  2. 일일카드에 "오늘 견적 회신" 작성
  3. 진행률 대시보드에서 "수주" 단계 시작
  4. 변경 알림 수신 → ack
  5. 티켓 1건 발행 (구매팀에 부품 견적 요청)
  6. 퇴근 전 업무현황 확인
- 각 단계가 **다음 단계로 자연스럽게 연결**되는지

### Level-4 부정 시나리오·엣지 (이번부터 추가)
- 동시성: 두 페르소나가 같은 카드/티켓 동시 편집
- 정보 누출: member 가 URL 직접 조작해서 다른 팀 데이터 접근 시도
- 백·새로고침: 저장 도중 새로고침, 뒤로가기 후 다시 진입
- 큰 입력: 본문 5,000자, 첨부 50MB
- 깨진 입력: 잘못된 URL, 존재하지 않는 ID
- 모바일 뷰포트: 375×667 / 414×896 에서 핵심 액션 가능?
- 다국어: vi/en 적용 후 라벨 잘림·번역 누락

## 2. 페르소나별 5 라운드 시나리오 (확장)

### PS-A 영업 (안지연·이현·이해림)
**Level-3 완주**:
1. 신규 고객 등록 (`/customer/new` — 라우트 존재하나?)
2. 신규 프로젝트 등록 (`/project/new` 또는 admin)
3. 일일카드 작성 (제목·분류·공수·메모·다음계획)
4. /progress 진입 → 첫 공정 [시작] 클릭
5. 공정 메모 입력 + 진척률 변경
6. 변경 알림 새로고침 → 신규 ack 처리
7. 티켓 1건 발행
8. 퇴근 카드 마감 (상태 "완료" 변경)

**Level-4 엣지**:
- 진척률 입력에 -10, 110, "abc" 입력
- 일일카드 본문 5,000자 입력
- /project/999999 (없는 ID) 접근 — 404 vs 500?

### PS-B 구매 (허동준·김선미·정성진)
**Level-3 완주**:
1. /tickets 수신함에서 자재 요청 티켓 1건 처리
2. /parts 부품 검색 (필터 조합)
3. /po/new PO 1건 작성
4. /rates 단가 등록·승인
5. 일일카드 작성

**Level-4 엣지**:
- 음수 수량/단가 PO 작성 시도
- 같은 부품 코드 중복 등록
- 미존재 공급사 선택

### PS-C 설계·전장·SW (정민규·김형렬·이한중)
**Level-3 완주**:
1. /changes/new 변경 등록 (target_kind=도면, urgency=긴급)
2. 영향 부서 자동 판별 결과 확인
3. 상세 진입 → ack 토글
4. 게시판 자동 글 생성 확인
5. 일일카드 작성

**Level-4 엣지**:
- 변경 본문 첨부 파일 (이미지·zip)
- 영향 부서 0개 케이스
- 같은 변경번호 중복

### PS-D 제조·가공·검사기 (마준영·임택훈·윤영조·윤광훈)
**Level-3 완주**:
1. 진행률 대시보드 → 가공 공정 [완료] 클릭
2. 이슈 등록 (/issues/new)
3. 자재 출고 (/stock/issue) — 권한 막히면 보고
4. 일일카드 작성

### PS-E 품질 (김정록)
**Level-3 완주**:
1. 이슈 검색 (/issues + 필터)
2. 새 이슈 등록 (고객·증상·처리·재발방지)
3. 동일 증상 과거 이슈 검색
4. /dashboard 차단 안내 배너 확인 (OPS-012)
5. 일일카드 작성

### PS-F CEO (kjr) + 경영진 (윤경호·이한빈)
**Level-3 완주**:
1. /cockpit 3분 내 회사 상태 파악
2. 병목 1건 드릴다운 → 담당자 확인
3. /dashboard 매출 KPI 확인
4. 미처리 결재 1건 확인
5. 의사결정 필요 항목 검색

### PS-G 베트남 (쑤아잉·땀·박지만·탕·이용식)
**Level-3 완주**:
1. vi 언어 전환
2. 일일카드 작성 (vi UI)
3. 게시판 본사 공지 읽기 (번역 적용)
4. /parts 부품 조회 (단위·화폐 표시)

**Level-4 엣지**:
- 베트남어 입력으로 카드 작성 (`Hôm nay làm việc gì?` 등)
- 시차 (KST vs ICT) 화면 표시
- 영업일 계산 (vi 공휴일)
