### OPS-P0-3 [C-004] /stock/issue 안전재고 미달 출고 차단 팝업

**파일**: `01_HAIST_WORKS/app/templates/stock_issue.html`

**현재 (line 86-109)**: 출고 폼 submit 시 안전재고 미달 부품 → 경고 없이 등록.

**수정** — 폼 submit 시 JS 검증 + confirm:
```html
<!-- form 태그에 onsubmit 추가 -->
<form method="POST" action="/stock/issue" onsubmit="return checkSafetyStock()">
  ...
</form>

<script>
function checkSafetyStock() {
  /* 출고 라인별 안전재고 미달 여부 검증 */
  var rows = document.querySelectorAll('tr.issue-line');
  var warnings = [];
  rows.forEach(function(r){
    var qty = parseFloat(r.querySelector('input[name="qty"]').value || '0');
    var cur = parseFloat(r.dataset.currentStock || '0');
    var safe = parseFloat(r.dataset.safetyStock || '0');
    if (qty > 0 && (cur - qty) < safe) {
      warnings.push(r.dataset.partName + ': 출고 후 안전재고 미달 (' + (cur - qty).toFixed(2) + ' < ' + safe + ')');
    }
  });
  if (warnings.length === 0) return true;
  return confirm('⚠ 안전재고 미달 출고 검출:\n\n' + warnings.join('\n') + '\n\n계속 진행하시겠습니까?');
}
</script>
```

**선결 조건**: stock_issue.html 출고 라인 `<tr>` 에 `data-current-stock` / `data-safety-stock` / `data-part-name` 속성 추가 필요. main.py 라우트에서 stock + safety 조인 쿼리 결과를 부품별 dict 로 전달.

**근거**: 임택훈/윤영조 페르소나 일상 출고 시 위험 차단.
**검증 grep**: `grep -n 'checkSafetyStock\|onsubmit=.*checkSafetyStock' app/templates/stock_issue.html` → 2매치 (정의 + 호출).
