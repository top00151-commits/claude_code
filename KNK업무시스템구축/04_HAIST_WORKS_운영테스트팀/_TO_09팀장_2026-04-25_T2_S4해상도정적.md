## 3. 중요 발견 (4건)

### S4-1. break point 표준 미준수 — 11건 비표준
- 표준 (320/768/1024/1366/1920) 대비 819/900/1000/1100/2200 등 산발
- 디자인 토큰 `00_design_tokens.css` 와 정합성 미확인
- **권고**: 05 디자인팀에 토큰 정합 요청 → `--bp-sm/md/lg/xl/xxl` 변수화

### S4-2. clamp() 사용 0건 — 유체 타이포그래피 부재
- 폰트 크기·여백이 break point에 따라 계단식 변경 (스무드 X)
- 1024 ↔ 1023 경계에서 폰트 점프 발생 가능
- **권고**: 핵심 헤더·KPI 카드 폰트에 `clamp(0.875rem, 1.2vw, 1.125rem)` 도입

### S4-3. px 하드코딩 1,470건 vs rem 8건 — 99.46% px 의존
- 사용자 브라우저 기본 폰트 크기 변경 시 UI 비례 깨짐 (접근성 결함)
- 시안 12B 이식 시점 px 단위 그대로 유지된 것으로 추정
- **권고**: `padding/margin/font-size` 우선 rem 전환 — 1차 100건만 시도해도 효과 큼

### S4-4. 폰트 비례 토큰 0건 — `--text-*` 미정의
- 사이드바만 인라인 변수 사용: `--sb-font-size:17px; --sb-head-size:14px` (line 618)
- 전역 텍스트 스케일 토큰 없음
- **권고**: 05 디자인팀에 `--text-xs(12) / --text-sm(14) / --text-md(16) / --text-lg(18) / --text-xl(22)` 토큰 정의 요청

## 4. PASS 영역 (계속 유지)

- 사이드바 `height:calc(100vh - 57px)` + `overflow-y:auto` 정의됨 (line 618) — 사이드바 스크롤 정상
- 모달류 9개 (`max-height:90vh` 등) `overflow-y:auto` 정의됨 — 작은 화면 모달 절단 방지
- `prefers-reduced-motion: reduce` 지원 (line 3588) — 모션 민감 사용자 배려
- 1024px·768px·480px 기본 break point는 적용됨
