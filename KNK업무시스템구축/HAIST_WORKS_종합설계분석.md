## 0. Executive Summary (2분 결정용)

### 핵심 발견 3가지

1. **baby V1이 설문 1순위 ① 진행률 대시보드의 80%를 이미 엑셀로 충족** (2_진행현황 + 부서별 진척률 자동 집계 + D-day)
2. **baby V2는 영업 중심(수금·잔고·매출예측·KNKVN)으로 재설계 진행 중** — 설문 영업팀 요구와 일치
3. **설문 1순위 ② 변경 Inform + 2순위 카톡 대체 티켓 + 3순위 모든 항목**은 **web HAIST_WORKS 영역** (이미 빅터 정리 완료)

### 즉시 의사결정

| 라인 | 책임 | 다음 작업 |
|---|---|---|
| **baby 라인** | 엑셀 V2 (영업 중심) | 수금·잔고·매출예측·KNKVN 이관 뷰 추가 → 영업팀 즉시 사용 |
| **web 라인** | FastAPI HAIST_WORKS | **변경 Inform + 요청 티켓** 우선 (10팀 공통) → 4·5단계 입고/재고는 후순위로 |
| **공통** | shared/knk_standard.py | 관리코드·수주번호·부서코드 표준 두 시스템에서 동일 |

→ **추천 진행**: web에서 ① 변경 Inform 시스템 + ② 요청 티켓을 먼저 만들고, baby V2는 사용자가 별도 세션에서 진행.

## 1. 시스템 분담 원칙 (이미 정립됨)

빅터(나)와 김정락 대표가 baby V1 README에서 이미 명확히 분리해 둔 것을 확장:

```
┌─────────────────────────────────────────────────────────────┐
│   baby (엑셀 + Python sync) — 매출흐름 / 일정 / 진척          │
├─────────────────────────────────────────────────────────────┤
│  • 검사기·자동화·소모품 PMS (관리코드·수주번호·납기·D-day)    │
│  • 부서별 진척 입력 → PMS 자동 집계                          │
│  • 매출총괄 대시보드 (KRW/USD/INV$)                          │
│  • 영업 중심: 수금·잔고·매출예측·KNKVN 이관 (V2 진행 중)     │
│  → 영업·구매·관리·PM 실무자가 엑셀로 일상 업무               │
└─────────────────────────────────────────────────────────────┘
                            ↕  공통 표준
                  관리코드 8자리 · 수주번호 · 부서코드
                            ↕
┌─────────────────────────────────────────────────────────────┐
│   web (FastAPI HAIST_WORKS) — 협업 / 알림 / 검색             │
├─────────────────────────────────────────────────────────────┤
│  • 일일업무 / 게시판 / 권한 / 알림                           │
│  • 변경사항 Inform (1순위 ②)                                 │
│  • 요청 티켓 — 카톡 대체 (2순위 ③)                           │
│  • 도면 버전 관리 (3순위)                                     │
│  • BOM 통합 마스터, 이슈·AS DB, 증빙·근태, 표준 Library      │
│  → 전사 80명+ 사용 (실시간 협업·검색·이력)                   │
└─────────────────────────────────────────────────────────────┘
```

**핵심 원칙**:
- **baby = 데이터 입력·집계의 단일 진실 (관리코드·수주·진척)**
- **web = 데이터 위에서 협업·알림·검색·이력 기록**
- web이 baby의 엑셀을 **read-only로 import** → 진척 / 매출 / 일정 = baby에서, 알림·티켓·검색·이력 = web에서

## 2. baby V1 현황 (이미 달성)

### 2.1 엑셀 PMS 4개 모듈

| 모듈 | 핵심 파일 | 부서 입력 |

| **02 자동화 완제품** | KNK_자동화_완제품_PMS_2026.xlsx (29열) | 7개 부서 (가공·구매·베트남·설계·SW·전장·제조2) |
| **05 소모품** | KNK_소모품_출하대장_2026.xlsx (소모품 10열 + 출하 40열) | — |
| **10 대시보드** | KNK_경영현황_2026.xlsx | 매출총괄 + 프로젝트목록 |

### 2.2 자동화 기능 (sync.py)

- 관리코드 자동 채번 `001T2604` (KNK PMS 표준)
- 수주번호 자동 채번 `KNK-T-2604-001`
- 날짜 정규화 (`2026/4/15`, `260415` → `2026-04-15`)
- 금액 자동 계산 + 통화별 포맷
- **부서 양방향 sync** — PMS 부서 컬럼 ↔ 부서 파일 담당자 (자동)
- 납품완료 자동 아카이브 → `4_완료이력`
- 정렬: 진행중 → 제안단계 → 보류 → 취소 (납품완료 제외)

### 2.3 사용자 편의

- **`.bat` 18개** — 더블클릭으로 sync 실행
- **헤더 메모** — 각 컬럼 클릭 시 설명
- **글틀고정 자동 계산** (freeze=auto)
- **시트 보호** (비밀번호 `knk2026`)

### 2.4 설문 1순위 ① 진행률 대시보드 충족도

✅ **80% 달성** — `2_진행현황` 시트 + `매출총괄` 대시보드 + 부서별 세부 진척률 자동 평균
🟡 **20% 갭** — 매일 아침 자동 발송, 모바일 가시화, 지연 알림 (web 영역)
