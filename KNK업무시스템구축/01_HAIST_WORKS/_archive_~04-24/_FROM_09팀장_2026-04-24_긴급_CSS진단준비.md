## 즉시 답할 것 — 코드 진단 5 항목

### 1. 폰트 시스템 현황
- `static/style.css` 의 모든 `font-size` 정의 줄 수·단위 분포 (px / rem / em / 비율)
- `:root` 변수 (`--font-base` 같은 토큰) 존재 여부
- 사이드바 폰트가 **어디서 상속**되는지 부모 체인 추적
- main 영역 폰트가 **어디서 상속**되는지 부모 체인 추적
- 줌 100→125 시 **메뉴 vs 본문 어느 쪽이 더 커지는지** 코드상 이유

### 2. 사이드바 overflow 진단
- `base.html` 사이드바 컨테이너 셀렉터 (예: `.sidebar`)
- 해당 셀렉터의 `height`·`max-height`·`overflow` 속성
- 사이드바 안의 항목 수 (현재 12 섹션)
- 화면 1080p 에서 사이드바 콘텐츠 높이 계산 가능 여부

### 3. 미디어쿼리 진단
- `@media` 정의 줄 수
- 브레이크포인트 (768·1024·1280?)
- 폰트 가변 (`clamp()`·`vw`) 사용 여부

### 4. 04 검증팀 OPS-014 이해림 로그인 실패 — 데이터 진단만
