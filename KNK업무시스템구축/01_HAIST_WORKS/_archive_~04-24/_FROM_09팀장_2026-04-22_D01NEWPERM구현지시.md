**설계 초안**: `_DESIGN_물류권한관리UI_초안.md` (commit ef8f849)

### 구현 범위

**A. /api/admin/user-logistics 엔드포인트 권한 확대**

현재: `require(roles=["admin", "ceo"])` → admin·ceo 전용  
변경: 다음 중 하나 선택 후 구현

**안 I**: 구매팀장(role=leader, team code="10")도 자기 팀원에 한해 호출 가능하도록 가드 변경  
```python
# 현재
me = require(request, roles=["admin", "ceo"])

# 변경 (안 I)
me = get_user(request)
if not me:
    return JSONResponse({"ok": False, "error": "로그인 필요"}, 401)
# admin·ceo: 전체 가능
# leader(구매팀): 자기 팀원만 가능
is_admin = me["role"] in ("admin", "ceo")
is_purch_leader = (me["role"] == "leader" and me["team_code"] == "10")
if not (is_admin or is_purch_leader):
    return JSONResponse({"ok": False, "error": "권한 없음"}, 403)
```

**B. 물류 권한 관리 UI 페이지 신설** (`설계 초안 참조`)

- Route: `/admin/logistics-users` (또는 기존 `/admin` 보안 탭 내 섹션 추가)
- 기능: 팀원 목록 + can_use_logistics 토글 스위치
- 접근 권한: admin·ceo·구매팀장(leader, team=10)
- UI 단순 원칙: 팀원 이름·직급·현재 권한 여부·토글 버튼만. 복잡한 필터 불필요.
