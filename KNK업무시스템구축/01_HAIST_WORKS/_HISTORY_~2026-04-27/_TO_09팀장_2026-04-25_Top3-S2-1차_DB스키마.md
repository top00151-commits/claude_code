## 1. 신규 테이블 4 + 뷰 1 (`app/database.py` SCHEMA)

idempotent 가드 (CREATE TABLE/VIEW IF NOT EXISTS) 전건 적용. 시안 §데이터 모델 인용.

| # | 객체 | 종류 | 핵심 컬럼 | 비고 |
|---|---|---|---|---|
| 1 | `receipts` | TABLE | id / po_id / received_at / received_by / total_qty / qc_inspection_id / status(PENDING/PASS/PARTIAL/FAIL) / note | 입고 GR 헤더 (시안 goods_receipts 별칭) |
| 2 | `qc_inspections` | TABLE | id / po_item_id / receipt_id / inspector_id / inspected_at / pass_qty / fail_qty / fail_reason / status(PENDING/PASS/FAIL/HOLD/PARTIAL) | QC 검수 |
| 3 | `qc_disposition` | TABLE | id / qc_inspection_id / action(RETURN/SPECIAL_ACCEPT/SCRAP) / decided_by / decided_at / note | 부적합 처리 |
| 4 | `issues_out` | TABLE | id / part_id / requester_id / approver_id / requested_at / issued_at / qty / purpose / status(PENDING/APPROVED/ISSUED/REJECTED) | 출고 GI |
| 5 | `stock_balances` | VIEW | part_id / part_no / part_name / on_hand / unit / last_movement_at | parts ⨝ stock_movements 그룹합 |

추가 인덱스 7종: `idx_receipts_po`, `idx_receipts_status`, `idx_qc_poitem`, `idx_qc_status`, `idx_qcdisp_qc`, `idx_issues_part`, `idx_issues_status`.

기존 활용 (변경 0): `parts`, `suppliers`, `purchase_orders`, `po_items`, `stock_movements`.
