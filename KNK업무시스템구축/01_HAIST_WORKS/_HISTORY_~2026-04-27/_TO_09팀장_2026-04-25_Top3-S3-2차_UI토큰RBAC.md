## 1. RBAC 컬럼 분리 (옵션 B 채택 · 시안 §7 원안 정합)

`app/database.py` SCHEMA 주석 보강 + `init_db()` 내 마이그레이션 idempotent 가드 추가.

| 변경 | 라인 | 비고 |
|---|---|---|
| permissions SCHEMA 주석 | database.py:617~626 | `name` → DEPRECATED 표시 / 신규 컬럼 안내 |
| ALTER ADD `resource` | init_db() | `if "resource" not in pcols` 가드 |
| ALTER ADD `action` | init_db() | `if "action" not in pcols` 가드 |
| ALTER ADD `description` | init_db() | `if "description" not in pcols` 가드 |
| UNIQUE INDEX `uq_perm_ras` | init_db() | `(resource, action, scope)` 부분 인덱스 (NULL 제외) |

옵션 B 선택 사유: A(DROP+재생성)는 이미 1차에서 사용 중인 환경 안전 가정 깨짐 → ALTER ADD COLUMN으로 호환성 유지.

```python
if pcols and "resource" not in pcols:
    c.execute("ALTER TABLE permissions ADD COLUMN resource TEXT")
if pcols and "action" not in pcols:
    c.execute("ALTER TABLE permissions ADD COLUMN action TEXT")
if pcols and "description" not in pcols:
    c.execute("ALTER TABLE permissions ADD COLUMN description TEXT")
c.execute("CREATE UNIQUE INDEX IF NOT EXISTS uq_perm_ras "
          "ON permissions(resource, action, scope) "
          "WHERE resource IS NOT NULL AND action IS NOT NULL")
```

## 3. 핸들러 — INSERT/UPDATE + audit 트랜잭션 (audit 누락 0건)

`app/main.py` 4544~4710 신규 라우트 6 본문 이식.

### POST /admin/permissions/grant
```
SELECT id FROM users WHERE name=? OR login_id=? LIMIT 1
SELECT id FROM permissions WHERE resource=? AND action=?
  └ 없으면 INSERT INTO permissions(name, resource, action, scope, description) ...
INSERT INTO delegation_tokens(from_user, to_user, permission_id, expires_at, can_redelegate, status) VALUES(?,?,?,?,?,'ACTIVE')
INSERT INTO delegation_audit(token_id, action='GRANT', actor, details)
→ db_session() 단일 트랜잭션 (commit/rollback 원자성)
```

### POST /admin/permissions/revoke (Cascade)
```
2단계 확인: confirm_text == '회수합니다' 강제
UPDATE delegation_tokens SET status='REVOKED' WHERE id=? AND status='ACTIVE'
INSERT INTO delegation_audit(action='REVOKE', actor, details)
SELECT id FROM delegation_tokens WHERE from_user=<수령자> AND status='ACTIVE'  ← Cascade
  └ 각 하위 토큰 UPDATE + INSERT delegation_audit (parent_token_id 표기)
```

### GET /admin/permissions/audit (필터)
```
SELECT da.id, da.timestamp, da.action, da.details,
       u.name AS actor_name,
       COALESCE(p.resource||'.'||p.action, p.name) AS perm_label
FROM delegation_audit da
LEFT JOIN users u ON u.id = da.actor
LEFT JOIN delegation_tokens dt ON dt.id = da.token_id
LEFT JOIN permissions p ON p.id = dt.permission_id
[WHERE da.action=?]
ORDER BY da.timestamp DESC, da.id DESC LIMIT 200
```
