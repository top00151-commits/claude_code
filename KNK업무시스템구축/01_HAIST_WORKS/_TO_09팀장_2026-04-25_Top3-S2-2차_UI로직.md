## 2. 비즈니스 로직 핸들러 5건 (INSERT/UPDATE)

### 2-1. `POST /stock/receipts` — 입고 등록 (main.py:4646)
```python
INSERT INTO receipts (po_id, received_by, total_qty, status, note)  -- status='PENDING'
stock_movement_create({"part_id":..., "kind":"IN", "quantity": +qty, "po_id":..., "reason":f"GR-{gr_id}"}, user_id)
```
→ receipts INSERT 1건 + stock_movements{kind=IN, qty +} 1건 동반.

### 2-2. `POST /stock/qc/{po_item_id}` — 검수 결과 (main.py:4729)
```python
INSERT INTO qc_inspections (po_item_id, receipt_id, inspector_id, pass_qty, fail_qty, fail_reason, status)
UPDATE receipts SET qc_inspection_id=?, status=? WHERE id=?
status ∈ {PASS, PARTIAL, HOLD, FAIL}; FAIL/PARTIAL → /stock/qc/disposition/{qc_id} redirect
```

### 2-3. `POST /stock/qc/disposition` — 부적합 처리 (main.py:4693, 신규)
```python
INSERT INTO qc_disposition (qc_inspection_id, action, decided_by, note)  -- action ∈ {RETURN, SPECIAL_ACCEPT, SCRAP}
UPDATE qc_inspections SET status='FAIL' WHERE id=?  -- RETURN/SCRAP만 (SPECIAL_ACCEPT는 status 유지)
```

### 2-4. `POST /stock/issues` — 출고 요청 (main.py:4782)
```python
INSERT INTO issues_out (part_id, requester_id, qty, purpose, status)  -- status='PENDING'
```
→ 1차 요청 등록만, 재고 차감은 승인 단계에서.

### 2-5. `POST /stock/issues/{id}/approve` — 출고 승인·실행 (main.py:4809, 신규)
```python
UPDATE issues_out SET status='ISSUED', approver_id=?, issued_at=now WHERE id=?
stock_movement_create({"part_id":..., "kind":"OUT", "quantity": qty, "reason":f"GI-{id}"}, user_id)
-- 이미 ISSUED 상태면 already-{status} 거부
```

## 4. 트랜잭션 무결성

- 입고: `receipts` INSERT (db_session #1) + `stock_movements` INSERT (`stock_movement_create` 자체 db_session #2). FIFO 레이어 동시 생성 + `_recalc_part_stock` 자동 갱신 → balance VIEW 자동 반영.
- 검수: `qc_inspections` INSERT + `receipts` UPDATE 동일 db_session 내. 원자성 보장.
- 출고 승인: `issues_out` UPDATE (db_session #1) + `stock_movements` INSERT (#2) → balance VIEW 자동 차감.
- balance VIEW(`stock_balances`)는 `SUM(quantity) GROUP BY part_id` → movements 누락 0건이면 자동 정합.
- movements 누락 0건 자기검증: 핸들러 5건 중 INSERT 발생 3건 (입고·출고승인·부적합 SPECIAL_ACCEPT 제외) 모두 stock_movement_create 호출 또는 미해당 (요청 단계).
