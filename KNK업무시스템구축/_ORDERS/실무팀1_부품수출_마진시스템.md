## 1. 🎯 발주 배경 / 비즈니스 가치

### KNK 부품 수출 매출 구조
- **부품(PARTS) 형태 매출**: 자동화 / 검사기 카테고리에서 부품 단위로 매출
- **수출 대상**: 대부분 베트남 법인 (KNK VINA) 수출
- **마진 구조**: 본사 자재 매입가 + 8~15% 마진 → 베트남 수출 인보이스
- **본사 이익**: 수출 매출 − 매입 단가

### 현재 시스템 한계
1. 자재구매 시스템 (part_prices) 에 매입 단가 보관 — 격리됨
2. 영업 수주 라인 (so_units PARTS) 에 수출 단가 표시 — 격리됨
3. 두 시스템 연결 없음 → 본사 이익률 한눈에 못 봄
4. PARTS 28컬럼은 인보이스 작성 사양 — 더 추가 곤란

### 본 발주의 가치
- **본사 이익률 가시화** — 부품 라인별 마진 % / 이익금액 자동 표시
- **자재구매 ↔ 영업 수주 자동 연결** — 매입가 변동 추적
- **베트남 수출 분석 대시보드** — 월별 / 부품별 / 마진 분포

## 4. 🏗 시스템 설계 (v13 제안서 기반)

### 4.1 DB 스키마

```sql
CREATE TABLE so_unit_part_link (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  so_unit_id   INTEGER NOT NULL,
  part_id      INTEGER,
  part_code    TEXT,

  -- 시점 스냅샷 (매입가 변동 대비)
  snapshot_cost_unit   DECIMAL(15,2),
  snapshot_cost_total  DECIMAL(15,2),
  snapshot_currency    TEXT DEFAULT 'KRW',
  snapshot_date        DATE,

  -- 마진
  margin_pct    DECIMAL(5,2),
  override_note TEXT,

  created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME DEFAULT CURRENT_TIMESTAMP,

  FOREIGN KEY (so_unit_id) REFERENCES so_units(id) ON DELETE CASCADE,
  FOREIGN KEY (part_id) REFERENCES parts(id) ON DELETE SET NULL,
  UNIQUE (so_unit_id)
);
CREATE INDEX idx_solink_part ON so_unit_part_link(part_id);
CREATE INDEX idx_solink_code ON so_unit_part_link(part_code);
```

### 4.2 자동 연결 로직
1. SO PARTS 라인 작성 시 자재코드 입력 (또는 부품 검색 dropdown)
2. part_prices 최신 매입 단가 조회 → snapshot 저장
3. 마진율 자동 계산: `(수출가 - 매입가) / 매입가 × 100`
4. 사용자 수동 override 가능 (협의가, 환차 등)

### 4.3 UI 사양
