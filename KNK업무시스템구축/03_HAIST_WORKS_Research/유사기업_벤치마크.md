## 4. 공개 정보로 확인된 IT·시스템 운영 현황

### 4.1 중요 한계 (사실)

**대부분 한국 중견 장비 제조업은 IT 시스템 구축 사례를 공개하지 않는다**:

| 회사 | 시스템 공개 여부 |
|---|---|
| 테크윙 | ❌ 미공개 |
| 코세스 | ❌ 미공개 |
| 원익IPS | ❌ 미공개 |
| 주성엔지니어링 | ❌ 미공개 |
| 넥스틴 | ❌ 미공개 |
| 인텍플러스 | ❌ 미공개 |
| 파크시스템스 | ❌ 미공개 (IR 자료에 비언급) |
| HB솔루션 | ❌ 미공개 |
| 하나기술 | ❌ 미공개 |
| 이오테크닉스 | ❌ 미공개 |
| 로보스타 | ❌ 미공개 |

**사유 추정**:
- 시스템 내부 IT 현황은 보안·경쟁 문제로 미공개
- 대기업(삼성·SK) 협력사라 시스템 스택 공개 불가능
- 자체 개발보다 **패키지 ERP (영림원·더존·SAP B1)** 또는 대기업 요구 Teamcenter·Windchill 사용 추정

### 4.2 간접 추정 (규모·업종 기반)

| 규모 | 예상 IT 구조 |
|---|---|
| 50~100명 중소 | 엑셀 + 더존 iCUBE·이카운트 |
| 100~300명 중견 | 영림원 K-System + SolidWorks PDM |
| 300~1000명 | SAP B1·SAP S/4HANA + Teamcenter·Windchill |
| 대기업 협력사 | 모회사 시스템 강제 연동 (Teamcenter 등) |

**KNK 135명** → 이론상 "영림원 + SW PDM" 범주. 단, **자체 구축은 업계 이례적 선도 포지션**.

### 4.3 채용 공고에서 추정한 기술 스택 (역공학)

- **SolidWorks** 또는 **NX UG** 사용 회사 다수 (기구 설계 공고)
- **AutoCAD Electrical** 또는 일반 AutoCAD 사용 (전장 설계)
- **PLC** (Siemens·Mitsubishi·LS) 프로그래밍
- **LabVIEW·C#·Python** (검사·제어)
- **ERP 용어** 공고 드물게 포함 (영림원·SAP 언급)

**관찰**: KNK가 "Python FastAPI + Claude AI" 스택으로 자체 구축 = **동종업계에서 기술적으로 앞선 포지션**.

| 항목 | 공식 정의 |
|---|---|

**SCM 부서 정의** (원익그룹 공식):

> "SCM(Supply Chain Management)은 **원재료 공급자부터 최종 고객**에 이르기까지 **모든 공급망 단계를 효율적으로 최적화**하는 부서이며, **자재 관리·공급사 관계 관리**와 관련된 업무를 담당합니다."

**채용 직무 범위** (공고 리스트):
- 공정개발 · 기구설계 · 부품개발 · 제어설계 · RF/플라즈마 · 시뮬레이션
- **Data Science · SW개발** (= 자체 개발 역량 보유)
- 준법경영 · 인사 · **구매** · 기획/예산 · 회계

| 시스템 | ERP + WMS 병행 | HAIST_WORKS가 둘 다 커버 |

| 채용 | "시스템 사용 경험" 필수 조건 | 사용 가이드 체계화 필요 |

**솔직한 한계 (재검증 필요)**:
- 원익IPS 이외 기업은 **구체 시스템 확인 불가** → 간접 추정만
- 실제 운영 노하우는 **업계 커뮤니티·전문 협회**에서만 공유
- 대표이사·구매팀이 동종업계 네트워크 통해 직접 청취하는 게 가장 정확

### 5.2 공통 비효율 (공식 언급)

|---|---|

| 2 | 데이터 입력 지연 → 생산 리드타임 오류 |
| 3 | 시스템 미도입보다 더 큰 비효율 |
| 4 | 엑셀 수작업 의존 |
| 5 | 부서 간 데이터 공유 지연 |

### 5.3 KNK 설문 12부서 응답과 일치도

| KNK 설문 고통 | 업계 공통 현실 |
|---|---|

### 5.4 중소 제조업 디지털 전환 격차 (학회 논문)

## 8. KNK 포지셔닝 관찰

### 8.1 동종업계 비교 매트릭스 (사실 관찰)

| 항목 | 동종업계 공통 | KNK |

### 8.2 KNK 선도 가능성 영역 (사실 관찰)

**공개 자료에서 KNK와 동일한 조합(자체 FastAPI + Claude + MCP + Make-to-Order 장비)을 구축한 한국 중견 제조업은 확인되지 않음**

→ KNK는 **"한국 중견 장비 제조업 IT 자체 구축 + AI 에이전트 통합의 레퍼런스 케이스"** 포지션 가능 (사실 관찰, 권고 아님)

### 8.3 KNK 공식 철학 ↔ HAIST_WORKS 구현 매칭

| KNK 공식 철학 (홈페이지) | HAIST_WORKS 구현 |
|---|---|
| "자동화와 검사 분리 안 함" | 변경 Inform + 진행률 대시보드 = 자동화·검사 통합 뷰 |
| "생산과 품질 하나의 시스템" | 티켓·이슈 DB = 생산 문제·품질 문제 단일 DB |

| "공정 중심 자동화 솔루션" | 12공정 매트릭스 대시보드 |
| "엔지니어링 하우스 도약" | 설문 기반 커스텀 설계 |

→ **HAIST_WORKS는 KNK 공식 철학의 시스템 구현체**

### 8.4 KNK가 참고할 수 있는 업계 공개 사례 (사실만)

**공개 정보가 부족한 이유**는 극복할 수 있는 기회:

**반면교사 수준의 공개 사례**:
