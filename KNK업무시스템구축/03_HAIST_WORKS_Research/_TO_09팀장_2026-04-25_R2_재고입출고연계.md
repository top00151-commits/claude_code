# R2 - 재고 입고·출고 연계 워크플로 표준
> 발신: 03 Research / 수신: 09 팀장 + 04 운영테스트
> 일자: 2026-04-25 / 분량: 약 135줄

## 1. 핵심 결론 (3줄)
- 한국 제조업 자재팀 일상 흐름은 **발주(PO) → 입고(GR/Goods Receipt) → 검수(QC) → 재고반영 → 출고(GI/Goods Issue) → 수불부(Stock Ledger)** 의 6단계가 표준이며, 검수 실패 시 **반품(Return) 또는 특채(Concession)** 두 분기로 갈린다.
- 시스템 구현 시 핵심 테이블은 **9개**로 수렴: po, po_items, goods_receipts, gr_items, qc_inspections, stock_movements, stock_balances, items(자재마스터), warehouses.
- "수불부"는 한국 자재관리의 법적·관습적 핵심 산출물 — 모든 입출고는 **stock_movements 단일 원장(append-only)** 에서 파생되어야 하며, balance를 별도 저장하면 정합성이 깨진다.

## 2. 표준 흐름 (도식)

```
[자재요청 PR]
   │ 승인
   ▼
①발주 PO ─────► 공급사 송부
   │ 납품
   ▼
②입고 GR (납품서 수령 + 수량 확인)
   │
   ▼
③검수 QC
   ├─ PASS ──► ④재고반영 (stock_movements +)
   │              │
   │              ▼
   │           ⑤출고 GI (생산투입/판매출하)
   │              │
   │              ▼
   │           ⑥수불부 자동집계 (조회용 view)
   │
   └─ FAIL ──► 부적합 처리
                ├─ 반품 (PO에 RETURN 차감)
                ├─ 특채 (조건부 사용 승인)
                └─ 폐기 (loss로 차감 + 사유)
```

| 단계 | 한국어 | 시스템 액션 | 회계 영향 |
|---|---|---|---|
| ① 발주 | Purchase Order | 미입고 채무 예약 | (없음, 약정만) |
| ② 입고 | Goods Receipt | 임시 보관 위치 적재 | 미검수 자산 |
| ③ 검수 | QC Inspection | PASS/FAIL/HOLD 판정 | (PASS 시 매입확정) |
| ④ 재고반영 | Stock In | 정상위치 이동 | 재고자산 + |
| ⑤ 출고 | Goods Issue | 차감 (생산/판매) | 재고자산 -, 비용/매출원가 + |
| ⑥ 수불부 | Stock Ledger | append-only 원장 조회 | 감사증빙 |

## 3. 데이터 모델 권고 (9개 테이블)

| 테이블 | 핵심 필드 | 비고 |
|---|---|---|
| `items` | id, sku, name, unit, safety_stock, lead_time | 자재마스터 |
| `warehouses` | id, name, type (RAW/WIP/FG), location | 창고/위치 |
| `purchase_orders` | id, vendor_id, status, expected_at, total | enum 6종 |
| `po_items` | po_id, item_id, qty_ordered, qty_received, unit_cost | |
| `goods_receipts` | id, po_id, received_at, received_by, dn_no | 거래명세서 번호 |
| `gr_items` | gr_id, po_item_id, qty_received, lot_no, expiry | 로트추적 |
| `qc_inspections` | id, gr_item_id, status (PASS/FAIL/HOLD), reason, inspected_by | 검수기록 |
| `stock_movements` | id, item_id, warehouse_id, qty (+/-), ref_type, ref_id, moved_at | **append-only 핵심** |
| `stock_balances` (VIEW) | item_id, warehouse_id, on_hand, allocated, available | movements 집계 뷰 |

PO 상태 enum:
```sql
CREATE TYPE po_status AS ENUM (
  'DRAFT', 'SENT', 'PARTIALLY_RECEIVED', 'RECEIVED',
  'CLOSED', 'CANCELLED'
);
CREATE TYPE qc_status AS ENUM ('PENDING', 'PASS', 'FAIL', 'HOLD', 'CONCESSION');
```

핵심 설계 원칙:
- `stock_movements`는 **절대 UPDATE/DELETE 금지** (append-only). 잘못 입력 시 reverse movement(반대부호)로 상쇄.
- `stock_balances`는 별도 저장하지 말고 **MATERIALIZED VIEW** 또는 트리거 기반 집계. PostgreSQL이라면 `REFRESH MATERIALIZED VIEW CONCURRENTLY`.
- 모든 movement는 `ref_type` (GR/GI/ADJ/TRANSFER/LOSS) + `ref_id` 로 원천 추적.

## 4. UI 시사점 (05 디자인팀이 받을 것)

1. **3단 화면 구성** — 좌(PO 리스트) · 중(선택 PO 상세) · 우(검수 패널). 자재담당자가 한 화면에서 입고-검수를 끝낼 수 있어야 함.
2. **검수 결과 입력은 큰 버튼** — PASS(녹색 大) / FAIL(빨강) / HOLD(노랑). 모바일/태블릿에서 현장 검수 가능해야 함.
3. **수불부는 "엑셀 같은 무한스크롤 표"** — 한국 자재팀은 익숙한 형태. 필터: 기간/품목/창고/movement type.
4. **안전재고 미달 알림 배지** — 자재마스터 화면에 빨간 점. 대시보드에 "오늘 발주 필요한 품목 N건" 위젯.
5. **부분 입고는 진행률 바** — PO 1건 = "12/30개 입고됨 (40%)" 시각화. 한국 중소제조에서 분할납품이 일반적.
6. **로트번호/유효기일 입력 필드** — 식품·화학·의료 자재라면 필수. KNK 사업영역 확인 후 활성화.

## 6. KNK 적용 권고 (4-5줄)
- 9개 테이블 골격 그대로 채택. 초기 MVP는 로트추적·유효기일은 빼고 시작 → 사업확장 시 컬럼 추가.
- `stock_movements` append-only 원칙은 **절대 타협 금지** — 한국 세무·회계 감사에서 수불부 변조 의심 시 치명적.
- 검수 단계는 KNK 현재 규모에서 "PASS만 자동, FAIL/HOLD는 팀장 결재" 워크플로 권고.
- 모바일 검수 화면은 2차 개발로 미루고, 1차는 PC 기반 + 큰 버튼으로 충분.
- 수불부 PDF/Excel 출력은 회계팀이 매월 요구할 것 — 1차 MVP에 반드시 포함.
