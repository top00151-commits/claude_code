## C. customers.default_language 컬럼 신설 가능성

### 현 상태
```sql
-- database.py:61~66
CREATE TABLE IF NOT EXISTS customers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT UNIQUE NOT NULL,
    tier TEXT DEFAULT '일반',
    note TEXT
);
```

### 신설 제안
```sql
ALTER TABLE customers ADD COLUMN default_language TEXT DEFAULT 'ko';
-- 허용 값: 'ko' / 'en' / 'vi' / (선택) 'zh'
```

### 작동 시나리오
1. P1·P11이 거래처 등록 시 기본 언어 선택 (한국 → ko / 베트남 → vi / 글로벌 → en)
2. 견적서 인쇄 라우터:
   ```python
   default_lang = customer.default_language or 'ko'
   # URL 쿼리 ?lang= 우선 → 거래처 기본 언어 → 사용자 언어 순
   lang = request.query_params.get('lang') or default_lang or user.lang or 'ko'
   ```
3. 자동 적용 + 라디오로 즉석 변경 가능 (옵션 1 결합 가능)

### 마이그레이션 안전성
- `ALTER TABLE ADD COLUMN ... DEFAULT 'ko'` → SQLite 기존 데이터 0 영향
- 기존 거래처 47사 모두 'ko' 자동 부여 → 한국 거래처는 동작 동일
- P11이 베트남 거래처만 'vi'로 수정 → 즉시 효과

**권고**: 옵션 3 + customers.default_language 컬럼 신설 = P11 매일 사용 시 라디오 클릭 0회로 자동 베트남어 견적 출력 가능.
