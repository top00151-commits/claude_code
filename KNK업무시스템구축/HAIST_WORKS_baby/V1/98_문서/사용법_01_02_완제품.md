# 사용법 — 01 검사기 / 02 자동화 완제품

> **대상**: 영업·PM·설계·전장·소프트웨어·구매·가공·품질·제조기술·베트남 담당자
> **목적**: 신규 수주부터 납품완료까지 PMS·부서진척 입력 + 자동 집계
> **기준일**: 2026-04-15 (V4 배포본)

## 2. 입력 순서 (3단계)

### 단계 1 — 영업·PM: 신규 수주 등록 (PMS 파일)

`KNK_*_완제품_PMS_2026.xlsx` 열기 → `1_프로젝트등록` 시트

| 입력 컬럼 | 내용 | 자동 채워지는 항목 |
|---|---|---|
| 고객사 | 삼성전자, 삼성전기, 드림텍 등 | NO·관리코드·수주번호 (자동) |
| 품명 | 장비명 | 진척률·D-day (자동) |
| PO유형 | 신규 / 추가 (드롭다운) | |
| 수주단계 | 제안작성 → 제안제출 → 수주확정 → 납품 → A/S | |
| 수량·통화·단가 | 수량×단가 → 금액 자동 계산 | 금액 (자동) |
| 수주일·납기일 | YYYY-MM-DD | D-day (자동) |
| 진행상태 | 수주예정 / 진행중 / 납품완료 / 취소 / 보류 | |

**관리코드 자동 채번 규칙**: `T0012604` (검사기 T + 일련번호 + YYMM)
**수주번호**: `KNK-T-2604-001`

`수주단계 = 제안제출`인 경우 관리코드/수주번호는 발번되지 않습니다 (수주확정 시점에 발번).

---

### 단계 2 — 부서별 담당자: 진척 입력

각 부서 파일 열기 → `1_부서진척입력` 시트

| 자동 표시 | 입력 항목 |
|---|---|
| 관리코드·수주번호·고객사·품명·납기일 | 세부 항목 1~N별 진척률 (0~100%) |

**입력 비율은 0~1 사이 소수**로 입력 (예: 0.8 = 80%)

---

### 단계 3 — sync 실행 (관리자)

```cmd
cd 01_검사기_완제품
python scripts/sync.py
```

자동 처리:
1. 신규 수주에 관리코드·수주번호 발번
2. 부서진척 → PMS 전체 진척률 자동 집계
3. D-day 갱신
4. 납품완료 항목 → `4_완료이력` 자동 아카이브
5. `2_진행현황` 시트 갱신
6. `3_출하현황` 시트 갱신

## 3. PMS 파일의 시트 7종

| 시트 | 역할 | 누가 보나 |
|---|---|---|
| 1_프로젝트등록 | 신규 수주 입력 | 영업·PM |
| 2_진행현황 | 진척률·D-day 자동 집계 | 전체 |
| 3_출하현황 | 출하 일정·운송 | PM·구매·베트남 |
| 4_완료이력 | 납품 완료 아카이브 | 경영진·영업 |
| 5_부서매핑 | 부서별 담당 자동 집계 | 관리자 |
| 6_관리코드발행대장 | 코드 발행 이력 | 관리자 |
| 7_수주번호생성대장 | 수주번호 발행 이력 | 관리자 |

## 6. 자주 묻는 질문

**Q. 수주가 취소됐어요. 어떻게 하나요?**
A. 진행상태를 `취소`로 변경 → sync 실행. 대시보드에서 자동 제외됩니다.

**Q. 납품 완료된 프로젝트가 PMS에서 안 보여요.**
A. `4_완료이력` 시트로 자동 이동했습니다. 검색은 거기서 가능합니다.

**Q. 진척률이 0%로만 표시돼요.**
A. 부서별 입력 파일에서 진척률을 입력하지 않았거나 sync를 실행하지 않은 상태입니다.

**Q. 관리코드가 발번되지 않았어요.**
A. 수주단계가 `제안제출`이거나 비어있습니다. `수주확정` 이후 sync 실행하면 발번됩니다.
