## 🚀 P0 — 즉시 적용 권장 (이번 작업 사이클)

### 1. D. 메신저 웹훅 (가장 빠른 첫 성공)

**왜 P0**:
- **토큰 발급 불필요** — Webhook URL만 있으면 됨
- 그룹대화방마다 URL 1개 → 부서별 채널 매핑이 깔끔
- 현재 `kakao_webhook_send()` 함수가 이미 라우팅 가능한 구조

**적용 작업** (코드):
1. `app_settings`에 `hiworks_webhook_team_<id>` 키 추가 (부서별 URL 저장)
2. `hiworks_client.send_messenger()`를 `send_webhook()` 분기 추가
3. admin 설정 페이지에 부서별 URL 입력칸 동적 생성

**선행 조건 (사용자)**:
- 하이웍스 메신저에서 부서별 그룹대화방 만들기
- 각 대화방에서 웹훅 등록 → URL 복사 → admin 설정에 붙여넣기

---

### 2. A. 인사관리 API (근태/조직 조회)

**왜 P0**:
- 매일 쓰는 정보 (오늘 출근자/휴가/출장)
- 오피스 토큰 1개로 끝
- HAIST 홈 대시보드에 "오늘 부서별 출근 현황" 위젯 가능

**적용 작업** (코드):
1. `hiworks_client.get_attendance_today()` 실제 호출 활성 (이미 stub 있음)
2. `/api/hiworks/attendance` 라우트 추가 (캐시 5분)
3. 홈 대시보드에 위젯 추가

**선행 조건 (사용자)**:
- 인사관리 API 토큰 발급 (`오피스 관리 > API 관리`)

---

### 3. C. 전자결재 웹훅 ⭐ (게임 체인저)

**왜 P0**:
- **2024.07 신규** — 가장 최근에 추가된 핵심 기능
- 폴링 없이 **실시간 이벤트 수신**
- 결재 승인/반려 → HAIST 변경/티켓 상태 자동 동기화
- 사용자가 두 시스템을 오갈 필요 없어짐

**적용 작업** (코드):
1. `/webhooks/hiworks/approval` 엔드포인트 신규
2. 페이로드 검증 (서명/토큰)
3. 이벤트 처리:
   - 기안 → DB 기록 (`approval_drafts` 테이블 신규?)
   - 승인 → `changes`/`tickets`/`issues`에서 `approval_url` 매칭하는 행 상태 자동 변경
4. idempotency key로 중복 방지
5. 재시도 큐 (이벤트 유실 방지)

**선행 조건 (사용자)**:
- 하이웍스 관리자 페이지 → 전자결재 → 관리자 설정 → 기본 설정 → 웹훅 등록
- HAIST URL 입력 (예: `https://haist.knk.com/webhooks/hiworks/approval`)
- **주의**: 페이로드 구조가 공개 매뉴얼에 없음 → 첫 수신 시 로그로 파악 후 파서 작성

**미해결 리스크**:
- 페이로드 구조 미공개 → 1차 구현 후 실측 필요
- HAIST가 외부 IP 접근 가능해야 함 (사내 서버면 NAT/방화벽 설정)

## ⚠️ 함정 회피 (총망라 문서 §위험·제약·함정 기반)

### 운영 함정
1. **오피스 토큰 만료 처리 부재** → 갱신 로직 명시
2. **웹훅 재시도 누락 시 이벤트 유실** → idempotency key + 재시도 큐
3. **SSO iframe X-Frame-Options 차단** → 가비아 협의 선행
4. **조직 동기화 양방향 충돌** → 한쪽을 master로 명시 (HAIST를 master)
5. **검수 승인 지연** → OAuth 2.0 필요 시점 1주일 전 신청

### 보안 함정
1. **웹훅 수신에 서명 검증 없음** → 위조 이벤트 수신 가능. **반드시 검증**
2. **외부 SaaS(Zapier)에 토큰 저장** → 절대 금지
3. **Rate Limit 미확인 Production 배포** → 실측 후 백오프 구현

### 코드 함정
1. **hiworker PyPI (2019 마지막)** → 사용 금지, `httpx`/`urllib` 직접
2. **엔드포인트 추정으로 구현** → Postman 컬렉션으로 먼저 검증 후 구현
