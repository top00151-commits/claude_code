## 0. 한 줄

03 Research가 세션 10용 메신저 구축 참고자료 작성 완료. **작성 후 검증 결과 세션 10은 이미 Flask+Socket.IO+SQLite로 1단계 MVP 가동 중**(memory/project_knk_messenger 일치) — greenfield 아님. 자료는 **확장·통합 단계 참조용**(§A Build vs Fork는 Phase 2+ / §C·D·E·F는 즉시 적용 가능)으로 재해석되어 있음.

**현 MVP 검증 사실** (10_KNK_Messenger/app.py 직접 검증):
- Flask 3.0 + Flask-SocketIO 5.3 + SQLite + FTS5
- 6 테이블 (users·rooms·room_members·messages·items·requests) 완성
- 첨부 25MB·CAD 확장자 포함·unread 카운터·시스템 메시지 자동 기록
- 시드: 김정락 대표·홍길동·이순신 + 4개 실제 아이템(003M2501·WP-LOA·HM-001·M2504, 삼성전자/하나머티리얼)
- 메모리 명시 다음 단계: 사진/파일 → PWA → iOS Capacitor → WORKS 통합

## 1. 핵심 사실 5가지 (즉시 활용 가능)

1. KNK 메인 스택(FastAPI+PostgreSQL)이 **챗 앱 표준 스택과 일치** → 처음부터 구축 시 학습 곡선 낮음
2. **140명은 단일 서버로 충분** — Mattermost 기준 2 vCPU/4GB RAM/45GB로 250~500명 지원 가능 (mattermost docs)
3. **푸시는 FCM 단일 통합** — Android는 FCM, iOS도 FCM이 APNs로 중계 (백엔드 1개 SDK만)
4. **iOS 오프라인 알림은 마지막 1건만 보관** (APNs 제약) → 인앱 unread 카운트 보강 필요
5. **한국어/베트남어 검색** — MySQL ngram 또는 PostgreSQL pg_trgm/pg_bigm + MeCab/Nori 사전 검증 필수
