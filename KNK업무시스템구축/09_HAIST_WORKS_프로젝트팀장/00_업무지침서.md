## 10. 추상 금지·빅뱅 방지 원칙 (2026-04-21 대표 추가, 절대준수)

> **"불합리한 걸 추상적으로 하면 좋을 수도 있겠다"** 고 기능을 마구 추가하지 말 것.
> 이렇게 하면 우리 스스로 **빅뱅에 빠진다** — 과거 ERP 실패들의 공통 패턴.

### 전 세션 공통 규칙
1. 기능 추가·화면 추가·프로세스 변경 제안 시 **케이엔케이 실제 직원** 입장에서 사용성을 판단해야 한다.
2. "있으면 좋을 것 같다" 식 추측 금지. **구체 페르소나 + 구체 업무 시나리오** 없이는 제안도 금지.
3. 기준 페르소나: 신입 / 1년차 / 3년차 / 5년차 / 경력직 / 팀장 / 타부서 / 외근 / 해외 / 베트남 / 대표. (04 운영테스트팀 정책과 일치)
4. 모든 제안서·설계서·gap 분석에 **"이 페르소나가 이 업무를 이렇게 한다"** 섹션이 없으면 감사팀이 반려.
5. 팀장은 페르소나 근거가 없는 제안을 **발행 전 차단**한다.
