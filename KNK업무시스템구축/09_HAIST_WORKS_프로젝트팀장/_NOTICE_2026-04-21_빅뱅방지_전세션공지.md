> "각 팀원들이 실제 작용 시 불합리한 걸 추상적으로 하면 좋을 수도 있겠다 해서 막 기능을 추가하는 건 안 돼.

## 공통 규칙 (전 세션 즉시 적용)

1. **추측 기반 기능 제안 금지**
   - "있으면 좋을 것 같다" / "이런 것도 할 수 있지 않을까" 금지
   - "통합 대시보드" · "AI 자동화" 류 대범위 추상 제안 금지

2. **모든 제안에 실제 KNK 직원 페르소나 근거 필수**
   - 11 페르소나: 신입 / 1년 / 3년 / 5년 / 경력직 / 팀장 / 타부서 / 외근 / 해외 셋업 / 베트남 / 대표
   - 형식: "페르소나 X 가 현재 Y 업무를 이렇게 함 → 불편점 Z → 제안 기능으로 해결"
   - 이 섹션 없는 제안서는 **감사팀 자동 반려**, 팀장 발행 전 차단

3. **빅뱅 위험 신호 — 발견 즉시 팀장 보고**
   - 구현 난이도 XL 누적
   - "모든 기능 재설계" · "대범위 통합" 류 제안
   - 외부 시스템 연계 수 대폭 증가
   - 84명 직원 실측 없이 교과서 ERP 기능 나열
