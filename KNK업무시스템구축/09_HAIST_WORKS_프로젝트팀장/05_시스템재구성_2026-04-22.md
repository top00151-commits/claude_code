## 대표 확정 결정 (2026-04-22)

### 결정 #1 — 수출입 서류 = 별도 모듈 (영역 10)
- CO(원산지증명서) · PL(Packing List) · Invoice 등은 매출(3)/구매(4)에 묻지 않고 **독립 영역**.
- 이유: 제품/상품 구분 + VN 수출·수입 양방향 + 서류 종류 확장성.

### 결정 #2 — 자금일보 = 엑셀 등록 수준
- 기존 작성 엑셀(또는 동형 데이터)을 **등록·조회** 하는 수준.
- 향후 전표·영수증 모듈로 확장 가능. 지금은 확장 안 함.
- 더존 ERP 이관·연동 **없음** (외부 유지).

### 결정 #3 — 카카오워크 사용 안 함 + 카톡 최소화
- 카카오워크: **완전 제거** (D01-06 안 A 확정).
- 카톡: 외부 유지. **티켓 모듈 완성 후 최소화** 로 전환.
- 담당자 일일 정리 옵션: "카톡 대화 내용(사진·동영상·텍스트)를 담당자가 정리해 일일 등록" 기능 **후속 검토**.

### 결정 #5 — 재구성 진행 = 1단계 진단만 먼저
- 1단계: 2일간 각 세션이 자기 영역 정직 진단. 구현 착수 금지.
- 2단계: 대표 승인 후 **확정 범위만** 순차 구현.

### 결정 #4 — 권한 위임 3계층 체계 (확정 2026-04-22)

**원칙**: 상위 권한은 항상 대표이사 또는 대표이사가 이임한 해당자.

**3 계층**:
```
대표이사 (kjr) — 모든 권한
   └─ 대리권한자 (CEO 이임)
        └─ 팀장 (자기 팀 범위 내)
             └─ 팀원 (부여받은 범위)
```

**관리자 모드(창)**:
- CEO · 대리자: 전사 권한 설정 가능
- 팀장: 자기 팀 범위 내 권한 설정
- 팀원: 본인 권한 조회만

**10 영역 적용 범위**:
- 영역 1·2·3·4·5·10 (내부): 권한 체계 적용
- 영역 6·7·8·9 (외부 유지): 링크만, 권한 체계 적용 안 함

**초기 시드 (OPS-011 해소 방식)**:
- DB 초기화: 대표(kjr) + 임원(executive) 전권 → 나머지 전원 0
- 운영 전환: 관리자 UI 에서 대리자·팀장·팀원 순 부여
- **"A 시드 + 관리자 UI 실효화"** 가 공식 방향 (단순 A 가 아님)

**확인 대기 (디폴트로 진단 진행, 대표 확답 시 갱신)**:
- Q1 대리권한자 수: 복수 가능 (CEO 언제든 추가/해제)
- Q2 팀장 대리: 팀장 본인이 지정, 대리자는 감독
- Q3 영역별 세분화: 영역 진단 후 영역별 결정
- Q4 퇴사자 권한 회수: CEO·대리자 즉시 수동

**1단계 진단 추가 주제**:
- 01: 현 `admin_settings.html` · `/api/admin/user-logistics` 등 관리자 UI 커버리지 실측 + **"팀장의 팀원 권한 설정 UI"** 존재 여부
