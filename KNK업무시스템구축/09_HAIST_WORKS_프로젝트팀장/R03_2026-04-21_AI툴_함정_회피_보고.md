## 2. 🎯 HAIST_WORKS가 절대 피해야 할 구체 함정 (체크리스트)

09 팀장이 01 메인에 지시서 발행 시 반드시 포함 권고 항목 (세션 03의 관찰):

### 2.1 기능 과잉 방지 (UX·안정성)

- [ ] 설문 12부서 **1~2순위 기능만 먼저 완성** — 3순위는 Week 4+ 이후
- [ ] 매 기능 배포 전 **로드 테스트** (10,000+ 행 시뮬)
- [ ] 페이지 로딩 **2초 이내** KPI 설정
- [ ] 검색 기능은 **부분 검색·태그·RAG 3단계** 중 하나 이상 필수
- [ ] 초기 학습 시간 **10분 이내** 가이드 필수 (가공팀 요구 원칙)
- [ ] **부서별 맞춤 메뉴** (영림원 패턴) — 직무와 무관한 기능 숨김

### 2.2 벤더 락인 방지

- [ ] 외부 SaaS 의존 최소 — **Hiworks·Claude API 둘뿐** 유지
- [ ] 데이터 소유: **PostgreSQL → CSV·JSON 전수 백업 가능** 설계
- [ ] Claude 외 모델 전환 가능성 확보 — **MCP·Anthropic SDK 추상화**
- [ ] 관리코드·수주번호 표준 **외부 포맷 종속 금지**
- [ ] 이메일·결재 = 하이웍스 외부 링크만 (system_scope_policy 유지)

### 2.3 AI 도입 실패 회피

- [ ] **5~10명 파일럿 → 검증 → 부서 확장** (빅뱅 배포 금지)
- [ ] 모든 AI 출력에 **"왜 이렇게 추천했나"** 설명 버튼 (Nike 교훈)
- [ ] **Human-in-the-loop 강제** — 변경 Inform 자동 발송 금지, 사람 승인 후 발송
- [ ] 부서별 사용 가이드 `.md` + **스크린샷** 필수
- [ ] 사용자 피드백 **매주 수집** (Copilot 폐기 4일 재현 방지)
- [ ] AI 사용 로그 **감사 가능** (00 감사팀 요건)

### 2.4 "화면상 좋지만 현장 실패" 방지 (Odoo 교훈)

- [ ] 모든 설계는 **실제 12부서 사용자 1명 이상 인터뷰** 후 구현
- [ ] 운영테스트팀(04) **페르소나 시뮬** 통과 없이 프로덕션 금지
- [ ] 감사팀(00)의 **실측 검증** 없이 "완료" 보고 금지

### 2.5 변수 대응 누락 (자재관리 관점)

03 Research 보고서 `자재관리_동적변수_대응시스템.md` 교차 참조 — 6대 변수 모두 설계 반영 필요:

- [ ] 시간별 단가 (적용일자·협의일자·통화·확정/가/견적)
