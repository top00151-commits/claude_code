### 언어 옵션 마크업 (홈 페이지 셀렉터)

```
[('ko', '🇰🇷 한국어'),

 ('en', '🇺🇸 영어'),           ← 중복
 ('ko', '🇰🇷 한국어'),         ← 중복
 ('ja', '🇯🇵 일본어'),         ← 의뢰서 외
 ('zh-CN', '🇨🇳 중국어')]      ← 의뢰서 §S3 명시 부재 요구 위반
```

### 판정

| 검증 항목 | 기대 | 실제 | 결과 |
|---|---|---|---|
| 한국어/베트남어/영어 옵션 존재 | 각 1개 | **각 2개** | **🟠 S2** — 옵션 중복 |
| 중국어 옵션 부재 | 부재 | **존재 (`zh-CN`, "🇨🇳 중국어")** | **🔴 FAIL** |
| 일본어 옵션 부재 (암묵) | 부재 | **존재 (`ja`, "🇯🇵 일본어")** | **🟠 S2** — 의뢰서 외 |
| `lang=ko` 적용 후 `업무현황·내 계정·로그아웃` | 노출 | 모두 노출 | 🟢 PASS |
| `lang=vi` 적용 후 `Công việc·Tài khoản·Đăng xuất` | 노출 | 모두 노출 | 🟢 PASS |
| `lang=en` 적용 후 `Overview·My Account·Logout` | 노출 | 모두 노출 | 🟢 PASS |

### 핵심 발견

- **번역 자체는 작동** (ko/vi/en 키워드 정확히 노출).
- 그러나 **언어 셀렉터 마크업에 zh-CN, ja 옵션이 살아있고 ko/vi/en 도 중복 노출**. 의뢰서 §S3 "한·베·영 3종만 표시" 위반.
- 중복 마크업 → 토대가 되는 select 컴포넌트가 base.html 과 home.html 등 두 곳에 렌더되거나, 같은 위젯이 한 페이지에 두 번 렌더되고 있을 가능성.

### 권장 수정 (01 세션)

1. `app/templates/base.html` 또는 사이드바 부분에서 언어 옵션을 `LANGS = {"ko","vi","en"}` 키만 순회하도록 변경.
2. `select` 의 옵션 렌더가 한 페이지에 1회만 일어나도록 위치 정리.
3. zh-CN·ja 옵션 정의를 `i18n.py`/템플릿에서 제거 (또는 빈 LANGS 기준으로 자동 필터).
