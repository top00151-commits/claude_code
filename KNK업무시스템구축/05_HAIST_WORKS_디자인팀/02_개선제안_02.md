## 문제 (What)

### F-20 — 모바일·태블릿 사용 불가
테이블 `min-width: 1400px` (line 99). 좁은 화면에서 가로 스크롤 필수.
- 페르소나 P3(제조팀 마준영·현장)·P8(해외출장)·P10(베트남 법인)·P6(팀장·스마트폰 결재 순간) 모두 모바일 환경.
- 현장 직원이 "지금 우리 프로젝트 어디까지 갔나" 확인 불가.

### F-21 — 셀 내부 과밀
각 셀에 **퍼센트 숫자 + 팀 약자 3자 + 배경색 + 테두리색 + 텍스트색**. 12 공정 × 최대 수십 프로젝트 = 수백 셀이 동일 밀도.
- 색 5종(예정·진행·완료·지연·보류) 배경+테두리 조합 → 한 번에 인식 부담.
- 팀 약자 3자는 축약이라 "ㅁㅈ1" "ㅈㅎㄹ" 식 가독성 낮음.

### F-22 — 첫 사용 모달이 큰 시야 가림
로그인 → /progress 첫 방문 시 모달 전체화면 가림 (line 6~48). localStorage 로 "다시 안 보기" 처리되나 **브라우저 바꾸면 다시 노출**. "?" 재표시 버튼은 있으나 작음.

### O-2 — JetBrains Mono 하드코딩 3곳
`progress_matrix.html:114, 140` 에 `font-family: 'JetBrains Mono', monospace` 인라인. 이미 상표권 정책 §O-2 에서 `--font-mono` 토큰 권고. 본 제안과 함께 처리 권장.

## 제안 (How)

### 제안 A (권장) — 화면 폭 자동 전환: 매트릭스 ↔ 카드

**데스크톱** (≥1024px): 현재 매트릭스 유지. CEO·팀장이 전체를 조망.

**모바일·태블릿** (<1024px): **프로젝트 카드 리스트**. 한 카드 = 한 프로젝트, 12 공정은 가로 스크롤 가능한 **미니 바** 12 개.

사용자가 수동으로 뷰 토글도 가능 (상단 `매트릭스 | 카드` 세그먼트 2버튼).

### 제안 B (필수 동반) — 셀 정보 재구성

현재 셀:
```
[배경색 + 테두리색 + 퍼센트 + 팀약자3자]
```

제안 셀:
```
[배경색 옅게] [퍼센트만]   ← 기본
      + 호버/탭 시 툴팁으로 팀명 · 담당자 · 시작일 · 예정종료일
```

- 색 5종 → **3종** (예정=회색/진행=주황/완료=초록). 지연·보류는 **배지 위첨자** (작은 빨간 점·회색 점) 로 과잉 색 억제.
- 팀 약자 제거 → 카드 클릭 시 상세에서 확인.

### 제안 C — 첫 사용 모달 → 인라인 코치마크

전체화면 모달 대신:
- 페이지 상단에 1줄 노란 배너: "💡 행 클릭 = 상세, 셀 색 = 상태. [자세히 보기]"
- [자세히 보기] 누르면 현재 모달이 사이드 패널로 펼쳐짐 (작게).
- `?` 안내 버튼은 page-head 오른쪽 고정.

### 제안 D — 필터 2단 분리

현재: 사업부 + 고객사검색 + 상태 + 필터 + 초기화 (5 요소 한 줄)

제안: 기본 "고객사 검색" + "사업부" 2 요소. **[고급 필터] 토글**로 상태·기타 필드 아래 펼침.

### 제안 E — JetBrains Mono 토큰화 (정책 O-2 동반 적용)

`style="font-family:'JetBrains Mono',monospace"` 2곳을 아래로 치환:
```html
style="font-family: var(--font-mono)"
```
`--font-mono` 변수는 `_세션01_전달/fonts_font-face_snippet.css` 에 이미 정의 (Pretendard 핸드오프 동봉).

## 위험·제약

- 제안 B 의 "색 5→3" 에서 **지연·보류 배지로 분리**는 CEO 의 "빨간색 = 지연" 즉시 인식 습관과 충돌 가능. 배지가 충분히 눈에 띄게 레드닷 + 테두리 레드 병행.
