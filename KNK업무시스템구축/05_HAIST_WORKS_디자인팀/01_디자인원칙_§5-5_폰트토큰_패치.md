## §5-5. 폰트 토큰 시스템 (신설)

전역 텍스트 스케일 부재(04 S4-4 0건) 및 폰트 비례 역전·해상도 적응 실패의 근본 원인 차단.

### §5-5-1. 토큰 6단계 의무 정의

`00_design_tokens.css` `:root`에 다음 6종이 모두 정의되어야 한다.

| 토큰 | 정의 형식 | 용도 |
|---|---|---|
| `--text-xs` | `clamp(11px, 0.75rem + 0.20vw, 13px)` | 캡션·메타·시간 |
| `--text-sm` | `clamp(13px, 0.875rem + 0.20vw, 15px)` | 사이드바·탭·버튼 보조 |
| `--text-md` | `clamp(15px, 1.000rem + 0.25vw, 17px)` | **본문 기준 (한국어 16px 권장)** |
| `--text-lg` | `clamp(17px, 1.125rem + 0.30vw, 20px)` | 카드 헤더·KPI 라벨 |
| `--text-xl` | `clamp(20px, 1.375rem + 0.40vw, 24px)` | 섹션 헤더 |
| `--text-2xl` | `clamp(24px, 1.625rem + 0.50vw, 30px)` | 페이지 타이틀·KPI 수치 |

- `clamp(min, base, max)` 3인자 형식 강제. 단일값 또는 vw 단독 금지.
- min은 320px 가독성 하한, max는 1920px+ 과대 방지.

### §5-5-2. 메뉴 ≤ 본문 비례 보장 룰

- 사이드바·탭·메뉴 텍스트는 **본문(`--text-md`) 이하** 단계만 사용.
- 메뉴에 `--text-lg/xl/2xl` 적용 금지 (역전 차단).
- 04 발견 인라인 변수 `--sb-font-size:17px` (line 618)는 `--text-sm`로 치환 권고 — 01 적용 발주.

### §5-5-3. 한국어 가독성 본문 16px 권장

- `body { font-size: var(--text-md); }` 1회 선언으로 전역 적용.
- 본문 텍스트(`<p>`, `<li>`, 일반 div)는 별도 font-size 지정 금지 — body 상속.
- 한글 폭이 영문보다 넓어 14px 미만 사용 시 가독성 급격 저하.

### §5-5-4. px → rem 전환 — 빅뱅 금지

- 04 실측 px 1,470건 vs rem 8건 (99.46% px 의존)을 단계 전환.
- 1차 적용 범위 = 헤더(P1) + 사이드바(P2) ≈ 130건. 1차 04 회귀 PASS 후 P3 이후 발주.
- 일괄 치환 발주 금지 (발주서 §6 제약).

### §5-5-5. 자기 검증 grep 패턴

```
# G17-1: 토큰 6종 정의
grep -cE "^\s*--text-(xs|sm|md|lg|xl|2xl)\s*:" 00_design_tokens.css
# 기대: 6

# G17-2: clamp() 함수 채택
grep -cE "clamp\(" 00_design_tokens.css
# 기대: 6 이상

# G17-5: 메뉴 ≤ 본문 역전 차단
grep -nE "(side|nav|menu)[^{]*\{[^}]*font-size\s*:\s*var\(--text-(lg|xl|2xl)\)" style.css
# 기대: 0건
```

검증 대상: `01_HAIST_WORKS/static/00_design_tokens.css` + `style.css` + 시안 03/*.
