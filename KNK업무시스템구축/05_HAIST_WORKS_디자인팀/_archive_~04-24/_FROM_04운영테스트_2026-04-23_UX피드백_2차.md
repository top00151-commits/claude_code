## 1. 3라운드에서 "디자인 관여가 도움될 것" 으로 확정된 영역

### UX-09 진행률 대시보드 — 12공정 라벨 시각화 미확정
**상황**: 설문 §6 ① 에 "**12공정** (수주 → 컨셉 → 설계 → 전장 → SW → 가공 → 구매 → 조립 → 검수 → 출하 → Set-up → KNKVN 이관)" 명시. 3라운드 PS-03 이해림 로그인 실패로 UI 실측 보류. 하지만 `/api/progress/summary` JSON 은 작동 (`{"my_open":0,"delayed":0}`).
**디자인 요청**:
1. **12공정 스테퍼(stepper) 패턴 제안** — 좌→우 흐름, 단계별 상태(예정/진행/완료/지연) 색상 토큰 정의.
2. 스테퍼 가로 길이가 모바일에서 축소될 때 어떻게 표시할지 (접힘·가로 스크롤·1열 세로).
3. 설문에서 "매일 아침 요약 메일" 요구 → 이메일용 **텍스트·경량 HTML 스테퍼** 초안도 함께.

### UX-10 변경 Inform — 영향 강도별 색상·아이콘 표준
**상황**: 01 구현 확인 (`notify_change_impacts` @ `app/main.py:2534`) — high/medium/low 3등급 분기 작동.
게시판 자동 글에 `🔴 직접 영향 / 🟡 일정 영향 / ⚪ 참고` 이모지 쓰임.
**디자인 요청**:
1. 변경 리스트(`/changes`) 카드 UI 에서 3등급 배지 색상·아이콘 토큰 확정. 현재 이모지는 접근성 약함.
2. 전장팀장 김형렬 페르소나가 "오늘 내가 봐야 할 건 몇 개?" 를 **1초 안에** 알도록 우선순위 정렬 패턴.
3. ack 버튼은 리스트에서 직접 처리 가능해야 하는가 vs 상세 페이지 진입 강제? — 대안별 시안.

### UX-11 티켓 상태 어휘·색상 표준표
**상황**: 3라운드 `/tickets` 실측 — `대기, 진행, 긴급` 확인. 완료/보류/취소 관찰 실패.
**디자인 요청**: 1차 피드백 **DSG-04-02** (용어 표준표) 중 "티켓 상태 5종 어휘 + 색상" 구체화. 01 세션이 구현할 수 있는 레벨의 토큰·라벨 제공.

### UX-12 권한 없는 페이지 접근 시 안내 배너
**상황**: OPS-012 (leader /dashboard) 는 `/team` 으로 silent 리다이렉트됨. OPS-001/011 은 `/home` 으로 silent 리다이렉트됨.
**디자인 요청**:
1. 리다이렉트 페이지 상단에 고정되는 **flash 배너 컴포넌트** 디자인.
   - 메시지 2줄 + 닫기 버튼 + 배경 색상 정의
   - 권한 부족·세션 만료·성공 피드백·경고 4종 variant
2. 01 세션은 `request.session['flash']` 값만 세팅하면 되는 단순 구조 권장.

### UX-13 홈 KPI 통합 레이아웃
**상황**: 3라운드 PS-01 정성진 홈에 `변경×34, 티켓×16, 진행률×17, 미확인×1, 지연×6, 요청×26` 키워드 등장 — **통합은 되었으나 정보 과밀 의심**.
**디자인 요청**:
1. 설문 §3 "매일 아침 원하는 정보" 8팀 요구를 KPI 카드 **6~8개** 그리드로 정리 (지금은 어디를 봐야 할지 분산).
2. 역할별(member/leader/executive/ceo) 으로 KPI 우선순위 다르게. 예: member 는 "내가 ack 해야 할 변경" 우선, ceo 는 "전사 지연율" 우선.
3. 카드 1개 클릭 → 해당 상세 페이지로 이동하는 **표준 인터랙션 토큰**.
