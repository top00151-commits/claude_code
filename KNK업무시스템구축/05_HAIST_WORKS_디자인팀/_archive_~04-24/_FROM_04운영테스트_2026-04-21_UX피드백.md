## 2. 구체 재현 증거 (04 세션 2라운드 로그 발췌)

### UX-01 권한 없는 폴백 (자재 모듈)
```
P4 허동준 (구매팀 주임):
  GET /logistics → 200, title=「🏠 업무현황 · KNK」, final=/home
  GET /parts     → 200, title=「🏠 업무현황 · KNK」, final=/home
  GET /suppliers → 200, title=「🏠 업무현황 · KNK」, final=/home
  GET /po        → 200, title=「🏠 업무현황 · KNK」, final=/home
```
**사용자 인식**: "내가 뭘 잘못 눌렀나?" / "시스템이 고장났나?" / "이 회사엔 PO 기능이 없나?" 3단계 자기 의심 발생. 실제론 관리자가 플래그만 켜주면 되는데 안내가 0.

**디자인 제안 요청 (05)**:
- 리다이렉트 결과 페이지 상단에 flash 배너 1줄: "🔒 자재 허브는 구매팀에서 사전 승인 후 사용 가능합니다. 관리자에게 요청하세요."
- 또는 **권한 없는 메뉴는 아예 topbar/sidebar 에 숨기되, 숨김 상태를 사용자가 눈치챌 수 있게 프로필에서 "요청 가능한 메뉴" 리스트 제공**.
- 05 가 `01_디자인원칙.md §실패 6` 의 "역할별 분기 명시화" 사례로 패턴 정의.

### UX-02 세션 살아있는데 `/login` 강제 노출
```
P5 김정록 (품질팀장 leader, 로그인 상태 유지):
  GET /dashboard → 200, title=「로그인 · HAIST WORKS」, final=/login
```
**사용자 인식**: "로그아웃됐나?" 공포. 비밀번호 다시 입력하게 됨 → 세션 세팅 훼손·시간 낭비.

**제안**: 권한 부족은 `/login` 이 아니라 `/home?no_perm=dashboard` 로 이동 + 배너 "🔒 전사 대시보드는 경영진만 볼 수 있습니다". 09 팀장 세션상태보드에서도 동일 논의가 있음.

### UX-03/04 i18n 반쪽 — 핵심 페이지에 한국어 잔존
- P9 이치권 (en 설정 후 `/cockpit`): `Overview·Dashboard` 0회, 「🎯 전사 코크핏」 그대로.
- P10 쑤아잉 (vi 설정 후 `/board/company`): "공지" 11회·"전사" 9회 여전 한국어.

**제안**: 05 디자인팀이 **"다국어 허용 라벨 가이드"** 작성. 어떤 라벨이 i18n 키로 있어야 하고, 어디가 현재 하드코딩되어 있는지 매핑. 01 세션이 실제 치환.

### UX-05 Task Card 카테고리 10종 — 선택 가이드 없음
현재 select: `["설계", "제조", "고객대응", "회의", "출장", "검토", "개발", "구매", "품질", "기타"]`.

**제안**:
- 05가 **각 카테고리의 30자 정의**를 작성하고, select 옆에 `ℹ️` tooltip/popover 배치.
- 추가로 사용자 **팀 sector 기반 기본 카테고리 3~4개를 먼저 노출**하는 2단계 select 패턴 정의.

### UX-06 cockpit 정보 과밀
측정: HTML 99,837 바이트 / KPI 블록 15개 / onclick 핸들러 226개.

**제안**: 05 가 "CEO 3분 룰" 에 맞춘 **축약 cockpit-mini 패턴** 설계 — 상단 5개 KPI + 1 그래프 + "전체 보기" 링크. 현 cockpit 은 드릴다운으로 격하.

### UX-07 모바일·저사양 응답 속도
P8 마준영 모바일 UA → 2,040 ~ 2,080 ms/요청. 개발모드 영향이더라도 p95 <800ms 타깃 필요.

**제안 (05 + 01 합동)**: **로딩 상태 UI 표준** 정의 — skeleton screen / 0.5s 지연 시 로더 / 2s 초과 시 "네트워크 느림" 배너.

### UX-08 로그인 후 도착지 예측 불가
- member → `/daily`
- leader → `/team`
- executive/ceo → `/dashboard`

**제안**: 05 가 "로그인 환영 화면" 설계. 역할별 기본 도착지를 **사용자가 직접 바꿀 수 있는 프로필 설정** 으로 승격. 현재 하드코딩은 신입에게 혼란.

## 3. 협업 요청

### [DSG-04-01] 디자인 원칙 v3 "메시지 UX" 절 신설
현재 원칙서는 시각 레이아웃 중심. **실패 발생 시 사용자에게 무엇을 말할지** (flash, toast, banner, 안내 페이지) 전용 절 필요. 04 세션 발견의 1/3이 "안내 부재" 문제.

### [DSG-04-02] 카테고리·상태·역할 용어 표준표
- 카테고리 10개, 상태 5개(`["진행중", "완료", "지연", "대기", "보류"]`), 역할 5개(admin/ceo/executive/leader/member) — 각 용어의 **확정된 정의서** 필요.
- 04 세션은 사용자 혼선을 감지하지만 "어떤 것이 표준이어야 하는지"는 디자인팀 결정 사항.

### [DSG-04-03] 다국어 라벨 가이드 (vi/en) 초안
- 01 세션이 i18n 키를 심기 전에, 05가 "한국어/영어/베트남어 라벨 3열 대조표" 를 만들어 주면 01이 바로 치환 가능.

### [DSG-04-04] 에러/폴백 페이지 템플릿
- 현재는 /home·/login 재사용. 전용 `403.html / no_perm.html / 500.html` 디자인 필요.

## 5. 기동 조건 (중요)

04 세션은 **비간섭 원칙**으로 다른 팀 작업 중 서버 기동 불가. 05 가 디자인 변경을 01에 전달하여 배포된 후, 09 팀장이 "검증 창(window)" 신호를 보내면 그때 04가 재현. 04·05 가 동시에 서버를 띄우면 01 reload 가 겹쳐 충돌. 서로 시간 겹침 금지.
