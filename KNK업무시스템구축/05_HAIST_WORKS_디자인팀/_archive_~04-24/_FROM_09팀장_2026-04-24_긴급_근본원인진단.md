## 05 가 즉시 답할 것 — 4 결함 분석

### A. 폰트 시스템 진단 (4번 결함의 직접 원인)
1. 현재 `style.css` 의 폰트 사이즈 토큰 (h1/h2/body/small/menu) 정의 여부
2. 단위 — `px` / `rem` / `em` 혼재 여부
3. 사이드바 메뉴 폰트가 **어디 부모 요소에서 사이즈 상속**받는지
4. main 영역 폰트가 **어디 부모 요소에서 사이즈 상속**받는지
5. 메인을 키우면 메뉴가 더 커지는 **CSS 전파 경로** 추적

### B. 사이드바 잘림 진단 (1번 결함)
1. `base.html` 사이드바 컨테이너의 height·max-height·overflow 속성
2. 화면 해상도별 (1080p / 1440p / 2K) 사이드바 실측 높이
3. 사이드바 항목 수 vs 화면 가용 높이 비교
4. scroll 또는 collapse 정책 존재 여부

### C. 반응형 진단 (2번 결함)
1. 미디어쿼리 (`@media`) 정의 위치·기준 px
2. 모바일·태블릿·데스크톱 분기 정책
3. 텍스트 가변 (`clamp()` · `vh/vw`) 사용 여부

### D. 가독성 진단 (3번 결함)
1. main 영역 기본 폰트 사이즈 (px 환산)
2. 권장 한국어 본문 14~16px 기준 비교
3. 시각적 위계 (h1 > h2 > body) 비례 비율
