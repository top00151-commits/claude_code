## 1. 로그인 화면 재설계 (🔴 최우선)

### 1-1. 기존 구조의 문제

`login.html` + style.css 기존 로그인:
- 중앙 단일 카드(380px) · 다크→레드 그라데이션 배경
- 힌트 블록에 **대표이사 ID `kjr`·`kdh` 직접 노출** (L-1, 보안·인지 부담)
- 초기 비밀번호 `knk1234` 평문 노출 (L-2, 04 OPS-008)
- "업무 플랫폼" 건조한 첫 인상 — "써보고 싶은" 브랜드 모멘트 없음

### 1-2. 제안: 브랜드 스플릿 레이아웃

### 1-3. 좌측 브랜드 패널 차별화

- **깊은 그라디언트** `#1A0508 → #8B1E22 → #A5282C` (4단계)
- **SVG 링 애니메이션 3개** (20초 period, 서로 다른 위상)
- **Dot 패턴 배경** (mask-image 로 특정 영역만 보이게)
- **큰 유동 타이포** — `clamp(32px, 4.5vw + 20px, 72px)` 헤드라인
- **그라디언트 텍스트** 일부 단어에 적용 (`linear-gradient` + background-clip)
- **세 가지 약속 배지** — 아이콘 + "3클릭 이내 · 실시간 공유 · 3개 국어"

### 1-4. 우측 폼 패널 차별화

- **미니멀 화이트** — 브랜드 패널과 시각 대비
- **플로팅 라벨** — 입력 포커스 시 라벨이 위로 축소 이동 (공간 절약·현대감)
- **아이콘 내장 필드** (user·lock SVG)
- **3px focus ring** (`box-shadow: 0 0 0 3px rgba(165,40,44,0.1)`)
- **버튼 호버 lift** — `transform: translateY(-2px)` + shadow 확장 + 화살표 아이콘 `translateX(+4px)`
- **언어 셀렉터 우상단** — KO/EN/VI 3 버튼 세그먼트 (04 OPS-003 대응)
- **힌트 블록 제거** — 대표이사 ID·초기비번 모두 제거. 대신 "비밀번호 문의는 관리자에게".

### 1-5. 반응형

- **768px 이하**: 좌 브랜드 (320px 높이) + 우 폼 세로 스택
- **모든 텍스트**: clamp 유동 스케일 — 4K·FHD·태블릿·모바일 모두 최적

## 2. 반응형 유동 타이포 (🔴 신규 도입)

### 2-1. 문제
현재 `--fs-xs: 11px ... --fs-2xl: 28px` 모두 **고정 px**. 결과:
- FHD (1920×1080) 24" 모니터: 적당히 보임
- 4K (3840×2160) 28" 모니터: **작아서 안 보임**. 직원이 "답답하다" 호소.
- 모바일 (375×667): 일부는 커서 잘림, 일부는 작아서 안 보임.

### 2-2. 해결 — `clamp(min, viewport, max)` 전면 도입

`00_design_tokens.css` 에 신설 (이미 시안에 적용됨):

| 토큰 | 공식 | 작은 화면 → 큰 화면 |
|---|---|---|
| `--fs-fluid-xs` | `clamp(10px, 0.55vw + 7px, 13px)` | 10→13px |
| `--fs-fluid-sm` | `clamp(12px, 0.60vw + 9px, 15px)` | 12→15px |
| `--fs-fluid-md` | `clamp(13px, 0.70vw + 10px, 16px)` | 13→16px |
| `--fs-fluid-lg` | `clamp(15px, 0.85vw + 11px, 19px)` | 15→19px |
| `--fs-fluid-xl` | `clamp(18px, 1.25vw + 13px, 26px)` | 18→26px |
| `--fs-fluid-2xl` | `clamp(24px, 2.00vw + 16px, 40px)` | 24→40px |
| `--fs-fluid-hero` | `clamp(32px, 4.50vw + 20px, 72px)` | 32→72px |

- **최소값**: 모바일에서도 읽히는 크기
- **viewport 식**: 창 폭에 비례 증가
- **최대값**: 4K 에서도 과하지 않게

기존 `--fs-xs ... --fs-2xl` 은 **하위 호환으로 유지**하되, 01 세션이 단계적으로 `--fs-fluid-*` 로 마이그레이션.

### 2-3. 즉시 적용 후보 (01 세션 작업)

| 파일 | 대상 |
|---|---|
| `style.css` 전역 | `body`·`.page-title h1`·`.section-head h2`·`.kpi-num`·`.hm2-kpi-val`·`.login-title` |
| `login.html` | 모든 텍스트 (시안 `06_login_브랜드스플릿.html` 그대로 차용 가능) |
| `home.html` | 환영 타이틀·KPI 숫자·섹션 헤더 |
| `progress_matrix.html` | 관리코드·공정 라벨 |

## 3. 시각 언어 차별화 6원칙 (🟠 전역 도입)

시안 `07_visual_language.html` 에 **Before / After** 6개 비교 포함.

### 3-1. KPI 카드 — 평면 → 입체
- **상단 3px 액센트 바** (KNK RED 그라디언트)
- **호버 시 4px 떠오름** + 큰 그림자
- 숫자 더 크게 (`--fs-fluid-2xl`)

### 3-2. 버튼 — 의지를 가진 표면
- 단색 배경 → **그라디언트** + **그림자**
- 호버 시 `translateY(-2px)` + shadow 확장
- 아이콘은 호버 시 `translateX(+4px)` (진행 방향 암시)

### 3-3. 상태 칩 — 색 도트 + 호흡 애니메이션
- 배경색만 바꾸던 기존 → **앞에 색 도트** + 글로우 링
- **진행중 칩만 은은한 pulse** (2초 주기, `box-shadow` 애니메이션). 시선 자연 유도.

### 3-4. 매트릭스 셀 — 테두리 → 배경만 + 호버 확대
- 테두리 5색 과밀 제거 → **배경색 3종만**
- 팀 약자 3자 제거 → **숫자만**, 상세는 호버 툴팁
- 호버 `scale(1.08)` + `z-index: 2` — 해당 셀이 커져서 읽힘

### 3-5. 플로팅 라벨 — 정적 → 동적
- 입력 포커스 시 라벨이 위로 축소 이동 (11px → 10px, 위치 -8px)
- 공간 절약 + 현대 UX

### 3-6. 브랜드 모멘트 (로그인)
- 중앙 단일 카드 → **좌 브랜드 + 우 폼 스플릿**
- SVG 애니메이션 · 그라디언트 텍스트 · 유동 헤드라인
