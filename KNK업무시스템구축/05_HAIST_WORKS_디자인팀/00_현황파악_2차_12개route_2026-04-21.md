## 1. /daily (`daily.html`)
- **실측**: 280 라인 · 9섹션 · 13 CTA · 5색 · depth 2
- **주요 정보**: 일일 업무카드 작성·관리. 주간 통계, 미완료 이월, 다중 상태 카드 그리드, 모달 폼.
- **어려움 후보**:
  - 🔴 F-01. 카드 내부 정보 밀도 높음 — 제목/메타(프로젝트·고객·시간·기한)/노트/계획/댓글 5개 섹션이 스택형 카드에 한 번에.
  - 🟠 F-02. 댓글 펼치기·접기 토글이 카드 내부에 숨겨짐. 상태 변경 버튼 4개와 시각 위계 약함.
  - 🟠 F-03. 상태 변경 퀵버튼 4개가 모두 같은 스타일 — 현재 상태 강조 부족.
- **1차 힌트**: 카드 상단에 댓글 수·대표요청 배지 이동 + 클릭 영역 명시. 현재 상태는 fill, 나머지는 outline.

## 3. /cockpit (`cockpit.html`)
- **실측**: 77 라인 · 4섹션 · 0 CTA · 5색 · depth 2

- **어려움 후보**:
  - 🔴 F-04. 멤버 테이블 6컬럼 (이름·팀·오늘·진행·지연·대표요청 미응답) — 좁은 화면 좌우 스크롤 가능성.
  - 🟠 F-05. 상태 행 스타일 3종 (ck-warn/ck-delay/ck-mute) 문서 명시 없음 — CSS 확인 필요.
  - 🟠 F-06. 병목 섹션(bs)과 상위 테이블 시각 연속성 끊김.
- **1차 힌트**: 테이블 대신 아이콘(⭐⚠️🔴) + 핵심 수치. 병목 섹션 위 구분선 강화.

## 6. /feed (`feed.html`)
- **실측**: 277 라인 · 7섹션 · 2 CTA · 8색 · depth 3
- **주요 정보**: 전사 액션 카드 보드. 팀장 요약 grid, alert-band(지연/리스크/멘션), 3뷰 탭(리스트/칸반/캘린더), 필터+검색, 팀별 dense-list.
- **어려움 후보**:
  - 🔴 F-12. 3뷰 전환 JS 의존 — load 시점·렌더 성능 불명확.
  - 🔴 F-13. Dense-list 기본 collapse 상태가 역할별·긴급도별로 달라져 사용자가 어디부터 봐야 할지 불명확.
  - 🟠 F-14. 필터 칩 9개 + alert-band 중복 칩 3개 — 선택지 과다.
- **1차 힌트**: 뷰 탭 기본 "리스트" 고정, 칸반·캘린더는 명시 전환. Dense-list 기본 전체 닫힘 또는 "내 팀" 강조. alert-band와 필터 칩 통일.
