## 4. 그림자 · 뎁스

힐링 그림자 = **부드러운 확산** (blur 크게, opacity 작게, 색조 있음).

```css
:root{
  --sh-xs:    0 1px 2px rgba(63,92,68,.05);
  --sh-sm:    0 4px 14px rgba(63,92,68,.08);
  --sh-md:    0 14px 36px rgba(63,92,68,.10);
  --sh-lg:    0 28px 60px rgba(63,92,68,.14);
  --sh-green: 0 10px 32px rgba(127,161,131,.28);   /* 세이지 글로우 */
  --sh-glow:  0 12px 36px rgba(165,40,44,.22);     /* CTA 글로우 (KNK 레드) */
}
```

**금지**:
- `0 Xpx Ypx rgba(0,0,0, .2+)` 같은 순수 검정 그림자
- 샤프 그림자 (`blur: 0`)

## 5. 모션

```css
:root{
  --ease:        cubic-bezier(0.25, 0.8, 0.25, 1);   /* 기본 ease-out */
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);  /* 드롭다운 팝 */

  --dur-fast: 200ms;   /* 호버·토글 (기존 150ms 에서 업) */
  --dur-med:  350ms;   /* 카드 lift·섹션 전환 */
  --dur-slow: 500ms;   /* 페이지·도크 슬라이드 */
}
```

**원칙**:
- 200ms 미만 전환 금지 (긴박감 유발)
- hover lift: `translateY(-3px)` 이내 (과한 -5px 는 불안감)
- 반복 애니메이션은 필수 시에만 (빅터 온라인 dot, CTA 없음 X)

**(A) 도크 패널 포지셔닝 · 상단 보존**

| 항목 | ❌ 금지 | ✅ 필수 |
|---|---|---|
| CSS 위치 | `position: fixed; top: 0; bottom: 0; right: 0` (전체 높이) | `.layout` **flex 컨테이너 내부**의 3번째 자식. `position: relative` 또는 `static` |

**이유**: 사용자 이름·로그아웃·워크스페이스 스위처는 **빅터 사용 중에도 계속 보여야** 한다. "빅터 켜면 내가 누군지 사라지는" 경험 금지.

**(C) 슬라이드 동작**

```css
/* 올바른 구현 */
.layout{display:flex;height:calc(100vh - 41px - var(--topbar-h))}
.dock{
  width:var(--dock-w);         /* 360px (1280↓ 320px) */
  flex-shrink:0;
  overflow:hidden;
  transition:width var(--dur-slow) var(--ease), opacity var(--dur-med);
}
body.dock-collapsed .dock{
  width:0;opacity:0;border-left:0;   /* 닫힘 상태 = 폭 0 */
}

/* 모바일 */
@media (max-width:1024px){
  .dock{display:none}                /* 데스크톱 전용 */
}
```

- **열림 기본값**: 페이지 로드 시 열려 있음 (localStorage 로 상태 유지)
- **메인 콘텐츠 밀림**: 도크가 열리면 메인 영역이 자연스럽게 좁아짐 (flex 레이아웃). `margin-right` 트릭 쓰지 않음 — 정적 flex 면 충분
- **모바일 < 1024px**: 도크 표시 안 함 (공간 부족). 추후 풀스크린 오버레이 별도 패턴 개발 필요 시 신규 제안

## 11. 민감 데이터 노출 권한 매트릭스 (절대준수)

> **배경**: 2026-04-24 대표 지적 — "매출을 전직원 공개는 아닌 것 같은데…"
> **원칙**: 시각 디자인이 바뀌어도 **데이터 노출 규칙은 백엔드 권한 체크를 UI 가 그대로 반영**해야 한다.

### 11-1. 노출 등급

| 등급 | 설명 | 예시 |
|---|---|---|
| **P0 · 전사 공개** | 모든 로그인 사용자 | 내 할 일, 내 알림, 팀 참여율(집계), 본인 출결 |
| **P1 · 팀장+** | leader, executive, admin, ceo | 팀 지연 공정 수, 팀원 진행률, 팀 KPI |
| **P2 · 경영진** | executive, admin, ceo | **매출·원가·마진**, 전사 대시보드, 병목 감지, 리포트 |
| **P3 · 도메인 권한자** | `can_use_sales` / `can_use_logistics` 보유자 | 매출·영업 센터, 자재·구매 센터 내부 |
| **P4 · 관리자 전용** | admin, ceo | 사용자 관리, 권한 부여, 시스템 설정, 로그 |

### 11-2. 지표별 노출 등급 (홈·대시보드 KPI 기준)

| 지표 | 등급 | 노출 근거 |
|---|---|---|
| 내 할 일 수 | P0 | 본인 데이터 |
| 내 마감 임박 | P0 | 본인 데이터 |
| 오늘 도착 알림 | P0 | 본인 데이터 |
| 팀 참여율 | P0 | 집계 정보 · 팀워크 유도 |
| 팀 지연 공정 수 | P1 | 팀장 이상 |
| 팀별 진행률 상세 | P1 | 팀장·경영진 (팀원 조회 시 자기 팀만) |
| **이번 달 매출** | **P2** | **경영진 전용 · 홈에서 전 직원 노출 금지** |

### 11-4. 시각 표시 규칙

권한 제한이 있는 KPI·섹션·버튼에는 **작은 자물쇠 뱃지** 표시 (경영진 본인 뷰에서만):

```html
<div class="kpi-lock-badge">🔒 경영진</div>
```

```css
.kpi-lock-badge{
  position:absolute;top:10px;right:10px;
  background:rgba(165,40,44,.08);color:var(--accent);
  font-size:9px;font-weight:800;padding:2px 7px;border-radius:4px;letter-spacing:.3px;
}
```

**이유**: 경영진 본인도 "이 지표가 다른 사람에겐 안 보이는구나" 를 인지해야 기밀 유지 인식 ↑.

### 11-5. 백엔드 이중 방어 (필수)

UI 조건 분기는 **시각 제어일 뿐**, 실제 데이터는 **백엔드 라우트 레벨에서 재차 체크**:

```python
# main.py 공통 컨텍스트 빌더
def home_context(user):
    ctx = {
        "todos": my_todos(user),
        "deadlines": my_deadlines(user),
        "team_rate": team_participation_rate(user.team_id),
    }
    # P2 · 경영진 전용 지표
    if user.role in ('ceo', 'admin', 'executive'):
        ctx["monthly_revenue"] = get_monthly_revenue()
        ctx["yoy_delta"] = get_yoy_delta()
    return ctx
```

UI 가 `{% if %}` 를 생략해도 `monthly_revenue` 가 컨텍스트에 애초에 없어 노출 불가. **UI 실수에 의한 유출 자동 차단**.

### 11-7. "매출" 단어 텍스트 vs 금액 데이터 분리 (2026-04-25 추가)

> **배경**: 04 운영테스트팀 1차 검증 (QA-H7) — *"평직원 본문에 '매출' 단어 3회 등장. 의도 여부 확인 필요."*

#### 원칙

| 노출 형태 | 권한 | 예시 |
|---|---|---|
| **단어 "매출"** (일반 비즈니스 용어) | **P0 전 직원** | 메뉴명("매출·영업"), 카테고리 라벨, 빅터 칩, 페이지 제목 |
| **금액 데이터** (₩·억·천만·숫자) | **P2 경영진만** | "₩4.2억", "전년 대비 +12%", JSON `monthly_revenue` |

#### 검증 패턴 (백엔드·QA용)

```python
# 평직원 화면에서 0건이어야 하는 패턴
revenue_amount_patterns = [
    r'₩\s*[\d,]+',           # ₩4,200,000,000 또는 ₩4.2억
    r'\d+\.?\d*\s*억',        # 4.2억
    r'\d+\.?\d*\s*천만',       # 1.5천만
    r'\b\d{1,3}(?:,\d{3})+\b', # 4,200,000
    r'monthly_revenue',
    r'yoy_delta',
]

# 모든 권한에서 노출 OK인 패턴 (단어로서의 매출)
revenue_word_patterns = [
    r'매출\s*·\s*영업',       # 메뉴명
    r'매출\s*KPI',
    r'매출\s*리포트',
]
```

#### 사이드바 "매출·영업 센터" 메뉴 라벨

- 메뉴 자체는 `user.role in (...) or user.can_use_sales` 조건부 렌더 (`base.html:140·143`)
- 평직원(role=member, can_use_sales=False) 에게는 메뉴 미노출
- → **권한자에게만 메뉴 라벨 노출**, 비권한자에게는 단어 자체 비노출

#### 빅터 자주 묻는 질문 칩 (예: "매출 알려줘")

- 화면 컨텍스트별 칩 (`/home`, `/sales` 등) 에 "매출" 키워드 포함 가능
- 단, 빅터 응답 시 **금액 데이터는 권한 체크 후만 반환** (백엔드에서 `is_executive` 분기)
- 평직원이 "이번 달 매출" 질의 시 → "권한이 없어요. 팀장님께 문의해 주세요" 류 응답
