## 2. 5 그룹 재편안

### 권고 명칭 (한국어 통일, 이모지 최소)

| # | 그룹명 | 포함 라우트 | 뱃지·특기 |
|---|---|---|---|
| **1** | **내 업무** | `/home`·`/daily`·`/calendar`·`/notifications` | 알림 미읽음 뱃지 |
| **2** | **협업** | `/feed`·`/feed?view=now`(구 `/now`)·`/search`·`/cockpit`·`/cockpit#bottlenecks`(구 `/bottlenecks`)·`/dashboard` | 긴급 뱃지 |
| **3** | **프로젝트·변경** | `/progress`·`/changes`·`/issues`·`/tickets` | 미확인 변경·미처리 티켓 뱃지 |
| **4** | **게시판** | `/board/company`·`/board/team` | 대기 승인 뱃지 |
| **5** | **외부·관리** | 하이웍스 전자결제·메일·`/logistics`·`/sales`·`/admin` | 외부 링크 ↗ 표시 |

"팀/프로젝트 동적 트리" 는 **그룹 3 하위의 상시 펼침 영역** 으로 통합 (별도 섹션 아님).

### 색 가이드 (라이선스 정책 §2-1 준수, 중립 팔레트)

| 그룹 | 헤더 배경 | 헤더 텍스트 |
|---|---|---|
| 1. 내 업무 | 기본 (배경 없음) | `--ink-900` |
| 2. 협업 | 기본 | `--ink-900` |
| 3. 프로젝트·변경 | 기본 | `--ink-900` |
| 4. 게시판 | 기본 | `--ink-900` |
| 5. 외부·관리 | 기본 (약간 음영) | `--ink-500` |

**이유**: 그룹별 색상 구분은 시각 과밀 유발. 헤더 타이포·간격 위계로 구분.

## 3. base.html 삽입 HTML 조각 (01 구현 참조)

기존 `.sb-section` 단위 그대로 유지하되, `.sb-head` 를 **5 그룹명**으로 통합:

```jinja
<!-- 그룹 1. 내 업무 -->
<div class="sb-section">
  <div class="sb-head">{{ i.sb_mywork|default('내 업무') }}</div>
  <a href="/home" class="sb-item {% if active=='home' %}sb-active{% endif %}">
    <span class="sb-ico">...</span> 업무현황
  </a>
  <a href="/daily" class="sb-item">...</a>
  <a href="/calendar" class="sb-item">...</a>
  <a href="/notifications" class="sb-item">
    ... 알림
    {% if unread_notif %}<span class="sb-badge">{{ unread_notif }}</span>{% endif %}
  </a>
</div>

<!-- 그룹 2. 협업 -->
<div class="sb-section">
  <div class="sb-head">{{ i.sb_collab|default('협업') }}</div>
  <a href="/feed" class="sb-item">...부서 피드</a>
  <a href="/feed?view=now" class="sb-item">...실시간 활동</a>
  <a href="/search" class="sb-item">...통합 검색</a>
  {% if user.role in ['leader','executive','ceo','admin'] %}
  <a href="/cockpit" class="sb-item">...코크핏</a>
  <a href="/cockpit#bottlenecks" class="sb-item">...병목 감지</a>
  {% endif %}
  {% if user.role in ['ceo','admin','executive'] %}
  <a href="/dashboard" class="sb-item">...전사 대시보드</a>
  {% endif %}
</div>

<!-- 그룹 3. 프로젝트·변경 -->
<div class="sb-section">
  <div class="sb-head">{{ i.sb_project|default('프로젝트·변경') }}</div>
  <a href="/progress" class="sb-item">...진행률 대시보드
    <span id="progress-delayed-badge" class="sb-badge-red"></span>
  </a>
  <a href="/changes" class="sb-item">...변경 알림
    <span id="changes-unread-badge" class="sb-badge-red"></span>
  </a>
  <a href="/issues" class="sb-item">...이슈·AS
    <span id="issues-open-badge" class="sb-badge-red"></span>
  </a>
  <a href="/tickets" class="sb-item">...요청 티켓
    <span id="tickets-pending-badge" class="sb-badge-blue"></span>
  </a>
  <!-- 팀/프로젝트 동적 트리 편입 -->
  <div id="sbTree" class="sb-subtree"></div>
</div>

<!-- 그룹 4. 게시판 -->
<div class="sb-section">
  <div class="sb-head">{{ i.sb_board|default('게시판') }}</div>
  <a href="/board/company" class="sb-item">...전사 게시판</a>
  <a href="/board/team" class="sb-item">...부서 게시판</a>
</div>

<!-- 그룹 5. 외부·관리 -->
<div class="sb-section">
  <div class="sb-head">{{ i.sb_external|default('외부·관리') }}</div>
  <a href="{{ hiworks_approval_url }}" target="_blank" class="sb-item">
    ...전자결제 <span class="ext-arrow">↗</span>
  </a>
  <a href="{{ hiworks_mail_url }}" target="_blank" class="sb-item">
    ...메일 <span class="ext-arrow">↗</span>
  </a>
  {% if user.can_use_logistics %}
  <a href="/logistics" class="sb-item">...자재 시스템</a>
  {% endif %}
  {% if user.can_use_sales %}
  <a href="/sales" class="sb-item">...매출·영업</a>
  {% endif %}
  {% if user.role in ['ceo','admin'] %}
  <a href="/admin" class="sb-item">...관리자</a>
  {% endif %}
```

## 6. 리스크·제약

- 기존 사용자의 **위치 감각** 변화: "코크핏 메뉴 어디 있지?" → 재학습 1~2회.
- "협업" 그룹에 항목 6개 (피드·실시간·검색·코크핏·병목·대시보드) → 원칙 v2 §5-1 "3개 이하 권장" 초과. **근거 있는 예외**: 협업은 본 시스템의 핵심 기능 집합이라 6개 노출이 타당. 그 외 그룹은 모두 4개 이하.
- role 기반 숨김(`{% if ... %}`)은 유지 → 미권한 사용자는 해당 항목 자체가 안 보임.
