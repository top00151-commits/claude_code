## 적용 내역

### A. 3탭 UI ✅ (컨텐츠는 단계적)

**라우트** (`main.py` `home_page`):
```python
async def home_page(req, sel_date="", tab=""):
    if tab not in ("my", "team", "all"):
        role = (u.get("role") or "member").lower()
        if role in ("ceo","admin","executive"): tab = "all"
        elif role == "leader":                  tab = "team"
        else:                                    tab = "my"
```

**템플릿** (`home.html` 본문 상단):
```jinja
<div class="hm-tabs" role="tablist">
  <a href="?tab=my"   class="hm-tab {% if tab=='my' %}is-active{% endif %}">👤 내 업무</a>
  <a href="?tab=team" class="hm-tab {% if tab=='team' %}is-active{% endif %}">👥 우리 팀</a>
  {% if user.role in ['ceo','admin','executive','leader'] %}
  <a href="?tab=all"  class="hm-tab {% if tab=='all' %}is-active{% endif %}">🏢 전사</a>
  {% endif %}
</div>
```

**스타일** (`style.css`):
- pill 배경 `#F1F5F9`, active 시 흰 배경 + KNK Red 글씨
- 8px 그리드, hover 미세 전환

**컨텐츠 분기 (지연)**: 9개 섹션 중 어느 것을 어느 탭으로 할지 정확히 분기하려면 사용자 시범 사용 후 피드백이 안전. **빅뱅 함정 회피 원칙**에 따라 탭 UI만 우선 도입, 컨텐츠는 모두 표시. 5세션이 추가 가이드 주시면 즉시 분기 적용.

### C. status cycle → 세그먼트 4버튼 ✅

**Before**: `<button class="hm-status-btn" onclick="cycleStatus(...)">{{ t.status }}</button>`
**After**:
```jinja
<div class="hm-status-seg" role="group" aria-label="상태">
  {% for s in ['진행중','완료','지연','대기'] %}
  <button class="hm-status-seg-btn{% if s == t.status %} on-{...}{% endif %}"
          onclick="setStatus({{ t.id }}, '{{ s }}', this)">{{ s }}</button>
  {% endfor %}
</div>
```

**JS** (`setStatus` 신규, `cycleStatus` 호환 유지):
- 1-클릭 직접 지정
- 활성 버튼만 색상 (--st-progress / --st-done / --st-delay / --st-wait)
- ARIA `role="group"`

**효과**:
- 클릭 수 최대 3회 → 1회 (디자인팀 시안 기준)
- 색상 대비 명확 (연한 배경 → 진한 색)
