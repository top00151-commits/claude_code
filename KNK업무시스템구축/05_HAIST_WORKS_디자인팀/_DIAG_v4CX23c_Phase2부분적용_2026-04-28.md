## 2. 01 응답서 검증 항목 평가

- ✅ style.css 토큰 추가 (`--paper`, `--ink`, `--accent`, `--font-serif` 등)
- ✅ 3 base 헤더 마스트헤드 추가
- ✅ main.py 컨텍스트 추가

**그러나 다음은 검증 항목에 포함 안 됨**:

- ❌ 페이지별 매거진 톤 적용 (KPI 카드·헤드라인 시리프·alert 빨간 라인 등)

→ **01의 응답서가 거짓은 아니지만 범위가 좁음**. 빅터(05) 발주서가 명확히 "페이지별 잔존 0건" 검증을 포함했어야 함.

## 3. 근본 원인 분석 (빅터 자기검증 7차 학습)

### 3-1. 발주서 grep 항목의 한계
`_TO_01 §11 grep 검증 20종`은 "신규 패턴 추가 ≥1" 위주. **"옛 패턴 잔존 0건"** 검증이 약했음:
- 옛 패턴: `sage-`, `--grad-sage`, 인라인 `#A5282C 외 하드코딩`
- 페이지별 카운트가 명시되지 않음

### 3-2. 01 응답서 평가 역할 누락
빅터(05) 시간단위 정책 §4-A에 "01 git push 시 즉시 점검" 있음. 그러나:
- 01 응답서를 "PASS" 그대로 수용 (액면 수용 금지 약속 위반)
- 응답서의 grep 항목 외 영역(페이지별)을 추가 검증하지 않음

### 3-3. "전체 페이지" 의미 모호
"전체 시스템 재스킨"이라는 표현이 글로벌 CSS만으로도 충족된다고 01이 해석할 여지 있었음. **명시적 페이지 리스트 + 파일별 검증 카운트** 필요.

11. `base.html` (1) ← 헤더 외 본문

처리 방식 (3가지 옵션):

#### 옵션 A — sage 변수 → CX23c 변수 일괄 매핑 (자동)
```css
/* style.css 끝에 호환 alias 추가 */
:root {
  --sage-50:    var(--paper);
  --sage-100:   var(--paper-2);
  --sage-300:   var(--line-soft);
  --sage-700:   var(--ink-2);
  --sage-800:   var(--ink);
  /* ... */
}
```
→ 페이지 변경 0줄로 자동 매거진 톤 변환. **빠름, 그러나 시각 결과는 100% CX23c 아닐 수 있음**.

#### 옵션 B — 페이지별 명시 교체 (수동·정확)
각 파일에서 sage 변수·인라인 색상을 CX23c 변수로 교체. **정확하지만 시간 소요**.

#### 옵션 C — A + B 혼합 (권장)
- 즉시: 옵션 A (alias) 적용 → 시각 즉시 변환
- 후속: 옵션 B (페이지별 정밀) 진행 → 디테일 정합

**빅터 권고: 옵션 C** — 즉시 시각 변경 + 점진적 정합.

## 5. 워크플로우 개선안 (대표 결재 대상)

### 5-1. 신 발주서 표준 (다음부터 적용)

`_TO_01` 발주서 §11 grep 검증을 다음 카테고리로 확장:

```
A. 신규 패턴 추가 (PASS = ≥1)
   - 토큰 / 클래스 / 함수 / 라우트
B. 옛 패턴 잔존 (PASS = 0)
   - 명시적으로 폐기되어야 할 변수·클래스·색상
C. 페이지별 적용 (각 페이지 PASS = 0 잔존)
   - 9개 페이지 × 옛 패턴 카운트 = 9 grep 명시
D. 인라인 색상 정리 (PASS = 0)
   - style="*background:#... * "color:#..." 잔존 0건
E. 페르소나 walkthrough (시각 검증)
```

→ 카테고리 5개 × 항목 약 5개 = **30종 grep 의무**.

### 5-2. 빅터 시간단위 점검 강화

**이번 학습**: 01 응답서 즉시 받았을 때:
1. 응답서의 grep 결과를 **그대로 수용 금지**
2. 빅터가 **응답서 항목 외 페이지별 잔존**을 즉시 추가 grep
3. 차이 발견 시 **즉시 보강 발주서 발행** (대표 점검 전에)

→ 정책 §4-A 확장: "01 응답서 도착 시 빅터 추가 grep (페이지별 잔존·인라인 색상) 의무".

### 5-3. 검증 매트릭스 사전 합의

발주서 발행 시점에 빅터·01·04 가 **검증 매트릭스에 사전 합의**:
