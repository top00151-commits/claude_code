```
+--------------------------------------------------------------+
| HAIST WORKS > 매출 > 주문 라이프사이클           [+ 신규 견적]|
+--------------------------------------------------------------+
| [견적QT] [수주SO] [생산WO] [출하DO/수금RC]                   |
|   ●━━━━━━●━━━━━━●━━━━━━○      ← 좌→우 단방향 (역행 방지)    |
```

## 3. 상태 enum 시각 표시 (R1 §3 — 8종)

```
DRAFT          → tag-wait     #F1F5F9 / #475569
CONFIRMED      → tag-progress #FEF3C7 / #92400E
IN_PRODUCTION  → tag-progress #FFEDD5 / #9A3412 (주황 강조)
READY_TO_SHIP  → tag-progress #DBEAFE / #1E40AF
SHIPPED        → tag-done     #D1FAE5 / #065F46 (옅은)
INVOICED       → tag-done     #BBF7D0 / #166534
PAID           → tag-done     #10B981 / #FFFFFF (녹색 강조)
CANCELLED      → tag-wait     #E5E7EB / #6B7280 (회색, 취소선)
```
- 모두 `00_design_tokens.css`의 `.tag-*` 유틸 클래스 + 위 보조 색상으로 확장.
- MVP는 4단계(DRAFT/CONFIRMED/SHIPPED/PAID)만 활성, 나머지 4종은 enum 정의만 두고 v2.

## 9. 01 코드팀 이식 메모

1. 탭 컴포넌트는 기존 `app/templates/base.html`의 sidebar 구조에 1라우트로 마운트 (`/sales/lifecycle`).
2. 상태 enum은 `app/models/sales.py`에 `SalesOrderStatus` Enum 추가 (R1 §3 SQL 그대로).
3. 부분출하/부분수금은 1:N 관계로 설계 (R1 §6 권고 준수).
4. "세금계산서 발행" 버튼은 S3 권한 미들웨어 `@require_perm('sales','approve')` 적용.
5. 빈 상태 메시지는 `templates/sales/_empty.html` 분리.
