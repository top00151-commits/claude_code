## 0. 핵심 명령

1. 외부 상표·저작물을 **모방·혼동 유발·무단 사용** 하지 않는다.
2. 모든 외부 자산은 **라이선스를 문서화**하고, 라이선스가 요구하는 고지(attribution)·고지 파일(LICENSE)·사용 제한을 준수한다.
3. 의심스러우면 **사용하지 않고** 대표에게 승인 요청을 올린다.
4. **타사 브랜드명을 내부 문서·UI에서 언급하지 않는다** (스타일 영감 표현도 금지). 예: "Stripe 스타일"·"Vercel 헤더" → "모던 대시보드 스타일" 로 서술.

## 1. 폰트 (Typography)

### 1-1. 허용 원칙

| 유형 | 정책 |
|---|---|
| **시스템 내장 폰트 (fallback chain 선언)** | ✅ 허용. 사용자 PC 에 설치된 폰트를 OS가 렌더. 우리가 배포·번들링하지 않으므로 상표/라이선스 문제 없음. |
| **오픈소스 폰트 파일 번들링** | ⚠️ 조건부. SIL OFL·Apache 2.0·MIT 허용, **LICENSE 파일 동봉 필수**. 대표 사전 승인 후 진행. |
| **상용 폰트 파일 번들링** | ❌ 금지. |
| **폰트 CDN (Google Fonts, Adobe Fonts 등)** | ❌ 기본 금지. 외부 트래픽 발생·사용자 추적 가능성. 필요 시 대표 승인. |

### 1-2. 현재 HAIST WORKS 폰트 감사

**05 시안 (`03_시안/00_design_tokens.css`)**:
```
--font-family: "Pretendard", system-ui, -apple-system, "Segoe UI",
               Roboto, "Apple SD Gothic Neo", "Malgun Gothic", sans-serif;
```
- **Pretendard**: SIL OFL 1.1 (상업 사용 OK). 현재 **번들링 안 함** → 사용자 PC 에 설치 없으면 폴백. 결정 필요 (§1-4).
- **Segoe UI**: Microsoft 상표. Windows 기본 내장 → 폴백으로 선언만 하는 것은 OK.
- **Apple SD Gothic Neo**: Apple 상표. macOS 기본 내장 → 폴백 OK.
- **Roboto**: Apache 2.0. Google 트레이드마크이나 폴백 선언 OK.
- **Malgun Gothic**: Microsoft Windows 기본 한국어 폰트. 폴백 OK.
- **system-ui / -apple-system**: OS 기본 UI 폰트 지시자. 상표 대상 아님.

**01 세션 템플릿** (정보 차원 감사 결과, 05 수정 영역 아님):
- `change_detail.html`·`issue_detail.html` 등에 `font-family: 'JetBrains Mono', Consolas, monospace` 선언.
- **JetBrains Mono**: Apache 2.0 (폰트 파일 자체 상업 OK). 그러나 **"JetBrains"는 JetBrains s.r.o. 상표** — 폰트명을 fallback 에 적는 것은 기술적 용법이라 문제 없음. 번들링·CDN 로드 시 LICENSE + NOTICE 포함 의무.
- 실제로는 Consolas(MS) 또는 monospace 폴백으로 렌더될 가능성 높음.

### 1-3. 결정 체계

```
    폰트 필요
        │
        ▼
   시스템 폴백으로 충분?
   ┌─── YES → 사용 (고지 불필요)
   │
   └─── NO  → 오픈 폰트 번들링 필요?
                │
                ▼
          대표 사전 승인 요청 (§5 양식)
                │
                ▼
          승인 시 LICENSE 동봉 + 출처 명기
```

### 1-4. Pretendard 처리 — **(A) 번들링 확정 (대표 승인 2026-04-21)**

**대표 응답**: "그렇게 해" → 옵션 (A) 번들링 확정.

진행 절차는 `_세션01_전달/_TO_01_Pretendard번들링가이드_01.md` 로 01 세션에 핸드오프 완료.
- `static/fonts/PretendardVariable.woff2` 배치

- `static/style.css` 에 @font-face 선언
- 프로젝트 루트 `LICENSE` 파일 생성 (`LICENSE_sample.txt` 제공)

## 2. 아이콘 (Icon)

### 2-1. 현재 사용

- **Lucide** inline SVG path. 01 세션 템플릿 전역에 `<svg viewBox="0 0 24 24" ...>` 형태.
- **라이선스**: ISC License — 상업 사용·수정·재배포 자유. **저작권 고지 포함 의무**.

### 2-2. 정책

1. 아이콘 팩은 **Lucide 단일 소스만** 사용. Font Awesome·Material Icons·Bootstrap Icons 혼용 금지.
2. 새 아이콘은 **Lucide 공식 아이콘만** 사용 (자체 그린 SVG 는 허용하되 브랜드 마크 모방 금지).
3. `LICENSE` 파일에 Lucide 고지 1회 포함:
   ```
   Lucide Icons — ISC License
   Copyright (c) for portions of Lucide are held by Cole Bemis 2013-2022.
   All other copyright (c) for Lucide are held by Lucide Contributors 2022.
   https://github.com/lucide-icons/lucide
   ```
4. 아이콘 **크기·색은 자유로 수정 가능** (ISC 허용).

### 2-3. 금지 사례

- ❌ Apple SF Symbols 사용 (Apple 상표)
- ❌ Material Design 아이콘의 유료 변형 팩 사용
- ❌ 타사 서비스 로고 아이콘 (카카오·구글·네이버 로고를 우리 UI에 직접 삽입)
- ❌ 게시판·자료실에 다운받은 출처 불명 SVG

## 3. 이미지·일러스트

### 3-1. KNK 자산 (문제 없음)
- `static/logo.png`, `static/logo.svg` — 자사 로고.
- `KNK업무시스템구축/KNK로고.png` — 자사 로고.
- 브랜드색 `#A5282C` (KNK RED) — 색상 hex 는 상표권 대상 아님.

### 3-2. 외부 이미지 금지
- Unsplash·Pexels·Freepik 등 **사용하지 않는다**. 업무용 B2B 시스템은 장식 이미지가 본질적으로 불필요.
- 일러스트가 필요하면 **Lucide SVG + 자체 도형**으로 구성.

### 3-3. 이모지 (Unicode)
- 🏠📝📅 등의 이모지 **사용은 가능** — 유니코드 코드포인트이므로 저작권·상표권 대상 아님. OS 렌더러가 렌더.
- 그러나 OS 별 디자인이 달라 **브랜드 일관성 저하**. 원칙서 §6 "아이콘은 Lucide, 이모지는 접근성 보조" 규칙 유지.

## 6. 검증 체크리스트 (매 제안 발행 전)

05 디자인팀이 01 세션에 제안을 보낼 때마다 이 체크리스트 5 항목을 통과해야 한다.

- [ ] **폰트**: 사용하는 폰트 중 번들·CDN 호출이 새로 추가되었는가? → 있으면 §5 양식.
- [ ] **아이콘**: Lucide 외 팩 혼용이 있는가? → 있으면 제거.
- [ ] **이미지**: 외부 서비스(Unsplash 등) 이미지가 포함되는가? → 있으면 제거.
- [ ] **타사명 언급**: 문서·CSS 주석·HTML 텍스트에 "Stripe/Notion/Vercel/Figma/Sketch" 등 노출? → 있으면 중립 용어로 치환.
- [ ] **상용 API 호출**: fetch/XHR 로 외부 상용 API 호출 추가? → 있으면 §5 양식.

통과하지 못한 제안은 **발행 중단**.
