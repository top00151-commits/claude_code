## 문제 (What)

### F-14 — alert-band 와 filter chip 의 "지연·리스크·멘션" 3칩 중복
- line 34–48 alert-band: `🔴지연 / ⚠️리스크 / ❓멘션` 3칩
- line 56–67 filter-bar: 같은 3칩 (+ 전체·진행중·완료·대기·댓글·초기화 = 총 9칩)
- 사용자는 **같은 기능이 두 군데** 노출됨을 학습하기 전까지 혼란.

### F-13 — dense-list 팀 collapse 기본값이 사용자별·데이터별로 다름
`feed.html:87–91`:
```
ceo/executive/admin → has_urgent 만 펼침
그 외            → has_urgent OR is_my_team 펼침
```
같은 화면을 다른 사용자가 보면 "어디가 열려 있나" 가 달라 → 공통 멘탈 모델 형성 실패. 접힌 팀 옆에 뱃지가 없어 **"왜 이 팀은 접혀 있지?"** 판단 근거 부재.

### F-12 — 3 뷰 탭(리스트·칸반·캘린더) 의 등가 배치
line 50–54 의 `.vtab` 3개가 시각적 등가로 노출. 실제로 칸반·캘린더는 lazy render (line 201–202) 지만, **사용자는 "어느 뷰가 기본인지"** 직관 없음. 첫 사용자는 칸반을 클릭했다가 "왜 보던 것과 다르지?" 로 돌아옴.

---

## 제안 (How)

### 제안 A (권장) — alert-band 통합 제거

```diff
  <div class="alert-band">
    <div class="ab-title">...지금 봐야 할 것:</div>
-   <button class="ab-chip ab-delay">🔴 지연 {{all_delay}}</button>
-   <button class="ab-chip ab-risk">⚠️ 리스크 {{all_risk}}</button>
-   <button class="ab-chip ab-ment">❓ 내게 멘션 {{all_mentioned}}</button>
+   {% if all_delay==0 and all_risk==0 and all_mentioned==0 %}
+     <span class="ab-allgreen">✅ 긴급건 없음</span>
+   {% else %}
+     <span class="ab-summary">지연 <b>{{all_delay}}</b> · 리스크 <b>{{all_risk}}</b> · 내게 멘션 <b>{{all_mentioned}}</b></span>
+   {% endif %}
  </div>
```
- **칩은 filter-bar 1곳으로 통일**. alert-band 는 "현재 긴급 수치만 보여주는 요약 라인"으로 축소.
- 클릭 기능은 filter-bar 칩으로 이양.

### 제안 B — filter chip 9개 → 주 4 + 더보기 5

- 상시: `전체 / 지연 / 진행중 / 완료` (4개)
- `▸ 더보기` 토글 클릭 시 펼침: `대기 / 💬댓글있음 / ⚠️리스크 / ❓내게 멘션 / ↺ 초기화` (5개)
- "내게 멘션" 은 **기본 펼침 후보** — 본인 관련도가 높아 자주 사용. P1 신입 활용도 측정 후 정책 재조정.

### 제안 C — dense-list 팀 collapse 규칙 단순화

```diff
- default_open = has_urgent (ceo/exec/admin)
- default_open = has_urgent OR is_my_team (others)
+ default_open = is_my_team   # 통일 — 역할 불문
+ 팀 헤드에 "긴급 N건" 뱃지 표시 (접혀 있어도 보임)
```

- **"내 팀은 항상 펼침"** 1규칙으로 통일. CEO·executive 는 "내 팀" 개념이 비어 있으면 **전체 접힘**. CEO 가 지연·리스크·멘션 칩을 클릭하면 **해당 팀만 자동 펼침** 로직은 현재 유지 (line 166–169).
- 접힌 팀 헤드에 긴급 N건 뱃지 → **"이 팀은 왜 접혀 있는지"** 가 아니라 **"이 팀에 뭐가 있는지"** 를 보여주는 방향 전환.

### 제안 D — 뷰 탭 위계 재설계

- "리스트" 를 기본 · 크고 · 볼드로 강조. 칸반·캘린더는 **"다른 방식으로 보기 ▾"** 토글 버튼 하나로 축약.
- 이유: 리스트가 기본 사용 90% 이상 추정 (dense-list 가 전체 정보 밀집). 칸반·캘린더는 **특수 목적** 뷰.

### 제안 E — 상단 고정 레이아웃 정돈

현재 스크롤 시 상단 고정이 없음. `.page-head` + `.alert-band` + `.view-tabs` + `.filter-bar` 가 **모두 한꺼번에 스크롤 위로 사라짐**. 이 중 `filter-bar` 만 `position: sticky; top: 0;` 로 고정 → 긴 리스트 중간에서도 필터 전환 가능.

## 예상 효과

- 필터 칩 **9→4** (기본 노출). 사용자 인지부담 55% 감소.
- alert-band·filter-bar 칩 **중복 제거** → "어디를 눌러야 하지" 혼란 해소.
- 팀 collapse 규칙 **2→1** 로 단순화 + 시각적 뱃지.
- 칸반·캘린더의 등가 노출 해제 → **리스트 기본 99%** 명확.

---

## 위험·제약

- 기존 alert-band 클릭에 익숙한 사용자는 1~2일 재학습 필요.
- 칸반을 상시 쓰던 사용자가 있으면 **"다른 방식" 토글이 한 번 더 클릭** 필요 — 설문으로 칸반 사용 빈도 확인 권장.
- 팀 collapse 통일은 설계팀장 P6 윤경호 페르소나가 "지연건 자동 펼침" 을 기대할 수 있음. **filter 에서 지연 클릭 시 자동 펼침** 로직은 유지.
