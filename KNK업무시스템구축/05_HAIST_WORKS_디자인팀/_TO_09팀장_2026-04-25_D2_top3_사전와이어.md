## D2-1. 매출 주문 라이프사이클 (4탭 가설)

```
+----------------------------------------------------------+
| HAIST WORKS  >  매출  >  주문 라이프사이클        [+신규] |
+----------------------------------------------------------+
| [견적] [수주] [출하] [수금]   ← 좌→우 진행 (단방향)       |
|  ●------●------●------○                                  |
+----------------------------------------------------------+
| 진행률: 75%  | 이번달 신규 12건 | 마감임박 3건            |
+----------------------------------------------------------+
| 거래처   | 품목     | 금액     | 단계   | D-day | 담당  |
| ────────────────────────────────────────────────────────  |
| (주)A    | XX장비   | 1,200만  | 출하   | D-3   | 김OO  |
| (주)B    | YY부품   |   800만  | 수금   | D-7   | 이OO  |
| ...                                                      |
+----------------------------------------------------------+
| [상세] 클릭 → 우측 슬라이드 패널: 단계별 타임라인 + 첨부   |
+----------------------------------------------------------+
```

- 핵심 UX: 4탭이 **단계 progress bar**로도 시각화 (이중 표현)
- 전환 액션: 견적→수주 / 수주→출하 / 출하→수금 (각 단계에 1버튼)
- 보강 필요(03 결과 후): 단계별 필수 필드 / 승인 라우팅 / KPI 정의

## D2-2. 재고 입출고 연계 (PO→입고→재고)

```
+----------------------------------------------------------+
| 재고  >  입출고 흐름                                      |
+----------------------------------------------------------+
| 흐름도:                                                   |
|                                                           |
|   [PO 발주]──→[입고 검수]──→[재고 적치]──→[출고]          |
|       12         8           ALL          5              |
|       건         건                       건              |
|                                                           |
+----------------------------------------------------------+
| 좌측 패널: PO 리스트 (대기/검수중/완료)                   |
| 중앙 패널: 입고 검수 워크플로 (수량/품질/사진)            |
| 우측 패널: 현재 재고 스냅샷 (안전재고 미달 빨강)         |
+----------------------------------------------------------+
| 알림: PO-2026-0421 입고 지연 D+2 / 안전재고 미달 3종      |
+----------------------------------------------------------+
```

- 핵심 UX: 한 화면에서 **흐름 전체 가시화** (드릴다운 가능)
- 차이 표시: PO 수량 vs 실입고 수량 차이 자동 빨강
- 보강 필요(03 결과 후): 바코드/QR 정책 · LOT 추적 · 회계 연동 시점

## D2-3. 권한 위임 (CEO·팀장·팀원 3계층)

```
+----------------------------------------------------------+
| 설정  >  권한 위임                            [위임 추가] |
+----------------------------------------------------------+
| 현재 위임 매트릭스:                                       |
|                                                           |
|   CEO ─── 결재(전결한도 ∞)                                |
|    │                                                      |
|    └── 팀장 ─── 결재(전결한도 500만) · 휴가 시 위임 ON    |
|         │                                                 |
|         └── 팀원 ─── 작성·상신만                          |
|                                                           |
+----------------------------------------------------------+
| 위임 슬롯:                                                |
| ┌──────────────────────────────────────────────────┐     |
| │ [위임자] CEO  →  [수임자] 09팀장                  │     |
| │ [기간] 2026-04-26 ~ 2026-04-30 (휴가)             │     |
| │ [범위] □전결 □인사 □지출  ☑일반결재               │     |
| │ [한도] 300만원                                    │     |
| │ [상태] 예약                          [취소]       │     |
| └──────────────────────────────────────────────────┘     |
+----------------------------------------------------------+
| 감사로그: 위임 발생 시 자동 기록 (감사팀 가시)            |
+----------------------------------------------------------+
```

- 핵심 UX: 위임은 **기간·범위·한도 3축**으로 항상 명시 (모호 위임 금지)
- 안전장치: 위임 중첩 시 경고 / 자기위임 차단 / 감사로그 강제
- 보강 필요(03 결과 후): 법적 전결규정 매핑 · 전자결재 라인 자동 재계산
