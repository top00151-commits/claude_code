### 1-2. 좌 사이드바 도메인 그룹 (`base.html:139~158`)

```html
<!-- 그룹 4: 도메인 (매출·영업 / 자재·구매 — 권한자만) -->
<div class="sb-section">
  <div class="sb-head">도메인 <span style="..."> 📈📦 </span></div>
  <a href="/sales"     target="_blank" rel="noopener" class="sb-item sb-logi-entry">
    매출·영업 센터 <span class="sb-logi-badge">↗</span>
  </a>
  <a href="/logistics" target="_blank" rel="noopener" class="sb-item sb-logi-entry">
    자재·구매 센터 <span class="sb-logi-badge">↗</span>
  </a>
</div>
```

| 문제 | 설명 |
|---|---|
| **규모 부조화** | 각 "센터" 는 별도 워크스페이스(수십 페이지·팀). 사이드바 한 줄로 두면 "그냥 페이지 하나" 로 보임 |
| **인지 부조화** | `target="_blank"` 로 새 탭 열림 → 외부 링크처럼 보이는데 왜 내부 사이드바에? |
| **탐색 비대칭** | 센터 안에 있을 때 ↔ 통합에 있을 때 **사이드바가 달라야** 하는데, 통합 사이드바에 센터 진입점이 섞이면서 상태 모호 |

### 1-3. 메인창 페이지 헤더

페이지마다 `<div class="page-title"><h1>...</h1></div>` + `<div class="page-sub">...</div>` 자유 형식.

| 문제 | 설명 |
|---|---|
| breadcrumb 부재 | "어디에 있는지" 인지 도구 없음 |
| 우측 액션 영역 불일치 | 페이지마다 버튼 위치·스타일 다름 |
| sticky 처리 일관성 X | 스크롤 시 제목 사라짐 vs 남음 페이지별 제각각 |

## 2. 해법 (3대 재설계)

### 2-A. 상단 헤더 재설계 (72px · 존재감 복원)

#### 구조 (좌 → 우)
```
[ ☰ ]  [ 🅚 HAIST WORKS / KNK 통합 업무 플랫폼 ]  | [ ▦ 통합 ▾ ]   〔 🔍 검색... Ctrl K 〕   [ 🤖 ] [ 🔔· ] [ 한국어 ] [ 👤 김정락 · 대표이사 ] [ ↗ ]
```

#### 시각 스펙
- **높이**: `clamp(64px, 4.5vw + 40px, 80px)` — 1920px 기준 72px
- **배경**: `--grad-brand-deep` (#1A0508 → #3B0B10 → #8B1E22 → #A5282C)
- **로고 마크**: 흰 배경 + KNK 레드 "K" 글자 / 40×40px 라운드 10px (로고 파일 `logo.png` 유지 옵션)
- **브랜드 네임**: `HAIST WORKS` — `--fs-fluid-lg` (15→19px) · 굵기 900
- **브랜드 부제**: `KNK 통합 업무 플랫폼` — `--fs-fluid-xs` (10→13px) · 70% 투명 흰색
- **구분선**: 1px × 32px 세로 · rgba(255,255,255,.2)

#### 워크스페이스 스위처 (신규)
```
[ ▦ 통합 ▾ ]    ← 클릭 시 드롭다운
┌─────────────────────────────┐
│ 🏢 통합                    ✓│ ← 현재
│    HAIST WORKS 메인         │
│ 📈 매출·영업 센터          ↗│
│    Sales Hub · 영업·수주    │
│ 📦 자재·구매 센터          ↗│
│    Logistics Hub · 자재     │
└─────────────────────────────┘
```

- 권한에 따라 노출 (`user.role in ['ceo','admin','executive']` or `user.can_use_sales/logistics`)
- 선택된 센터는 ✓ 표시 · 다른 센터는 ↗ (새 탭 의도 명시)
- 로그인 사용자가 지금 "어느 센터에 있는지" **항시 인지**

#### 중앙: 글로벌 검색바
- 너비: `min(520px, 100%)`
- 배경: 반투명 흰색 (`rgba(255,255,255,.1)`) + 호버 시 강화
- 플레이스홀더: "업무·사람·문서 검색..."
- 단축키 배지: `/` (GitHub / Linear 패턴)
- 검색 엔드포인트: 기존 `/search` 재사용 가능
- **단축키 분리 확정** (2026-04-24):
  - `Ctrl+K` / `⌘K` → 빅터 도크 전용 (#08 계약 고수)
  - `/` → 글로벌 검색 포커스 (input/textarea 포커스 아닐 때만, 한글 IME 조합 중 무시)
  - `Esc` → 공용 해제

#### 우측: 아이콘 그룹
| 요소 | 역할 |
|---|---|
| 🤖 빅터 버튼 | 주황 그라디언트 (눈에 띔) · 클릭 = 빅터 도크 토글 (#08 연동) |
| 🔔 알림 | 빨간 점 (읽지 않은 알림 표시) |
| 언어 셀렉터 | 반투명 배경 · 3개 언어 |
| 사용자 칩 | 아바타 + 이름 + 역할 / pill 모양 / 호버 lift |
| ↗ 로그아웃 | 아이콘 버튼 · 호버 시 빨강 |

### 2-C. 메인창 페이지 헤더 표준화

#### 새 구조
```
┌──────────────────────────────────────────────────────────┐
│ 통합 / 내 업무 / 업무 현황              [내보내기] [+ 신규] │ ← page-header (sticky)
│ 통합 업무 현황                                            │

```

#### 표준 요소
1. **Breadcrumb** (`<nav class="breadcrumb">`) — `워크스페이스 / 그룹 / 페이지`
2. **Page Title** (`<h1 class="page-title">`) — `--fs-fluid-xl`
3. **Page Sub** (`<div class="page-sub">`) — `--fs-fluid-sm` · 520px max-width
4. **Actions** (`<div class="ph-right">`) — 우측 고정 · `btn-ghost` + `btn-primary` 조합

#### 반복 패턴 (body)
- `.kpi-row` — auto-fit 그리드 minmax(180px, 1fr) · 4~5 KPI
- `.section` — 흰 카드 + `.section-head` (제목 + count + `전체 보기 →`)
- `.card` — `.card-title` + `.card-meta` 아이콘 메타라인
- `table` — 헤더 대문자 캡스 + 행 호버 배경 전환

## 4. 권한·라우팅 (백엔드 연계)

### 4-1. 워크스페이스 스위처 노출 조건

```python
# main.py 공통 컨텍스트 빌더
WORKSPACES = [
    {"key": "hub",   "name": "통합",         "desc": "HAIST WORKS 메인",     "icon": "🏢", "href": "/home"},
    {"key": "sales", "name": "매출·영업 센터", "desc": "Sales Hub · 영업·수주", "icon": "📈", "href": "/sales",     "external": True},
    {"key": "logi",  "name": "자재·구매 센터", "desc": "Logistics Hub · 자재", "icon": "📦", "href": "/logistics", "external": True},
]

def workspaces_for(user):
    out = [WORKSPACES[0]]
    if user.role in ('ceo','admin','executive') or user.can_use_sales:
        out.append(WORKSPACES[1])
    if user.role in ('ceo','admin','executive') or user.can_use_logistics:
        out.append(WORKSPACES[2])
    return out
```

### 4-2. 현재 워크스페이스 판별
- 기본: `hub`
- URL `/sales/*` → `sales`
- URL `/logistics/*` → `logi`
- 템플릿 컨텍스트: `{"current_workspace": ..., "workspaces": [...]}`

### 4-3. `target="_blank"` vs 동일 창
- **유지**: 센터 전환은 새 탭 (기존 구현 준수)
- 스위처 옵션에 ↗ 아이콘으로 "새 탭 열림" 시각 명시
