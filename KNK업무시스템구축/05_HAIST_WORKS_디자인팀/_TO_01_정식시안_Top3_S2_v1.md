## 1. 6단계 흐름 시각 표현 (R2 §2)

```
[자재요청 PR]
      │ 승인
      ▼
①발주 PO ─────► 공급사 송부
      │ 납품
      ▼
②입고 GR (납품서 수령 + 수량 확인)
      │
      ▼
③검수 QC ─┬─ PASS ──► ④재고반영 (stock_movements +)
          │              │
          │              ▼
          │           ⑤출고 GI (생산투입/판매출하)
          │              │
          │              ▼
          │           ⑥수불부 자동집계 (조회용 view)
          │
          └─ FAIL ──► 부적합 처리
                       ├─ 반품 (PO에 RETURN 차감)
                       ├─ 특채 (조건부 사용 승인)
                       └─ 폐기 (loss로 차감 + 사유)
```
KPI Strip은 위 흐름의 각 단계 건수를 읽어 6칸으로 표시한다.

## 4. `stock_movements` append-only 원장 + balance view 분리 표현

### ⑥ 수불부 화면 (엑셀형 무한스크롤, R2 §4-3)
```
+--------------------------------------------------------------+
| 수불부  [기간][품목][창고][movement type]   [엑셀] [PDF]      |
+--------------------------------------------------------------+
| 일시       | 품목     | 창고 | 수량  | 유형 | 참조      |   |
| 2026-04-25 |SS400-10T | RAW1 | +12   | GR   | GR-9981  |   |
| 2026-04-25 |SS400-10T | RAW1 | -8    | GI   | WO-101   |   |
| 2026-04-24 |SS400-15T | RAW1 | +20   | GR   | GR-9970  |   |
| ...                                                          |
+--------------------------------------------------------------+
| 잔고 보기 → /stock/balances (별도 라우트, VIEW 조회)         |
+--------------------------------------------------------------+
```

핵심 시각화 분리:
- **`/stock/movements`** — append-only 원장. 모든 행은 **수정·삭제 불가** (행 우측 자물쇠 아이콘).
- **`/stock/balances`** — `stock_balances` MATERIALIZED VIEW 조회. "마지막 갱신: HH:MM" 라벨 + [새로고침] 버튼.
- 두 화면은 **시각적으로 구분**: 원장은 회색 무채색 표(읽기전용 톤), 잔고는 카드형 수치 강조.

## 7. 한국 수불부 법적 양식 (R2 §2 인용)

R2 §2 인용: "수불부 = 한국 자재관리의 법적·관습적 핵심 산출물". 본 시안 §4 수불부 화면은 다음을 보장한다.
- 모든 입출고는 `stock_movements` 단일 원장에서 파생 (R2 §1).
- balance를 별도 저장하지 않음 (R2 §1·§3 — VIEW 사용).
- append-only: UPDATE/DELETE 금지, 잘못 입력 시 **반대부호 reverse movement** 행 추가 (R2 §3 핵심설계원칙).
- 월별 PDF/Excel 출력 버튼 (R2 §6 — 회계팀 매월 요구).
- 컬럼: 일시 / 품목 / 창고 / 수량(±) / 유형(GR/GI/ADJ/TRANSFER/LOSS) / 참조(`ref_type`+`ref_id`).

## 10. 01 코드팀 이식 메모

1. `stock_movements`는 INSERT 전용 — DB 레벨 트리거로 UPDATE/DELETE 거부.
2. `stock_balances`는 PostgreSQL MATERIALIZED VIEW + `REFRESH ... CONCURRENTLY` 5분 주기.
3. 검수 분기는 모달 컴포넌트 1개로 통합, `qc_status` enum 분기 처리.
4. 1차 MVP는 로트번호·유효기일 컬럼 정의만 두고 폼 비활성 (R2 §6).
5. 모바일 검수 화면은 v2로 미룸 (R2 §6).
