### 1-3. 실제 번역 갭 원인 (OPS-013 근본 원인)

UX-03/04에서 "번역이 안 됨"으로 보인 원인은 **라벨이 아니라 DB 데이터**:

| 구분 | 항목 | 현황 |
|------|------|------|
| 병목 룰 문자열 | `{{ b.rule }}`, `{{ b.title }}` | Python `detect_bottlenecks()` 반환값이 한국어 하드코딩 |
| 업무 카드 제목 | `{{ s.title }}`, `{{ t.title }}` | 사용자가 한국어로 입력한 데이터 → 번역 불가 |
| 직급명 | `{{ m.rank }}`, `{{ t.user_rank }}` | DB에 한국어 고정값 ("매니저", "프로" 등) |

**01 메인 조치 권고**: `detect_bottlenecks()` 함수의 룰 문자열을 `i18n.py` 키로 교체 (P1).  
직급명은 rank 코드 표준화 + 번역 테이블 필요 (P2, 별도 제안).

## 4. 다국어 구현 시 주의사항

### 4-1. 베트남어 문자열 길이

베트남어는 한국어 대비 평균 **1.4~1.8배** 길이 증가.

| 적용 위치 | 문제 | 해결책 |
|----------|------|--------|
| 토스트 알림 | 고정폭 컨테이너에서 잘림 | `max-width: 320px; white-space: normal` (2줄 허용) |
| 버튼 문구 | `min-width` 고정 시 베트남어 넘침 | `padding` 기반으로 변경, `min-width` 금지 |
| 배지/태그 | 짧은 상태 단어도 vi는 길어짐 | CSS `text-overflow: ellipsis` 대신 줄바꿈 허용 |

### 4-2. 직급명 vi/en 대응표 (권고안)

| 한국어 DB값 | 영어 | 베트남어 |
|------------|------|---------|
| 대표이사 | CEO | Giám đốc |
| 이사 | Director | Giám đốc bộ phận |
| 매니저 | Manager | Quản lý |
| 프로 | Senior | Chuyên viên |
| 주임 | Staff | Nhân viên |
| 사원 | Associate | Nhân viên mới |

**구현 방법**: i18n.py에 `rank_*` 키 추가 OR rank 코드 표준화 + DB 마이그레이션 (별도 제안 #07 예정).
