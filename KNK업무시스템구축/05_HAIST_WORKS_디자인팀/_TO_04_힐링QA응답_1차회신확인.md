## 3. QA-H5 (ko/vi/en 옵션 중복) — 부분 false alarm 회신

### 04 발견
> ko/vi/en 옵션 마크업 중복 (각 2회 등장)

### 05 분석
실제로 페이지에 **2개의 다른 셀렉터 컴포넌트**가 존재:

| 셀렉터 | ID | 역할 | 옵션 출처 |
|---|---|---|---|
| 상단 헤더 UI 언어 | `#langSel` (`base.html:35`) | 시스템 인터페이스 언어 변경 | `LANGS.items()` (ko/vi/en) ✓ 정상 |
| 카드 모달 자동 번역 | `#trLang` (`base.html:404`) | 카드 본문 자동 번역 대상 언어 | 5종 하드코딩 ✗ 이상 |

### 판정
- **중복 자체는 의도된 구조** (다른 컴포넌트). false alarm.
- 그러나 `#trLang` 의 5종 하드코딩 문제는 별도 (QA-H3·H4 의 본질).
- 패치 ③ 에서 `#trLang` 도 `LANGS.items()` 기반으로 통일 → **자연스럽게 ko/vi/en 만 노출, 중복도 해소된 것처럼 보임**.

### 04 2차 검증 시 권장
```bash
# 헤더 UI 셀렉터 (base.html:35)
grep -c '<select id="langSel"' app/templates/base.html
# 기대: 1

# 모달 번역 셀렉터 (base.html:404)
grep -c '<select id="trLang"' app/templates/base.html
# 기대: 1

# 옵션 텍스트 검색 (어디서 나왔는지 ID 함께 표시 권장)
grep -B1 'option value="ko"' app/templates/base.html
```

→ 두 셀렉터 모두 LANGS 기반이면 ko/vi/en 각 2회는 **정상 (서로 다른 컴포넌트)**.

| 시간대 | 노출 텍스트 |

| 18~22시 | "오늘도 수고하셨어요, {이름}님" |

### 가능성
- 04 검증 시각: 12:40 ~ 14:00 추정 → 기대 텍스트는 "점심은 드셨나요" 또는 "오후도 힘내세요"
- 04 검색 키워드: "오늘도 평안", "좋은 아침" → **시각과 불일치**
- → 인사말은 **정상 노출 중일 가능성 높음**. 검색 키워드 매트릭스 부족.

### 2차 검증 권장 매트릭스

```python
# 04 qa_healing_round.py 에 추가 권장
GREETING_KEYWORDS = {
    "morning":   ["좋은 아침", "☀️"],
    "lunch":     ["점심은 드셨나요", "잠깐 쉬어가요"],
    "afternoon": ["오후도 힘내세요", "🌿"],
    "evening":   ["오늘도 수고하셨어요"],
    "night":     ["늦은 시간까지", "애쓰시네요"]
}
import datetime
hour = datetime.datetime.now().hour
expected_bucket = "morning" if 6<=hour<11 else "lunch" if 11<=hour<14 else "afternoon" if 14<=hour<18 else "evening" if 18<=hour<22 else "night"
expected_keywords = GREETING_KEYWORDS[expected_bucket]
# HTTP 응답 본문에서 위 키워드 중 1개 이상 매칭되면 PASS
```

또한 01에 패치 ④ 로 `data-greeting-bucket="{{ bucket }}"` 속성 추가 지시. 적용 후엔 시각 무관하게 인사말 존재 검증 가능.

## 5. QA-H7 (사이드바 "매출" 단어 노출) — 정책 결정

### 04 발견
> 평직원 본문에 "매출" 단어 3회 등장. 사이드바 메뉴 라벨로서의 매출은 모든 권한에서 노출.

### 05 분석
- `base.html:140, 143`: 매출·영업 센터 메뉴는 `user.role in [...] or user.can_use_sales` 조건부 렌더
- 즉 평직원(role=member, can_use_sales=False) 에게는 **메뉴 자체가 안 보임**
- 04 가 발견한 "매출" 단어 3회는 다른 위치 (예: 변경 알림 카테고리 라벨, 빅터 자주 묻는 질문 칩, 도메인 그룹 헤더의 일반 텍스트 등)

### 정책 결정 (05 디자인팀 명문화)

> **"매출" 단어 자체는 회사의 일반 비즈니스 용어**이므로 평직원에게도 텍스트로 노출 가능. 단, **금액(₩, 억, 천만, 숫자)은 P2 경영진만**.

이 원칙을 힐링 원칙서 §11-7 로 추가 (별도 작업).

### 04 2차 검증 권장
"매출" 단어 노출 자체는 PASS 처리. **금액 패턴**(`₩\d`, `\d+\s*억`, `\d+,\d{3}`, `monthly_revenue` JSON 키)만 평직원 화면에서 0건이면 충분.

```python
# 권장 검증 정규식
revenue_amount_patterns = [r'₩\s*[\d,]+', r'\d+\.?\d*\s*억', r'\d+\.?\d*\s*천만', r'monthly_revenue']
```
