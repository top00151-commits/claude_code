## 2. 추정 원인 (정확 진단은 디자인팀 회신 부탁)

`static/style.css` healing-c2 모듈의 다음 규칙들이 본 페이지 컨테이너 폭/높이 컨텍스트에서 의도와 다르게 거대 박스로 렌더링되었을 가능성:

| 셀렉터 | 의심 부분 |
|---|---|
| `.hero-kpi::before` | `width:140px;height:140px;border-radius:50%;background:radial-gradient(...)` — 다른 페이지(dashboard)에선 정상이지만 본 페이지에서 폭주 |
| `.hello-banner::before` | `width:240px;height:240px;` 같은 거대 가짜요소 |
| `.hk-ico svg` | `width:20px;height:20px` 강제하지만 부모 `.hk-ico` 컨텍스트 누락 시 SVG 가 viewBox 기본 300px 로 폭주 가능 |

→ 디자인팀이 상기 규칙들에 **`max-width`/`max-height` 강제** + **`overflow:hidden`** 추가 제안 부탁.

## 4. 05 디자인팀 후속 요청

### 4-1. healing-c2 모듈 안정성 점검 (전 페이지 영향)
- `.hero-kpi::before` 가 본 페이지 외 다른 곳(dashboard·home)에서도 검정 폭주 우려 있는지 확인
- 가능하면 `::before` 에 `pointer-events:none;contain:strict;` 추가 권장
- `.hk-ico`, `.hello-avatar`, `.compose-icon` 등 SVG 컨테이너에 `flex-shrink:0;width:Xpx !important;height:Xpx !important;` 강제

### 4-2. 시안 #21 (팀원 권한) 작업 가이드 갱신
- 본 핫픽스 마크업을 기반으로 진행 권장 (`.kpi-grid + .kpi-card` · 인라인 토스트)
- 시안 마크업도 동일 안정 클래스 사용 부탁 (재폭주 방지)

### 4-3. (선택) 시안 #21 시각 디자인 추가 제안
- KPI 5개에 작은 진행률 바 (kpi-grid 카드 하단 sage progress bar) — 권한 부여율 시각화
- 권한 매트릭스 표 모바일(<1024px) 카드뷰 레이아웃
- 다른 팀 점프 칩에 인원수 외 "권한 부여율 %" 표시 (예: "검사기팀 9 · 78%")
