# 02 개선제안 #04 — `/admin` 관리자 대시보드 축약

> **대상**: `/admin` (`01_HAIST_WORKS/app/templates/admin.html`, **220 라인**)
> **근거**: 현황파악 2차 §9 — F-17 (🔴) + F-18·F-19 (🟠)
> **우선순위**: 🟠/🔴 경계. 사용자는 CEO·admin 극소수지만 **비번 재생성·관리코드 업로드 같은 고위험 기능**이 섞여 있어 정돈 가치 높음.
> **정책 준수**: 외부 자산 0건.

---

## 문제 (What)

### F-17 — 8 섹션 중 4개가 Table · 스크롤 매우 김
- KPI 4 + 그라디언트 카드 2 + **비번 재생성** + **관리코드 업로드** + 팀 테이블 + 사용자 테이블 + 프로젝트 테이블 + 고객사 테이블 = 8 섹션
- CEO 가 한 작업 (예: 사용자 추가) 을 하려면 스크롤 길게 내려야 함. 데이터 2배 늘면 더 비대.
- **위험 기능(비번 재생성)** 과 **일반 조회(팀 목록)** 가 같은 페이지에 나열 → 실수 유발 가능.

### F-18 — "물류" 컬럼 시각 불일치
사용자 테이블 line 97 에 `<th style="color:#A5282C">물류</th>` — ceo/admin 은 "자동", member 는 toggle 이지만 같은 컬럼이라 **현재 규칙을 한눈에 읽기 어려움**.

### F-19 — 그라디언트 카드 2개가 section-head 와 시각 위계 다름
line 20–29. 파란·초록 그라디언트는 page-head·section-head 와 별개 시각 언어. 디자인 원칙 §8 "컴포넌트 일관성" 위배.

---

## 제안 (How)

### 제안 A (권장) — 탭 3분할: 대시보드 · 데이터 · 보안

```
/admin  (기본)
├─ [대시보드]  KPI 4 · 팀 카드 · 요약 "사용자 84명 →", "프로젝트 23건 →", "고객사 12건 →"
├─ [데이터]    관리코드 업로드, 데이터 내보내기
└─ [보안]      비번 일괄 재생성 (2단계 확인)
```

- **일반 조회는 대시보드**, **일회성 대량 작업은 데이터**, **위험 기능은 보안** 으로 물리 분리.
- 실수 방지: 사용자가 "오늘 업무" 하러 왔다가 비번 재생성 버튼 실수 클릭할 확률 0.

### 제안 B — 4 테이블을 "요약 카드 + 관리 페이지 링크" 로 축약

기존:
```
사용자 (84)
[이름 · 로그인ID · 팀 · 직급 · 역할 · 상태 · 물류]  ← 84행 모두 렌더
```

제안:
```
┌─ 사용자 ────────────────┐
│ 84명 (활성 80 · 비활성 4)  │
│ [+ 추가]  [목록 관리 →]  │
└─────────────────────────┘
```
목록 관리는 `/admin/users` 새 페이지. 페이지네이션·검색·필터 정식 제공.

- `/admin` 자체는 "현황 파악"에 집중, 실제 관리는 전용 페이지.
- 기존 admin.html 의 테이블 로직은 `/admin/users` · `/admin/projects` · `/admin/customers` 로 **이동만**. 재작성 아님.

### 제안 C — 그라디언트 카드 2개 → 일반 카드로 통일

line 20–29 의 "시스템 설정 / 건전성 점검" 을 일반 카드 스타일로 변경:
```html
<div class="admin-nav-grid">
  <a class="admin-nav-card" href="/admin/settings">
    <div class="anc-icon">⚙️</div>
    <div class="anc-title">시스템 설정</div>
    <div class="anc-sub">URL·토큰·알림 마스터</div>
  </a>
  <a class="admin-nav-card" href="/admin/health">
    <div class="anc-icon">🩺</div>
    <div class="anc-title">건전성 점검</div>
    <div class="anc-sub">동작·설정 필요 항목</div>
  </a>
</div>
```
배경은 그라디언트 아닌 단색 카드 + hover 시 border-color만 변경.

### 제안 D — 비번 재생성에 2단계 확인 모달

현재: 버튼 1클릭 → 즉시 실행.
제안:
1) 1차 버튼 → 모달 "정말 전체 비밀번호를 재생성하시겠습니까? 이 작업은 즉시 적용됩니다."
2) 확인 체크박스 "위 내용을 이해했습니다" + 입력란 "대표이사명 입력" (예: "김정락")
3) 문자열 일치 시 버튼 활성 → 2차 확인 → 실행

**근거**: 04 OPS-008 (비번 동일 knk1234) 과 연계. 본 기능은 전체 운영에 영향. 실수 방지 장치 필수.

### 제안 E — 사용자 테이블 "물류" 컬럼을 "권한" 토글로 일반화

- ceo/admin 은 role 자동 부여 → "자동 (관리자)" 표시
- leader 이상은 명시 토글
- member 는 team sector 따라 기본값 + 토글 (04 OPS-001 권고와 일치)

컬럼명도 "물류" → "자재 권한" 으로 명료화.

## 예상 효과

- `/admin` 응답 HTML 길이 약 **60% 감소** (4 테이블 → 4 요약 카드). 렌더링 빠름.
- 비번 재생성 실수 확률 ≈ 0 (2단계 확인).
- CEO 가 `/admin` 에서 필요한 작업까지 평균 클릭 수: 현재 1~3 → **탭 선택 1 + 작업 1** = 2.
- 시각 언어 일관성 확보 (그라디언트 카드 제거).

---

## 위험·제약

- 테이블 분리 시 `/admin/users`·`/admin/projects`·`/admin/customers` **3개 신규 라우트 필요**. 01 세션 작업 1~2시간 추정.
- 기존 CEO 가 "한 페이지에서 모두 보는" 습관이 있으면 2~3회 재학습.
- 비번 2단계 확인은 배포 킥오프에 1회만 쓰이는 기능 → 막기보다 **"실수 방지" 교훈**. 04 OPS-008 기준으로도 필요.
