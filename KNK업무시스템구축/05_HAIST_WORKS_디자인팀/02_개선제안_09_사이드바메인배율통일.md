## 0. 핵심 방침 (v5 · 2차 정리)

> **"좌 사이드바도 우측 빅터 도크처럼 고정 폭. 드래그·폰트 스케일링 JS 전부 제거. 메인은 `--fs-fluid-*` 유동 타이포로 통일."**

- 좌 사이드바 = **고정 260px** (벽)
- 우 빅터 도크 = 고정 420px (벽, 제안 #08)
- 메인창 = 두 벽 사이에서 유동 (창 폭에 따라 `--fs-fluid-*` 자동 스케일)
- **3영역 모두 같은 스케일 규칙 공유** → 배율 절대 안 깨짐

## 1. 현재 구조 · 문제 재현

### 1-1. 좌 사이드바 (동적 폰트, 폭 드래그)

`base.html:322~380` — JS 가 드래그 폭에 따라 폰트 13 → 24px 선형 변경
```js
const MIN_W = 180, MAX_W = 520;
const MIN_F = 13,  MAX_F = 24;
function applyFont(w){
  const r = Math.max(0, Math.min(1, (w - MIN_W)/(MAX_W - MIN_W)));
  sb.style.setProperty('--sb-font-size', (MIN_F + (MAX_F-MIN_F)*r).toFixed(1) + 'px');
  /* ... */
}
```

→ 사이드바 폭 바뀔 때마다 **폰트 들쭉날쭉**. 메인은 고정 px. 비율 파괴.

### 1-2. 메인창 (정적 고정 px · 30+ 개소)

`style.css` 일부
```css
body               { font-size: 18px }
.page-title h1     { font-size: 25px }
.kpi-num           { font-size: 33px }
.kpi-lab           { font-size: 15px }
.section-head h2   { font-size: 18px }
.btn-primary       { font-size: 16px }
/* ... 총 30+ */
```

→ 뷰포트·줌 변경 시 절대 크기만 일률적으로 커지거나 작아짐. 비율은 그대로 유지되지만 **사이드바가 드래그로 따로 움직여서** 전체 균형 망가짐.

### 1-3. 우측 빅터 도크 (고정)

제안 #08 로 `width: 420px` 고정. 내부 요소들이 본문과 같은 뷰포트 줌 배율로 함께 움직여 "딱 좋음" (대표 평).

### 1-4. 유동 타이포 토큰 미사용

`style.css:43~49` 에 `--fs-fluid-*` 7종 정의되어 있으나 로그인 (제안 #07) 외 본문·사이드바 **사용 0건**.

## 2. 해법 (2대 수정)

### (A) 좌 사이드바: 드래그 제거, 고정 폭, 본문 스케일 공유

**CSS 변경** (`style.css`)
```css
/* AS-IS (line 567) */
.sidebar{
  position:relative;width:260px;min-width:0;
  /* ... */
  --sb-font-size:17px;
  --sb-head-size:14px;
}

/* TO-BE */
.sidebar{
  position:relative;
  width: 260px;                    /* 고정. 드래그 제거 */
  flex-shrink: 0;
  background:#fff;
  border-right:1px solid var(--bd);
  padding:14px 8px;
  overflow-y:auto;
  height:calc(100vh - 57px);
  /* ★ 본문과 같은 유동 스케일 */
  --sb-font-size: var(--fs-fluid-md);   /* 13 → 16px, 뷰포트 연동 */
  --sb-head-size: var(--fs-fluid-xs);   /* 10 → 13px */
}
/* .sb-resize, .sb-resize::after, .sb-resize:hover 블록 전부 제거 */
```

**HTML 변경** (`base.html:57`)
```html
<!-- AS-IS -->
<div class="sb-resize" id="sbResize" title="드래그하여 너비 조절"></div>

<!-- TO-BE: 삭제 -->
```

**JS 변경** (`base.html:322~380`)
```js
/* AS-IS: 드래그 리사이즈 + applyFont IIFE 전체 */
(function(){
  const sb = document.getElementById('sidebar');
  const handle = document.getElementById('sbResize');
  /* ... MIN_W MAX_W MIN_F MAX_F ... applyFont ... mousemove/mouseup ... */
})();

/* TO-BE: IIFE 전체 제거. localStorage 'sb-width' 도 더 이상 무의미 → 정리 */
```

**결과**:
- 폭 **항상 260px** (데스크톱 기본). 드래그 핸들 시각적 요소도 사라져 깔끔.
- 글씨는 뷰포트 폭 기반으로 13~16px 부드러운 자동 조정 (본문과 동일 스케일).
- 우측 도크와 **대칭 구조** (양쪽 모두 고정 벽, 가운데만 유동).

### (B) 좁은 해상도 미디어 쿼리 (기존 유지 · 축소 버전)

`style.css:751~789` 의 기존 미디어 쿼리는 **자동 폭 조정용** 이므로 유지:
```css
@media (max-width: 1280px) { .sidebar { width: 240px } }
@media (max-width: 1100px) { .sidebar { width: 220px } }
@media (max-width: 1024px) { .sidebar { width: 200px } }
@media (max-width:  768px) {
  .sidebar { position:fixed; left:-240px; top:52px; z-index:50; width: 240px; transition: left .25s; }
  .sidebar.sb-open { left: 0; }
  /* 기존 모바일 서랍 패턴 유지 */
}
```

→ 사용자가 수동으로 조절할 필요 없음. 화면이 작으면 OS 가 알아서 좁히고, 모바일은 햄버거 서랍.

### (C) 메인 고정 px 30+ 개소 → 유동 토큰 일괄 교체

매핑 규칙:

| 기존 고정 px | 대체 토큰 | 용도 예시 |
|---|---|---|
| 10~12px | `--fs-fluid-xs` | 배지, 메타, 작은 힌트 |
| 13~14px | `--fs-fluid-sm` | 보조 텍스트, 테이블 헤더 |
| 15~17px | `--fs-fluid-md` | **본문 기준** (= 사이드바와 동일) |
| 18~19px | `--fs-fluid-lg` | 섹션 제목, 강조 |
| 21~27px | `--fs-fluid-xl` | 페이지 타이틀, KPI 라벨 |
| 30~36px | `--fs-fluid-2xl` | KPI 숫자, 대시보드 수치 |
| 40px+ | `--fs-fluid-hero` | 로그인·홈 영웅 타이포 |

주요 교체 셀렉터 (`style.css`):
```
body, input, select, textarea   → var(--fs-fluid-md)
.page-title h1                  → var(--fs-fluid-xl)
.page-sub                       → var(--fs-fluid-sm)
.kpi-num                        → var(--fs-fluid-2xl)
.kpi-lab, .kpi-sub              → var(--fs-fluid-sm) / xs
.section-head h2                → var(--fs-fluid-lg)
.section-head .count            → var(--fs-fluid-xs)
.btn-primary                    → var(--fs-fluid-sm)
.btn-sm, .btn-xs                → var(--fs-fluid-sm) / xs
.team-tbl, .hist-tbl, .mini-tbl → var(--fs-fluid-sm)
.feed-task                      → var(--fs-fluid-md)
.cal-d, .cal-stat, .cal-hr      → var(--fs-fluid-sm) / xs
.hl-body                        → var(--fs-fluid-md)
.modal-title                    → var(--fs-fluid-lg)
.mem-name, .mem-stat            → var(--fs-fluid-md) / sm
.reminder-banner .rb-ico        → var(--fs-fluid-2xl)
.bell                           → var(--fs-fluid-lg)
.bell-badge                     → var(--fs-fluid-xs)
/* ...전체 30+ 개 */
```

### (D) 인라인 `style="font-size:10px"` 정리

`base.html:116, 121, 126, 131, 142, 162, 165, 169` 의 인라인 배지 → CSS 클래스(`.sb-inline-badge`)로 이동:
```css
.sb-inline-badge{
  margin-left:auto;color:#fff;
  font-size:var(--fs-fluid-xs);
  font-weight:800;padding:2px 8px;border-radius:999px;
}
```

## 3. 시각 비교 (예상 효과)

### 3-1. Before / After (1920px 데스크톱)

| 요소 | AS-IS (드래그 260px 기준) | TO-BE (고정 260px) |
|---|---|---|
| 사이드바 메뉴 | 17.0px (JS 계산) | **15.7px** (`--fs-fluid-md`) |
| 사이드바 섹션 헤더 | 14.0px | **12.5px** (`--fs-fluid-xs`) |
| 메인 본문 | 16px 고정 (여기저기 15~18px 혼재) | **15.7px** (공통) |
| 페이지 제목 | 25px 고정 | **24.0px** (유동) |
| KPI 숫자 | 33px 고정 | **32.0px** (유동) |
| 테이블 셀 | 15px 고정 | **13.8px** (유동) |

→ 사이드바 메뉴 15.7px = 본문 15.7px **완전 동일**. 시각 비율 통일.

### 3-2. 사이드바 폭은 사용자가 건드릴 수 없음

| 뷰포트 폭 | 사이드바 폭 (미디어 쿼리) | 사이드바 폰트 | 본문 폰트 | 비율 |
|---|---|---|---|---|
| 1920px | 260px | 15.7px | 15.7px | 1.00 |
| 1440px | 260px | 14.5px | 14.5px | 1.00 |
| 1280px | 240px (자동) | 14.0px | 14.0px | 1.00 |
| 1100px | 220px (자동) | 13.8px | 13.8px | 1.00 |
| 1024px | 200px (자동) | 13.7px | 13.7px | 1.00 |
| 768px 이하 | 서랍 240px | 13.4px | 13.4px | 1.00 |

→ **어떤 해상도에서도 사이드바 ↔ 본문 비율 1:1 유지**.

### 3-3. 제거되는 요소

- ❌ `.sb-resize` 드래그 핸들 (시각적 요소)
- ❌ 호버 시 붉은 라인 피드백
- ❌ JS `applyFont(w)` 함수 + mousedown/mousemove/mouseup 3개 리스너
- ❌ localStorage `sb-width` 키
- ❌ `--sb-font-size`, `--sb-head-size` CSS 변수의 JS 동적 설정 (CSS 정적 바인딩으로 남음)

총 **코드 30여 줄 순감소**.
