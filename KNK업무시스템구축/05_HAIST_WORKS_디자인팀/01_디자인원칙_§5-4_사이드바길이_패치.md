## §5-4. 사이드바 세로 길이/overflow 정책 (신설)

본 절은 사이드바가 뷰포트 하단을 초과해 짤리거나 본문 스크롤과 충돌하지 않도록 강제하는 규칙이다. 핫패치 사고(2026-04-24 17:30) 근본원인 차단을 목적으로 한다.

### §5-4-1. max-height 강제 — `calc(100vh - var(--topbar-h))`

- 사이드바 컨테이너는 반드시 다음 중 하나로 세로 길이를 강제한다.
  - `height: calc(100vh - var(--topbar-h));` (고정형)
  - `max-height: calc(100vh - var(--topbar-h));` (가변 콘텐츠형)
- `--topbar-h`는 디자인 토큰(00_design_tokens.css)에서 단일 정의 — 인라인·매직넘버(57px 등 직접 입력) 금지.
- 04 PASS 영역(line 618 `height:calc(100vh - 57px)`)은 토큰 도입 후 `var(--topbar-h)`로 치환 발주.

### §5-4-2. overflow-y 정책 — `auto` 또는 `collapse` 1택

- **A안 (기본 권장)**: `overflow-y: auto;` — 항목이 뷰포트를 넘으면 사이드바 내부 스크롤.
- **B안 (5그룹 이상 시)**: 카테고리 접기(collapse) 패턴 — 아래 §5-4-4 참조.
- A·B 동시 사용 금지(스크롤 + 접힘이 동시에 일어나면 멘탈모델 붕괴).
- `overflow: hidden`은 항목 손실 위험으로 **절대 금지**.

### §5-4-3. depth 2 이내 유지 (PASS 상태 보존)

- 04 T1 §1 PASS 항목 "사이드바 depth 2 이내" 상태는 변경 없이 유지.
- 그룹 헤더 1단계 + 링크 1단계 = 총 2단계. 3단계 트리(예: 그룹 > 서브그룹 > 링크) 도입 금지.
- 신규 메뉴 추가 시 depth 추가가 필요하면 새 그룹을 만들 것 — depth를 늘리지 말 것.

### §5-4-4. 5그룹 이상 시 카테고리 접기 권고

- Plan Y S1 시안에서 5그룹 도입 학습.
- **그룹 수 ≥ 5** 일 때 다음 중 1택 의무:
  1. 첫 1~2개 그룹만 expanded, 나머지는 collapsed (사용자 클릭으로 펼침).
  2. 사용자별 즐겨찾기 그룹 1개 상단 고정 + 나머지 collapsed.
- 그룹 접기 상태는 `localStorage` 또는 사용자 설정에 저장 — 새로고침마다 초기화 금지.

### §5-4-5. 자기 검증 grep 패턴 (시안 + LIVE 양쪽)

본 §5-4 적용 후 다음 grep이 모두 PASS여야 함.

```
# G16-1: 사이드바 max-height 또는 height 정의
grep -nE "(side|nav|sidebar)[^{]*\{[^}]*(max-height|height)\s*:\s*calc\(100vh" <target.css>
# 기대: 1건 이상 (해당 사이드바 셀렉터)

# G16-2: overflow-y 정의
grep -nE "(side|nav|sidebar)[^{]*\{[^}]*overflow-y\s*:\s*(auto|scroll|hidden)" <target.css>
# 기대: auto 또는 scroll 1건 이상, hidden 0건

# G16-3: 매직넘버 57px 잔존 점검 (토큰 치환 추적)
grep -nE "calc\(100vh\s*-\s*57px" <target.css>
# 기대: 0건 (토큰 치환 후) — 1건 이상이면 §5-4-1 위반

# G16-4: 3 depth 트리 점검 (depth 2 강제)
grep -cE "<ul[^>]*>[^<]*<li[^<]*<ul[^>]*<li[^<]*<ul" <target.html>
# 기대: 0건
```

검증 대상: `01_HAIST_WORKS/static/style.css` (LIVE) + `05_HAIST_WORKS_디자인팀/03_시안/*` (시안)
검증 책임: 05 발행 → 01 적용 → 00 grep 게이트 (3단 게이트, 00 A1 권고 #2)
