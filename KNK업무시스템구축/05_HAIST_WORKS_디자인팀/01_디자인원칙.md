## 0. 외부 상표권·저작권 절대 비침해 (최상위 조항)

> 대표 지시 (2026-04-21): "글씨체, 그림 등부터 외부 상표권에 위반되지 않게 만들어야 해."

1. 타사 상표·저작물을 모방·무단 사용하지 않는다.
2. 모든 외부 자산은 **라이선스 문서화** + **고지 의무 준수**.
3. 타사 브랜드명(Stripe·Vercel·Notion·Figma·Linear·Material 등)을 **내부 문서·UI·CSS 주석에도 언급 금지**. 영감 표현도 중립 용어로.
4. 의심스러우면 사용하지 않고 대표 승인 (`04_라이선스_상표권_정책.md` §5 양식).
5. 매 제안 발행 전 **§6 체크리스트 5 항목 통과** (같은 정책 문서 참조).

위반 발견 시 즉시 작업 중단 → 대표·09 팀장 보고 → 롤백.

## 1. 실패 회피 10계명 (First, Do No Harm)

1. **메뉴트리 2 depth 이내**. 사이드바 → 페이지 → (있다면) 모달. 3 depth 이상 금지.
2. **한 화면에 강조 CTA 1개**. 여러 버튼이 있어도 "가장 눌러야 할 버튼"은 색·크기로 유일해야 함.
3. **KPI 그리드는 한 화면에 1개**. 두 그리드가 필요하면 탭으로 분리.
4. **장식 데이터 금지**. 스파크바·그래프는 **실제 데이터**일 때만. 하드코딩 장식은 감사팀 원칙(자료 정확성) 위반.
5. **라벨 없는 아이콘 금지**. SVG만 있으면 신입은 못 읽음. 반드시 텍스트 라벨 병기.
6. **역할별 화면 분기는 route 로 (또는 탭으로)**. 같은 route 가 role 따라 섹션이 붙었다 사라지는 구조는 멘탈모델 붕괴 원인. 예: `/home` 을 탭 3분할하거나 `/home-lead` 분리.
7. **placeholder 는 7자 이내**. 긴 문장은 안내문으로 분리 (placeholder 는 힌트, 안내는 `<small>` 또는 i 아이콘).
8. **테이블 컬럼 7개 초과 시 반응형 재구성 필수** (v2 추가). min-width: 1400px 같은 "데스크톱만 가능" 테이블은 모바일에서 "카드 뷰"로 자동 전환 (`/progress` 제안 #02 패턴).
9. **위험·비가역 기능은 2단계 확인** (v2 추가). 비밀번호 일괄 재생성·데이터 삭제·대량 처리 등 되돌릴 수 없는 기능은 체크박스 + 문자열 입력 2단계 확인. `/admin` 제안 #04 패턴 참조.
10. **필터·네비는 sticky** (v2 추가). 긴 리스트 중간에서도 필터 전환 가능해야 함. `position: sticky; top: 56px;` 기본. `/feed` 제안 #03 패턴.

## 2. 색 (Color)
> ⚠️ 본 절의 모든 색은 HEX 값으로만 정의 — 특정 타사 팔레트명(Tailwind·Material 등) 인용 금지. §0 참조.

### 2-1. 브랜드 고정
- **KNK RED 주색**: `#A5282C`
- **KNK RED 다크**: `#8B1E22` (그라디언트 종점)
- **Primary gradient**: `linear-gradient(135deg,#A5282C,#8B1E22)` — **하나의 화면에 단 1곳**에만 적용 (예: 주 CTA).

### 2-2. 상태 색 (4종, 이미 코드베이스에 정착 — 유지)
- 완료 `#10B981` / 진행중 `#F59E0B` / 지연 `#EF4444` / 대기 `#94A3B8`.
- 이 4색은 **상태 태그 이외 용도로 사용 금지**. KPI 아이콘이 이 색을 쓰면 "상태"로 오독됨.

### 2-3. KPI 아이콘 색 (새 규칙)
- KPI 는 **중립 계열**만 사용: 슬레이트 `#475569`, 다크 `#1A1A1A`, 배경 `#F1F5F9`.
- 다채색(`red/blue/green/purple`) KPI 아이콘 팔레트는 **폐기**. 홈 화면 시각 과밀 주범(H-2).

### 2-4. 배경
- Page `#FAFBFC` (현재 유지).
- Card `#FFFFFF`, border `#E5E7EB`, shadow `0 1px 2px rgba(15,23,42,0.04)`.

## 3. 폰트 (Typography)

> ⚠️ 폰트 라이선스는 `04_라이선스_상표권_정책.md` §1 참조. 번들링·CDN 로딩은 대표 사전 승인.

- Family: **Pretendard** (SIL OFL 1.1, 상업 사용 OK) → 시스템 폴백 `system-ui, -apple-system, "Segoe UI", Roboto, "Apple SD Gothic Neo", "Malgun Gothic", sans-serif`.
- Pretendard 파일 **번들링 vs 선언만** 여부는 대표 결정 사항 (정책 §1-4).
- 크기 스케일 (고정 6단계):
  | 토큰 | px | 용도 |
  |---|---|---|
  | `fs-xs` | 11 | 캡션·배지 |
  | `fs-sm` | 13 | 본문 부가 |
  | `fs-md` | 14 | 본문 기본 |
  | `fs-lg` | 16 | 섹션 타이틀 |
  | `fs-xl` | 20 | 페이지 타이틀 |
  | `fs-2xl` | 28 | KPI 숫자 (한 화면 최대 1–2개) |
- 폰트 가중치: 400 / 600 / 800 (이 3단계만). 500·700 혼재 금지.
- 한국어 가독을 위해 **letter-spacing 0 기본**. 영어 헤더만 `0.02em` 허용.

## 4. 간격 (Spacing)

- 8px 그리드: 4 / 8 / 12 / 16 / 24 / 32 / 48.
- **섹션 간 간격 최소 24px**. 섹션 구분이 약하면 이용자는 "한 덩어리"로 보고 압도됨 (홈 H-1 원인).
- 카드 내 패딩 16–20px.

## 5. 네비게이션 (Navigation)

### 5-1. 사이드바 규칙
- 섹션 묶음은 최대 6개 (현재 12개 — 감축 필요).
- 각 섹션 당 링크 3개 이하 권장.
- **역할별 숨김 메뉴는 사이드바에서 "비활성 표시"** 가 아니라 **완전 미노출** (현재와 동일 방향 유지).

### 5-2. 클릭 예측 원칙
- 아이콘 옆 텍스트 **필수**.
- 현재 보는 메뉴는 **active 상태 명확히**: 배경 `#FFEBEE`, 좌측 3px 빨간 바.
- 하위 기능으로 들어갈 때 페이지 이동(전체 refresh)보다 **인라인 펼침·모달 우선**.

### 5-3. 홈(`/home`) 특별 규칙
- **역할별 뷰 분리** (실패 회피 6번): `/home` 은 member 기본. 리더 이상은 `/home`에서 상단에 "리더 대시보드 보기" 버튼으로 `/home-lead` 로 이동.
- 또는 홈 내 **탭** 으로 "내 업무 / 우리 팀 / 전사" 분할. 같은 화면에 9개 섹션 세로 나열 금지.

## 6. 아이콘 (Icon)

- **Lucide** (ISC 라이선스 — 상업·수정·재배포 자유, 고지 포함 의무). inline SVG path 로 사용.
- **단일 소스만**. Font Awesome·Material Icons·Bootstrap Icons 혼용 금지 (정책 §2-2).
- 프로젝트 루트 `LICENSE` 파일에 Lucide 고지 1회 포함 의무.
- 크기 `16 / 18 / 20 / 24` 4단계만.
- stroke-width `2` 표준. 강조는 `2.5`.
- 이모지는 **접근성 보조** 로만 사용 (🏠📝📅). 본문 대체 불가. OS 별 렌더 차이로 브랜드 일관성 저하.
- ❌ Apple SF Symbols / 타사 서비스 로고(카카오·구글·네이버 등) / 출처 불명 SVG 금지.

## 8-bis. 반응형 · 디바이스 (v2 신설)

### 8-bis-1. 브레이크포인트
- 데스크톱: **≥ 1024px**. 매트릭스·다컬럼 테이블·대시보드 풀폭 허용.
- 태블릿: **768–1023px**. 2단 그리드 → 1단, 보조 컴포넌트 숨김.
- 모바일: **< 768px**. 카드 뷰 필수. 테이블은 반드시 카드 또는 수평 carousel.

### 8-bis-2. 자동 뷰 전환 규칙
- 정보 밀도가 높은 화면(progress·cockpit·dashboard·admin) 은 **매트릭스 ↔ 카드** 뷰 2종 제공.
- 뷰 전환은 **CSS media query 자동** + 사용자 수동 세그먼트 버튼 병행.
- 자동 전환만 쓰면 "데스크톱에서 카드로 보고 싶은 사용자" 를 차단 → 수동 버튼 필수.

### 8-bis-3. 공장·현장·출장 환경 가정
- 회선 약한 곳 · 저사양 PC · 스마트폰 예상.
- **페이지당 HTML ≤ 100KB** 목표 (04 OPS-004 참조: cockpit 100KB 경계).
- 인라인 onclick 50개 이상 금지 → event delegation.
- 외부 CDN 호출 0 (정책 §0 이미 규정).
