## TL;DR

- **좌 사이드바 = 고정 260px** (드래그 핸들·폰트 스케일링 JS 제거)
- **메인 = `--fs-fluid-*` 유동 타이포 일괄 적용** (고정 px 30+ 개소 → 토큰)

## 6. 리스크 · 제약

- 기존 드래그 기능에 익숙한 사용자 **재학습 1회 필요**. 04 세션 피드백에서 실제 사용률 낮게 기록되어 영향 미미 예상.
- 사용자별 localStorage `sb-width` 저장값 무시됨. 깨끗이 정리하려면:
  ```js
  localStorage.removeItem('sb-width');
  ```
  1회 실행 or 자연 방치 (영향 없음).
- 폰트 축소 체감(17 → 15.7px) 발생 가능 → 사용자 피드백 후 토큰 상한 1회 튜닝 권장.
- 1100~1280px 중간 해상도에서 테이블 컬럼 폭 재조정 필요할 수 있음 (제안 #04 병행 권장).

## 7. 예상 효과

1. **좌 사이드바 + 우 빅터 도크 = 대칭 벽 구조**. 시각 인지 단순화.
2. 1024 ~ 4K 전 해상도에서 **사이드바 ↔ 메인 폰트 비율 1:1**.
3. 드래그·폰트 스케일링 JS **30줄 순삭제** → 유지보수 부담 감소.
4. 대표 지시 **완전 충족**: "왼쪽 사이드바도 조정하지 않게 정리 · 오른쪽 사이드바 처럼".
