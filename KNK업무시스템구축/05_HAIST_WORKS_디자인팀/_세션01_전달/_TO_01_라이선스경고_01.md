### 🟡 O-1. `translate.googleapis.com` 비공식 endpoint (main.py:903)

**코드**:
```python
url = f"https://translate.googleapis.com/translate_a/single?client=gtx&sl=auto&tl={target}&dt=t&q={encoded}"
```

**관찰**:
- 이 엔드포인트는 **Google 공식 Cloud Translation API가 아님**. `client=gtx`는 브라우저용 비공식 경로.
- Google Cloud Translation API(유료)는 별도 키 필요. 현재 코드는 **무료 사용**이지만 **Google 이용약관(TOS) 회색지대**.
- 레이트 리밋·차단·예고 없는 서비스 중단 가능성. 법적 회색지대.
- 04 OPS-003 에서 베트남 직원 언어 UI 이슈와 별개의 문제.

**권고** (05 의견, 대표 판단 필요):
1. **현상 유지**: 내부 업무 트래픽만. 낮은 볼륨이면 실질적 리스크 낮음.
2. **공식 API 전환**: Google Cloud Translation API 가입 (월 50만 자 무료, 이후 유료). 예산 필요.
3. **대체**: DeepL API 무료 플랜 (월 50만 자), Papago API (네이버, 한-외 특화).
4. **제거**: 카드 번역 기능 자체를 제거하고, 사용자가 직접 번역 도구 사용하도록 안내.

**대표 승인 필요**: 어느 방향으로 갈지 결정.

### 🟢 O-2. `font-family: 'JetBrains Mono'` 하드코딩 (4 템플릿)

**위치**:
- `app/templates/change_detail.html:14, 61, 65`
- `app/templates/issue_detail.html:14`

**코드 예시**:
```html
<span style="font-family:'JetBrains Mono',Consolas,monospace;...">
```

**관찰**:
- **JetBrains Mono** 폰트는 **Apache 2.0** 라이선스 — 상업 사용 OK.
- **"JetBrains"는 JetBrains s.r.o.의 상표**. 그러나 **font-family 폴백 체인에 선언만**하는 것은 기술적 용법으로 상표권 문제 없음.
- 실제 렌더는 사용자 PC 에 JetBrains Mono 설치 시 → JetBrains Mono, 없으면 → Consolas(Microsoft, Windows 내장) → monospace 로 폴백.
- **법적 리스크 없음**. 그러나 **일관성 관점**에서 문제.

**권고**:
- 프로젝트 공통 토큰화:
  ```css
  :root {
    --font-mono: "JetBrains Mono", Consolas, "Courier New", monospace;
  }
  ```
- 각 템플릿은 `style="font-family:var(--font-mono)"` 로 교체.
- 우선순위 낮음.
