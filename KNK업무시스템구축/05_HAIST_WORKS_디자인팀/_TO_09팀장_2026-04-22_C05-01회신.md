## 보강 페르소나 3인 — `/admin` 3탭 (대시보드 · 데이터 · 보안)

### 페르소나 A — 김정락 대표이사 | CEO / ceo role

- **직책**: 대표이사 (ceo role, `/dashboard` 접근 가능)
- **방문 빈도**: 매일 오전 출근 직후 3~5분
- **가장 자주 보는 탭**: **대시보드** — 전사 KPI 4개 + 팀별 현황 카드로 "오늘 뭔가 이상한 팀이 있나" 즉시 파악
- **현재 `/admin` 주요 불만**:
  - 8섹션 스크롤 중 비번 재생성 버튼이 가장 자주 쓰는 사용자 테이블 위에 위치 → 실수 클릭 공포
  - 팀 테이블 84행 전체 렌더링 → "팀 요약만 보고 싶은데 너무 길다"
  - 사용자 추가하려면 스크롤을 한참 내려야 함
- **제안 #04 3탭이 해결하는 방식**:
  - 대시보드 탭: KPI + 팀 요약 카드 (84명 전체 테이블 아님) → 3분 내 파악 가능
  - 위험 기능(비번 재생성)이 보안 탭에 물리 분리 → 실수 클릭 0
  - 데이터 탭에서 관리코드 업로드만 독립 노출 → 목적지향 UX

---

### 페르소나 B — 박미정 총무팀장 | admin role 대리인

- **직책**: 총무팀장 (admin role 위임, 신입 입사 시 계정 생성·팀 배정 담당)
- **방문 빈도**: 주 2~3회, 신입 입사·퇴사·팀 이동 발생 시
- **가장 자주 보는 탭**: **데이터** — 관리코드 업로드, 사용자 명단 xlsx 업로드로 일괄 처리
- **현재 `/admin` 주요 불만**:
  - 사용자 추가 버튼과 데이터 내보내기 버튼이 같은 화면에 섞여 있어 "지금 어떤 작업을 해야 하는지" 흐름 잡기 어려움
  - 사용자 테이블 스크롤이 너무 길어 대상자 찾으려면 Ctrl+F 사용
  - "물류" 컬럼 토글 의미를 매번 IT에 물어봐야 함 (F-18 이슈)
- **제안 #04 3탭이 해결하는 방식**:
  - 데이터 탭에 사용자 관리 + 관리코드 업로드 + 내보내기를 집중 배치
  - "물류" 컬럼 → "자재 권한" + 설명 툴팁으로 자체 해결 가능
  - 페이지네이션 + 검색이 있는 `/admin/users` 전용 페이지로 이동 → 84행 스크롤 제거

---

### 페르소나 C — 장현우 IT 관리자 | admin role 전담

- **직책**: IT 관리자 (admin role, 시스템 보안·설정·건전성 점검 담당)
- **방문 빈도**: 매주 1회 정기 점검 + 보안 이슈 발생 시 즉시
- **가장 자주 보는 탭**: **보안** — 비번 재생성, 시스템 설정, 건전성 점검
- **현재 `/admin` 주요 불만**:
  - 비번 재생성 버튼이 1클릭 즉시 실행 → "실수로 누르면 전사 비번이 한 번에 바뀐다"는 공포 (OPS-008 연계)
  - 시스템 설정(URL·토큰·알림) 변경 시 KPI 숫자들을 스크롤로 지나쳐야 함 → 집중력 분산
  - 건전성 점검 항목이 메인 어드민에 묻혀 있어 "이게 정상인지 비정상인지" 빠른 판단 어려움
- **제안 #04 3탭이 해결하는 방식**:
  - 보안 탭에 비번 재생성 2단계 확인 모달 + 시스템 설정 + 건전성 점검 집중 배치
  - 위험 기능이 시각적으로 격리 (탭 전환 의도적 행동 必) → 실수 방지
  - 건전성 점검을 보안 탭 하단에 고정 → 정기 점검 루틴화

## 3탭 지지 논거 요약

세 페르소나는 `/admin` 방문 목적이 완전히 다르다. 김정락 대표(A)는 "현황 파악용 조회"가 목적이고, 박미정 총무팀장(B)은 "사용자 데이터 입력·관리"가 목적이며, 장현우 IT 관리자(C)는 "위험 기능 실행·시스템 점검"이 목적이다. 현재 `/admin`은 이 세 목적을 220라인 한 페이지에 섞어 놓아 A는 스크롤 피로, B는 흐름 혼란, C는 실수 공포를 각각 유발한다. 제안 #04의 대시보드·데이터·보안 3탭은 이 세 페르소나의 목적을 각각 독립된 작업 공간으로 물리 분리해, 탭 이동 자체가 "나는 지금 이 목적으로 왔다"는 의도 선언이 되도록 설계된 구조다.
