## 배분 원칙

| 탭 | 핵심 사용자 의도 | 컨텐츠 방향 |
|----|----------------|-------------|
| **내 업무** (`tab=my`) | "오늘 내 할 일을 빠르게 확인·처리" | 개인 KPI + 이월 + 나의 업무 카드 + 개인 집계 |
| **우리 팀** (`tab=team`) | "우리 팀이 오늘 어떻게 돌아가고 있나" | 팀 작성률 + 실시간 피드 + 팀 현황 그리드 |
| **전사** (`tab=all`) | "전사 흐름과 주간 추세를 경영적으로 파악" | 주간 트렌드 + 전사 팀 현황 + 상위 집계 |

> **빅뱅 방지 원칙**: 서버 측 데이터 쿼리는 탭 전환 시 그대로 유지. 화면에서 CSS `display:none` 처리로 우선 배분하고, 추후 성능 최적화(탭별 lazy-load)는 별도 제안으로 분리.

## 구현 메모 (01 메인 개발팀)

### 방법 A — Jinja `if tab==` 조건부 렌더 (권장)

```jinja
{# 내 업무 탭 전용 섹션 #}
{% if tab == 'my' %}
<div class="hm2-kpi-grid"> ... KPI 4카드 ... </div>
{% if pending_yday %}<div class="hm2-carry"> ... </div>{% endif %}
{% if hw_counts is defined %}<div class="hm2-kpi-grid"> ... 협업 KPI ... </div>{% endif %}
{% endif %}

{# 우리 팀 탭 전용 섹션 #}
{% if tab in ('team','all') and user.role in ['leader','executive','ceo','admin'] %}
<div class="hm2-grid-2"> ... 실시간 피드 + 팀 작성률 ... </div>
{% endif %}

{# 전사 탭 전용 섹션 #}
{% if tab in ('team','all') and user.role in ['leader','executive','ceo','admin'] %}
<div class="hm2-grid-2"> ... 주간 트렌드 + 도넛 ... </div>
{% endif %}

{# 나의 업무 카드 — 내 업무 탭 + 팀 탭에서도 상단 간소화 버전 표시 #}
{% if tab == 'my' %}
<div class="hm-my-section"> ... 전체 카드 ... </div>
{% endif %}

{# 팀/전사 현황 그리드 #}
{% if tab in ('team','all') and user.role in ['leader','executive','ceo','admin'] %}
<div class="hm-team-zone"> ... </div>
{% endif %}
```

### 방법 B — CSS 클래스 토글 (빠른 구현)

서버 변경 없이 `<div class="tab-section tab-my">` / `<div class="tab-section tab-team">` 클래스 추가 후 JS로 탭 전환 시 show/hide. 단, 모든 HTML이 항상 전송되므로 페이지 무게 감소 없음. 단기 빠른 배포에 유리.
