> **대상**: `/home` (`01_HAIST_WORKS/app/templates/home.html`, 815 라인)
> **선정 근거**: `00_현황파악_2026-04-21.md` §3 — H-1(9섹션)·H-2(KPI 이중)·H-3(스파크바 장식)·H-4(cycle 버튼)·H-5(긴 placeholder) 5개 후보 중 홈 집중.

## 문제 요약 (What)

현재 홈은 한 페이지에 **9개 섹션**이 세로로 쌓여 있음. 일반 사용자는 "내 업무" 한 가지만 보고 싶은데, 시야에 들어오는 정보는 "전사 주간 트렌드 차트·팀별 작성률·14팀 현황" 까지 모두 펼쳐짐(role 별 감춰지지만 role=leader 이상은 전부 보임). 신입(P1 엄주영)·현업(P3 마준영)은 **압도 → 이탈** 위험.

## 왜 문제인가 (Why)

1. 대표가 정의한 실패조건: "메뉴가 너무 깊고 뭘 눌러야 할지 모름" → 홈은 **깊이는 얕지만 너비가 과도**. 같은 원리로 압도 발생.
2. 04 운영테스트팀 OPS-005(카테고리 10개 안내 없음)·OPS-003(언어변경 UI 묻힘)과 같은 "신입의 멘탈 로드" 패턴.
3. 스파크바 28개가 **하드코딩 height %** — 감사팀 자료 정확성 원칙 저촉 가능 (`home.html` line 82~162).

## 제안 (How)

### 제안 A (최소 변경 · 권장) — 홈을 **3개 탭으로 분할**

```
┌──────────────────────────────────────────────┐
│ [탭] 내 업무  |  우리 팀  |  전사           │  ← 탭 3개만
├──────────────────────────────────────────────┤
│                                              │
│  (선택된 탭 내용만 보여줌)                   │
│                                              │
└──────────────────────────────────────────────┘
```

- **탭 1 "내 업무"** (기본, 모든 역할):
  - 상단 고정: 환영 + 날짜 네비 + "새 업무 등록"
  - KPI 4카드 (내 업무 / 완료율 / 누적 공수 / 지연) — **스파크바 제거**
  - 이월 배너
  - 나의 업무 카드 리스트
- **탭 2 "우리 팀"** (role=leader 이상):
  - 실시간 피드 + 팀별 작성률
  - 팀 현황 카드 그리드 + 디테일 패널
- **탭 3 "전사"** (role=ceo/admin/executive):
  - 주간 트렌드 + 내 업무 분포 도넛
  - HAIST 협업 핵심 KPI (진행률·변경·티켓·게시판)

### 제안 B (중간) — 홈 분리 + KPI 그리드 통합

- `/home` (member), `/home-lead` (리더), `/home-exec` (경영진) 3 route 로 분리.
- KPI 2그리드를 1그리드로 통합하고, HAIST 협업 핵심 4개는 **사이드바 상단 "내 할 일" 아래 배지로** 이동 (진행률 N · 변경 N · 티켓 N · 미확인 M).

### 제안 C (큰 변경 · 참고) — 대시보드/홈 경계 재정의

- 역할별 도착 route 를 로그인 직후 `user.role` 로 분기 (`/home` → member, executive 는 `/dashboard` 로 바로).
- `/home` 은 "내 업무" 전용으로 완전 단순화.
- `/dashboard` 는 "전사" 전용.

→ 대표·09 팀장과 협의 후 단계적 전환. 본 1차 제안의 핵심은 **제안 A**.

## 상태 cycle 버튼 (H-4) — 세그먼트 4버튼으로

현재 (line 458~462):
```html
<button class="hm-status-btn ..." onclick="cycleStatus({{ t.id }}, '{{ t.status }}')">
  {{ t.status }}
</button>
```

제안:
```html
<div class="hm-status-seg" role="group" aria-label="상태">
  <button data-s="진행중" class="{{ 'on' if t.status=='진행중' }}">진행중</button>
  <button data-s="완료"   class="{{ 'on' if t.status=='완료'   }}">완료</button>
  <button data-s="지연"   class="{{ 'on' if t.status=='지연'   }}">지연</button>
  <button data-s="대기"   class="{{ 'on' if t.status=='대기'   }}">대기</button>
</div>
```

- 1-클릭으로 원하는 상태 지정. 실수 복귀 즉시.
- 접근성 향상.

## 예상 효과 (측정 기준)

- 홈 응답 HTML 길이 **약 40% 감소** (스파크바·중복 KPI·미사용 role 섹션 서버측 제거).
- "어디를 봐야 하는지" 탭 3개로 축약 → 신입 P1 로그인 후 3초 내 "내 업무" 위치 확인.

## 위험·제약

- 탭 분할은 SPA 가 아니라 페이지 전환(query param ?tab=my/team/all) 으로 구현 권장 (현재 코드베이스가 full refresh 계열). 북마크 가능성 유지.
- role 별 자동 탭 default 는 로그인 후 1회 계산 (member=1, leader=2, exec=3). 사용자가 수동 선택 시 localStorage 로 기억.
