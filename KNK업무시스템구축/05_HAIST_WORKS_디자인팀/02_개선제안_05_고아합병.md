## 2. 합병 권고 상세

### 2-1. `/bottlenecks` → `/cockpit` 합병 (🔴 명백한 중복)

**근거**:
- `cockpit.html:65–76` 이 `bottlenecks.html:11–22` 와 동일 로직(`b.severity`, `b.rule`, `b.title`, `b.owner`).
- 데이터 소스 동일 (`bottlenecks` DB 쿼리).
- 사용자 입장: "코크핏 가면 병목 다 있는데 왜 메뉴가 또 있지?"

**권고 구현**:
```python
# main.py 기존 /bottlenecks 핸들러
@app.get("/bottlenecks")
async def bottlenecks_redirect(req):
    return RedirectResponse("/cockpit#bottlenecks", status_code=302)
```
- `bottlenecks.html` 템플릿은 **삭제 보류** (유사 페이지 재사용 가능)
- 사이드바에서 "🚨 병목" 링크를 **제거 또는** `#bottlenecks` 앵커로 변경
- `/cockpit#bottlenecks` 섹션에 `id="bottlenecks"` 추가

**예상 효과**: 메뉴 1개 감소, 사용자 학습 부담 감소.

### 2-2. `/now` → `/feed` 합병 (🟡 보완적)

**근거**:
- `/now` 는 "활동 로그 실시간 흐름" · `/feed` 는 "업무 카드 액션 보드"
- 같은 task 가 양쪽에서 각각 다른 관점으로 등장 → 사용자는 **"둘 다 보러 다니며 파편화"**
- 05 개선제안 #03 에서 `/feed` 에 "다른 방식으로 보기 ▾" 토글 제안 이미 있음 — 네 번째 옵션 `/feed?view=now` (실시간 활동) 편입

**권고 구현**:
```python
@app.get("/now")
async def now_redirect(req, scope: str = "all"):
    return RedirectResponse(f"/feed?view=now&scope={scope}", status_code=302)
```
- `/feed?view=now` 핸들러에서 now.html 의 실시간 피드 블록 렌더
- 사이드바 "⚡ 실시간" → `/feed?view=now` 로 링크 변경 (또는 제거)
- `now.html` 의 **30초 폴링 스크립트** 는 그대로 이식

**예상 효과**: 사이드바 1개 감소, 두 뷰의 맥락 통합.

### 2-3. `/cockpit` 은 유지 (단, 별도 이슈)

**근거**:
- 경영진이 "오늘 팀 상태 한 화면" 에서 쓰는 주요 진입점. 합병 시 `/dashboard` 와 성격 충돌.
- 04 OPS-004 에서 **cockpit 100KB · onclick 226회** 관찰 → 성능·복잡도는 별도 개선 필요 (05 제안 #N 후속).

**장기 계획 (05 제안)**:
- `/dashboard` (전사 숫자·KPI) vs `/cockpit` (누가 막혔나·미작성자) 의 **역할 명문화**
- `/cockpit` 모바일 대응 (현황파악 2차 F-04·F-06 연계)
- 현재 제안 #05 에서는 "유지" 로만 결정.
