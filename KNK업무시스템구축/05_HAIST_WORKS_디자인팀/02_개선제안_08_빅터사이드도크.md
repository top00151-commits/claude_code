## 0. 기존 구조 vs 새 패턴

### 기존 (전체화면 모달)
- 우측 상단 "빅터에게 물어보기" 버튼 → **전체화면 모달 오픈**
- 모달이 **현재 화면을 완전히 가림**
- 답변 받고 링크 클릭 → 페이지 이동 → 빅터 창 **닫힘**
- 다시 쓰려면 다시 클릭 → 다시 질문

**문제점**: 쓰던 업무 맥락이 끊김. 반복 질문 시 매번 열고 닫고 번거로움.

### 제안 (사이드 도크 / 코파일럿 패턴)
- 우측 고정 세로 탭 "🤖 빅터 ⌘K" — **항상 보임**
- 클릭 → 패널이 **오른쪽에서 슬라이드 인** (420px 너비)
- 메인 영역은 **margin-right 로 밀림** (덮지 않음, 양립)
- 답변의 링크 클릭 → 메인 영역만 이동, **빅터 패널은 열린 채 유지**
- 답변 하단 "바로 실행" 초록 버튼 — 1클릭 액션 즉시 수행
- 닫기(화살표 or Esc) → 패널 접힘, 탭만 남음
- **localStorage 로 상태 유지** — 다른 페이지 이동해도 열림 상태 그대로

## 1. 핵심 설계

### 1-1. 레이아웃 (3영역)
```
┌─────────────────────────┬──────────────────┐
│                         │  🤖 HAIST 빅터    │ ← 어두운 헤더
│                         │  사내 AI 컨시어지 │
│                         │  [↻] [—]          │
│   메인 콘텐츠           ├──────────────────┤
│   (항상 살아있음)       │  이 화면에서      │ ← 맥락 칩
│                         │  자주 묻는 것     │
│                         │  [지연 공정]      │
│                         ├──────────────────┤
│                         │                  │
│                         │   👤 내 할 일?    │ ← 대화
│                         │                  │
│                         │      🤖 오늘…    │
│                         │      [바로 이동] │
│                         │      [바로 실행] │
│                         │                  │
│                         ├──────────────────┤
│                         │ [내 할일][매출]  │ ← 퀵 액션
│                         │ [입력창     →]   │ ← 입력
│                         │ Enter·Shift·Esc  │
│                         └──────────────────┘
                          (420px 너비)
```

### 1-2. 인터랙션 규칙
| 액션 | 결과 |
|---|---|
| 우측 탭 클릭 | 패널 슬라이드 인 · 메인 영역 margin-right 420px |
| `Ctrl+K` / `⌘K` | 토글 (열기·접기) |
| `Esc` | 접기 |
| 패널 헤더 `—` 버튼 | 접기 |
| 패널 `↻` 버튼 | 대화 초기화 |
| 메시지의 링크 클릭 | 페이지 이동, **패널은 열린 상태 유지** |
| 모바일 (<768px) | 풀스크린 오버레이로 전환 |
| 열림 상태 | localStorage 저장, 새 페이지 이동 후에도 복원 |

### 1-3. 답변 카드 3버튼 타입
- **primary (빨강)**: "XX 페이지로 이동 →" — 메인 네비게이션
- **do (초록)**: "바로 실행" — 즉시 액션 (발주·알림 전송·확인 처리 등)
- **secondary (회색)**: "관련 자료" — 보조 링크

**왜 중요**: "바로 옆에서 실행해볼 수 있게" (대표 지시 핵심). 빅터가 답만 주고 끝이 아니라, **답 옆에 실행 버튼** 이 있어야 함.

### 1-4. 맥락 인식 (Context Chips)
패널 헤더 아래에 **"이 화면에서 자주 묻는 것"** 3칩 노출. 현재 경로(`window.location.pathname`)에 따라 질문 목록이 바뀜:
- `/home` → "내 할 일 / 오늘 등록 / 팀 현황"
- `/progress` → "지연 공정 / 내 미완료 / 납기 임박"
- `/changes` → "미확인 변경 / 내 관련 / 긴급"
- `/admin` → "사용자 추가 / 권한 변경 / 데이터 내보내기"

## 2. 시각 차별화 포인트 (원칙서 v4 준수)

| 요소 | 디자인 |
|---|---|
| 헤더 | 깊은 그라디언트 `#1A0508 → #A5282C` + radial glow |
| AI 아바타 | 40px 원형 + 초록 온라인 점 (pulse 애니메이션) |
| 메시지 버블 | 사용자=그라디언트, AI=흰색 카드, 둘 다 둥근 모서리 (한 귀퉁이 4px) |
| 타이핑 인디케이터 | 3점 bounce 애니메이션 |
| 입력창 | 자동 높이(`textarea`) + 포커스 3px ring |
| 전송 버튼 | 그라디언트 + hover lift |
| 링크 칩 | primary=빨강 / do=초록 / secondary=회색 |
| 탭 (세로) | 그라디언트 세로 텍스트 + 아이콘 + 단축키 힌트 |

**유동 타이포**: 모든 텍스트 `--fs-fluid-*` 적용 — 4K 모니터·모바일 양쪽 최적.

## 3. 01 세션 구현 가이드

### 3-1. 백엔드 (변경 없음)
`app/victor.py` + `POST /api/victor/ask` 는 **그대로 재사용**. 응답 구조(type/title/text/links) 호환됨.

단, 링크 `style` 필드에 `"do"` 추가 허용 권장:
```python
# victor.py 의 _link() 함수 시그니처
def _link(label, href, style="primary"):  # "primary" | "secondary" | "do"
    return {"label": label, "href": href, "style": style}
```
"do" 스타일은 `onclick` 으로 AJAX 실행 (예: 확인 처리·알림 전송·Excel 다운로드).

### 3-2. 프런트 (base.html 교체)

1. 기존 `#victorBox` (line 31~38) · `#victorModal` (line 858~958) **제거**
2. 시안 `08_victor_side_dock.html` 의 **Dock 탭 + Dock 패널 HTML + CSS + JS** 를 `base.html` 말미(`</body>` 직전)에 이식
3. `body` 에 `dock-open` 클래스 토글 CSS 가 동작하도록 `.main-wrap` 혹은 `.main.has-sidebar` 에 `margin-right transition` 추가
4. 단축키 `Ctrl+K` 는 기존 `victorOpen()` 에 대체 — 기존 함수 제거 또는 `toggleDock()` 로 매핑
5. "이 화면에서 자주 묻는 것" 칩은 `{{ request.url.path }}` Jinja 로 서버에서 렌더 또는 JS 라우트 매칭

### 3-3. localStorage 키
- `haist_victor_dock_open` — 도크 열림/닫힘 (0/1)
- `haist_victor_recent` — 최근 질문 10개 (후속 기능)

### 3-4. 단축키 확장 제안 (선택)
- `Ctrl+K` 토글
- `Esc` 접기
- `Shift+Enter` 줄바꿈
- `Ctrl+↑` 최근 질문 불러오기

## 4. 접근성·성능

- `aria-label="AI 빅터 어시스턴트"` · `role="dialog"` (도크 전체)
- `prefers-reduced-motion: reduce` 미디어쿼리로 애니메이션 비활성화
- 모든 색 대비 WCAG AA 이상
- 타이핑 인디케이터는 스크린 리더에 `aria-live="polite"` 로 읽힘
- 도크 자체는 외부 리소스 0 — Lighthouse 성능 영향 미미

---

## 5. 리스크·제약

- 기존 모달 사용자가 **"왜 팝업이 안 뜨지?"** 재학습 필요. 1회성.
- 1280px 미만 화면에서는 메인 영역이 **860px** 가 됨 — 테이블 스크롤 가능성. 모바일(<768px)은 풀스크린 폴백.
- 도크가 항시 보이면 시각 산만 가능성 → **탭만 노출**, 사용자가 의도적으로 열었을 때만 확장.
- "바로 실행" (do) 스타일은 실제 액션 이전에 **확인 토스트** 권장 (실수 방지).

## 6. 예상 효과

- 빅터 사용 빈도 **3~5배 증가** 예상 — 열고 닫는 비용 제거
- 페이지 간 이동 중에도 빅터와 대화 유지 → "기억하는 비서" 느낌
- "바로 실행" 버튼 → 빅터가 단순 안내를 넘어 **자동화 실행기** 역할로 진화 (Phase 2 준비)
