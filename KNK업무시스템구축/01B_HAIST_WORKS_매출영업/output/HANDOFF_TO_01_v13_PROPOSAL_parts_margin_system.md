## 1. 🎯 비즈니스 배경

### 현재 KNK 부품 수출 매출 구조
- **부품(PARTS) 형태 매출**: 자동화 / 검사기 카테고리에서 별도 부품 단위로 매출 발생
- **수출 대상**: 대부분 **베트남 법인 (KNK VINA)** 으로 수출
- **마진 구조**: 본사가 자재 매입가에 **8~15% 마진 추가** → 베트남 수출 인보이스
- **본사 이익**: 수출 매출 − 매입 단가 = 본사 마진

### 현재 시스템 한계 (대표 정확한 지적)
1. **자재구매 시스템 (part_prices)** 에 매입 단가 보관 — 격리됨
2. **영업 수주 라인 (so_units PARTS)** 에 수출 단가 표시 — 격리됨
3. **두 시스템 연결 없음** → 본사 이익률 한눈에 못 봄
4. PARTS 28컬럼은 **인보이스 작성 사양** (HS/DUTY/VAT/관세/USD/PALLET 등) — 시각 복잡, 더 추가 곤란

### 대표 요구사항
1. 매입 단가 ↔ 수출 단가 **자동 연결**
2. **마진 % / 본사 이익** 한눈에 표시
3. PARTS 28컬럼 유지 (인쇄/인보이스 요건)
4. 마진 관리는 **다른 위치**에서 해도 좋음 (28컬럼 부담 회피)

## 2. 🏗 빅터 권장 시스템 설계

### 2.1 데이터 모델 (DB 스키마 추가)

**신규 매핑 테이블** `so_unit_part_link`:
```sql
CREATE TABLE so_unit_part_link (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  so_unit_id   INTEGER NOT NULL,     -- so_units.id (PARTS 라인)
  part_id      INTEGER,              -- parts.id (자재구매)
  part_code    TEXT,                 -- 자재코드 (parts.code 미존재 케이스 대비)

  -- 시점 스냅샷 (매입가 변동 대비)
  snapshot_cost_unit   DECIMAL(15,2),    -- 라인 작성 시점의 매입 단가
  snapshot_cost_total  DECIMAL(15,2),    -- 매입 단가 × 수량
  snapshot_currency    TEXT DEFAULT 'KRW',
  snapshot_date        DATE,

  -- 마진 계산 (자동 또는 수동)
  margin_pct    DECIMAL(5,2),            -- 마진율 % (8.0 ~ 15.0)
  override_note TEXT,                    -- 수동 조정 사유 (협의가 등)

  created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME DEFAULT CURRENT_TIMESTAMP,

  FOREIGN KEY (so_unit_id) REFERENCES so_units(id) ON DELETE CASCADE,
  FOREIGN KEY (part_id) REFERENCES parts(id) ON DELETE SET NULL,
  UNIQUE (so_unit_id)
);
CREATE INDEX idx_solink_part ON so_unit_part_link(part_id);
CREATE INDEX idx_solink_code ON so_unit_part_link(part_code);
```

### 2.2 매핑 로직 (자동 연결)

**우선순위**:
1. 사용자가 SO PARTS 라인 작성 시 **자재코드 입력** (또는 자재구매 부품 검색 dropdown)
2. 시스템이 part_prices 최신 단가 자동 조회
3. snapshot으로 저장 (이후 매입가 변동되어도 이 라인의 마진 계산은 시점 단가 기준)
4. 마진율 자동 계산: `(수출가 - 매입가) / 매입가 × 100`
5. 수동 override 가능 (협의가, 환차 등 특수 케이스)
