# 공식 경로 (6가지)

<a id="a-서비스-api-오피스-토큰"></a>
## A. 서비스 API (오피스 토큰)

**출처**: developers.hiworks.com/support/article/3, /docs, /about

### A.1 개요

| 항목 | 내용 |
|---|---|
| 프로토콜 | REST API (HTTP) |
| 인증 | **오피스 토큰** (서비스별 발급) |
| 호출 | URI + 파라미터 + 헤더 |
| 관리 위치 | `오피스 홈 → 오피스 관리 → 환경 설정 → API 관리` |
| 조건 | **하이웍스 오피스 정보가 있는 회원만** |

### A.2 제공 API 5종 (오피스 토큰 기반)

1. **전자결재 API** — 외부에서 기안 POST → 하이웍스 결재 생성
2. **메신저 알림 API** — 하이웍스 메신저로 푸시 발송
3. **공용 문자 API (SMS)** — 단문·장문·포토문자
4. **카카오 알림톡 API** — 정보성 메시지 (기업 → 고객)
5. **세금계산서 API** — 전자세금계산서 발행 + 국세청 자동 전송
6. **인사관리 API** — 조직도·근태·직원정보 조회

### A.3 토큰 발급 절차 (관리자 작업)

```
[오피스 홈]
   ↓ 로그인 (하이웍스 관리자 계정)
[오피스 관리]
   ↓
[환경 설정]
   ↓
[API 관리]
   ↓ 사용할 API별로 토큰 생성
[토큰 확보]
   ↓ HAIST_WORKS의 .env에 저장
   HIWORKS_APPROVAL_TOKEN=...
   HIWORKS_MESSENGER_TOKEN=...
   HIWORKS_HR_TOKEN=...
```

### A.4 인사관리 API 상세 (팩트)

**확인된 기능**:
- 조직도 조회 (`조직 및 조직원 정보`)
- 조직원별 근태 현황
- 파트너사 솔루션에서 이름·이메일 등 조회
- 파트너사 솔루션 출입 정보 ↔ 하이웍스 근태 양방향 연동 가능

### A.5 전자결재 API 상세 (팩트·사례 기반)

**실사례 패턴 (에스원·아이퀘스트·영림원)**:
- 외부 시스템 "하이웍스 기안하기" 버튼 클릭
- POST로 하이웍스에 기안 요청
- **기안자 자동 인식** + 본문 자동 작성
- 하이웍스 결재 페이지 자동 열림 (사용자는 확인·제출만)

**필수 파라미터 (추정, Postman 확인 필요)**:
- doc_type (결재 양식)
- title
- content (본문)
- attachments
- approval_line (결재선)

<a id="b-계정-연동-api-oauth-20"></a>
## B. 계정 연동 API (OAuth 2.0)

**출처**: developers.hiworks.com/support/article/1, /support/article/3

### B.1 개요

| 항목 | 내용 |
|---|---|
| 인증 | **OAuth 2.0 Authorization Code Flow** |
| 역할 | 하이웍스 = **Authorization Server** (개인정보 저장, Token·AuthCode 생성) |
| 관리 위치 | 개발자센터 (`developers.hiworks.com`) → 애플리케이션 등록 |
| **사전 검수·승인 필수** | 승인 전 Production 사용 불가 |

### B.2 제공 API 2종

1. **내 정보 조회** — 로그인한 사용자 정보
2. **메일 API** — 자동 이메일 발송·팝업

### B.3 OAuth 2.0 플로우

```
[HAIST_WORKS 사용자]
   ↓ "하이웍스 로그인" 클릭
[브라우저]
   ↓ GET https://auth.hiworks.com/authorize?client_id={}&redirect_uri={}&scope={}
[하이웍스 로그인 페이지]
   ↓ 사용자 로그인 + 권한 승인
[하이웍스 Redirect]
   ↓ GET {redirect_uri}?code={authorization_code}
[HAIST_WORKS 서버]
   ↓ POST /token (code + client_id + client_secret)
[하이웍스]
   ↓ JSON { access_token, refresh_token, expires_in }
[HAIST_WORKS]
   ↓ 이후 API 호출: Authorization: Bearer {access_token}
```

### B.4 애플리케이션 등록 절차

1. 가비아 계정으로 개발자센터 로그인
2. **애플리케이션 등록** 메뉴
3. 정보 입력:
   - 서비스명
   - 리다이렉트 URI (`https://knk.example.com/auth/hiworks/callback`)
   - Scope 선택 (읽기 권한·쓰기 권한 등)
4. **사전 검수 요청**
5. 승인 대기 (기간 공식 명시 없음)
6. 승인 후 `Client ID`·`Client Secret` 발급

### B.5 Refresh Token 갱신

- Access Token 만료 시 Refresh Token으로 갱신
- 새 Access Token 발급받아 재사용

## C. 🔥 전자결재 웹훅 (아웃바운드 이벤트 수신)

**출처**: hiworks.com/cs/notice/33021 (2024.07.04 공지), customer.gabia.com/manual/hiworks/2490/7160

### C.1 🔥 핵심 (2024.07 신규 기능)

> **하이웍스가 외부 시스템으로 이벤트를 POST 발송.**
> **HAIST_WORKS가 수신 엔드포인트만 제공하면 기안·승인·수정 실시간 수신 가능.**

### C.2 등록 경로

```
[전자결재]
   → [관리자 설정]
   → [기본 설정]
   → [웹훅 등록]
```

### C.3 지원 이벤트 (확인된 팩트)

| 이벤트 | 발생 시점 |
|---|---|
| **문서 기안** | 사용자가 결재 기안 시 |
| **승인** | 결재자가 승인 시 |
| **내용 수정** | 결재 문서 수정 시 |

### C.4 수신 흐름 (추정 패턴)

```
[하이웍스 전자결재]
   ↓ 이벤트 발생
[등록된 웹훅 URL로 POST]
   ↓ https://haist-works.knk.com/webhooks/hiworks/approval
[HAIST_WORKS FastAPI 엔드포인트]
   @app.post("/webhooks/hiworks/approval")
   async def receive(payload: Dict):
       # payload 검증 (서명·토큰)
       # 이벤트 처리 (DB 기록·알림 트리거)
```

### C.5 KNK 활용 가능 시나리오 (기능적 사실만)

- 변경 Inform 결재 완료 감지 → HAIST_WORKS에서 `change.status = 'approved'` 자동 업데이트
- 증빙 결재 승인 → HAIST_WORKS 티켓 상태 자동 변경
- 기안 동시 알림 → HAIST_WORKS 내 사이드바 뱃지 실시간 갱신

### C.6 상세 페이로드·재시도 정책

- **공개 매뉴얼에 명시된 페이로드 구조는 본 리서치로 확인 불가** (가비아 고객센터 로그인 필요)
- 실제 구현 시 관리자 페이지에서 테스트 → 수신 페이로드를 로그로 기록 후 구조 파악 필요

## D. 메신저 웹훅 (인바운드 알림 발송)

**출처**: customer.gabia.com/manual/hiworks/2490/7160

### D.1 개요

> **외부 시스템에서 하이웍스 메신저 그룹대화방 또는 "나와의 대화방"으로 메시지 발송**

### D.2 특징 (팩트)

- 수신 대상: 그룹대화방 / 나와의 대화방
- 발송 포맷: 일정한 포맷의 메시지
- **계정 연동 불필요** — Webhook URL만 있으면 발송 가능

### D.3 사용 패턴 (추정)

```python
import httpx

webhook_url = "https://messenger.hiworks.com/webhook/xxx"
payload = {
    "text": "변경 Inform #123 승인 완료",
    "links": [{"url": "...", "text": "상세 보기"}]
}
await httpx.AsyncClient().post(webhook_url, json=payload)
```

### D.4 KNK 활용 시나리오 (기능적 사실만)

- HAIST_WORKS 변경 Inform 등록 → 관련 팀 그룹대화방 자동 알림
- 티켓 긴급도 높음 → 담당자 "나와의 대화방"으로 푸시
- 매일 아침 8시 일일 요약 → 대표·팀장 대화방

## E. SSO 연동 (iframe·조직 동기화)

**출처**: customer.gabia.com/manual/hiworks/3403/16181, /21903/21907

### E.1 2가지 방식

| 방식 | 설명 |
|---|---|
| **iframe 삽입** | 하이웍스 화면을 HAIST_WORKS 페이지에 iframe으로 포함 |
| **조직 API 동기화** | HAIST_WORKS의 조직 정보 ↔ 하이웍스 인사 계정 양방향 반영 |

- HAIST_WORKS에 "전자결재" 탭 → 클릭 시 하이웍스 결재 페이지 iframe 표시

- 사용자는 HAIST_WORKS를 떠나지 않고 하이웍스 기능 사용

### E.3 조직 API 방식

- HAIST_WORKS users 테이블 ↔ 하이웍스 인사 계정 sync
- 부서·직급 변경 양방향 반영

### E.4 구체 매뉴얼 상세

- 가비아 고객센터 로그인 필요
- 단계별 URL 형식·토큰 생성·암호화 방식 = **고객센터 매뉴얼에서 직접 확인**

## G. POP3/SMTP 메일 연결

**출처**: customer.gabia.com/manual/hiworks/118/1569

### G.1 용도

HAIST_WORKS (또는 외부 메일 클라이언트)에서 하이웍스 메일 송수신

### G.2 절차

1. **하이웍스 측 선행 설정**:
   - 로그인 → `메일 → 환경설정 → 기본 설정`
   - **POP3/SMTP "사용함"** 설정
2. 외부 클라이언트 (Gmail·Outlook·Python `smtplib`)에서 POP3/SMTP 정보 입력

### G.3 Python 예시 (SMTP 발송)

```python
import smtplib
from email.mime.text import MIMEText

smtp = smtplib.SMTP('smtp.hiworks.com', 587)  # 포트·도메인 확인 필요
smtp.starttls()
smtp.login('user@knk.com', 'app_password')
msg = MIMEText('변경 Inform 알림 본문')
msg['Subject'] = '[KNK] 변경 #123'
msg['From'] = 'user@knk.com'
msg['To'] = 'target@knk.com'
smtp.send_message(msg)
smtp.quit()
```

### G.4 주의

- 회사 정책에 따라 POP3/SMTP **차단 가능** (관리자 설정)
- **IMAP 공식 지원 여부 불명확** — POP3 기준 안전

<a id="h-이메일-파싱"></a>
## H. 이메일 파싱 (알림 메일 IMAP 수신)

**용도**: 하이웍스가 보내는 알림 메일(결재·메신저 등)을 HAIST_WORKS가 받아서 자동 처리

### H.1 기본 원리

1. 하이웍스 → 전자결재 알림 메일 자동 발송
2. HAIST_WORKS가 지정된 메일 계정 (예: `haist@knk.com`)으로 수신
3. **IMAP 접속** → 메일 본문·제목 파싱
4. 제목/본문에서 결재 번호·상태 추출 → DB에 기록

### H.2 Python IMAP 기본 예시

```python
import imaplib
import email

imap = imaplib.IMAP4_SSL('imap.gmail.com', 993)  # 또는 imap.hiworks.com (IMAP 지원 시)
imap.login('haist@knk.com', 'app_password')
imap.select('INBOX')

# 읽지 않은 하이웍스 알림만
_, data = imap.search(None, '(UNSEEN FROM "hiworks.com")')
for num in data[0].split():
    _, msg_data = imap.fetch(num, '(RFC822)')
    msg = email.message_from_bytes(msg_data[0][1])
    subject = msg['Subject']
    body = msg.get_payload(decode=True)
    # 파싱 로직 (결재 번호·상태 추출)
```

### H.3 장점

- **하이웍스 IMAP 지원 없어도 Gmail 등 중계 서버 경유 가능**

from playwright.async_api import async_playwright

async def hiworks_login_and_download():

### 공식 Postman 컬렉션

- **URL**: https://documenter.getpostman.com/view/6863253/S1TVWcri
- Postman 설치 후 Import → 각 엔드포인트 상세 확인
- 실제 구현 전 테스트 필수

### 공식 API 기초 Python 템플릿 (팩트 기반 구성)

```python
import httpx
import os

HIWORKS_BASE = "https://api.hiworks.co.kr"  # 실제 호스트 확인 필요
TOKEN = os.getenv("HIWORKS_APPROVAL_TOKEN")

async def create_approval(doc_type, title, content, approval_line):
    async with httpx.AsyncClient() as client:
        res = await client.post(
            f"{HIWORKS_BASE}/api/v1/approval/documents",  # 엔드포인트 Postman 확인 필요
            headers={"Authorization": f"Bearer {TOKEN}"},
            json={
                "doc_type": doc_type,
                "title": title,
                "content": content,
                "approval_line": approval_line
            }
        )
        res.raise_for_status()
        return res.json()
```

⚠️ **엔드포인트·파라미터 이름은 Postman 컬렉션 확인 필수**
