## 1. 이카운트 ERP

**출처**: korea-erp.com, techview.best, 제조/유통 사용자 리뷰

### 1.1 개요 (팩트)

| 항목 | 내용 |
|---|---|
| 가격 | **4만원/월 올인원 (모듈 전체 포함)** |
| 사용자 수 | **무제한** (5명이든 50명이든 동일 비용) |
| 주 사용층 | **77% 이상이 제조/유통 업종** |
| 평균 평점 | 약 4/5점 (다수 커뮤니티) |

### 1.2 🟢 장점

- 낮은 비용 대비 풍부한 기능
- 판매·구매·재고 모듈 충실
- 제조업·도매업 재고 관리 적합
- 소매·서비스업 매출 관리도 가능
- 전 직원 무제한 계정

### 1.3 🔴 약점

| # | 불만 | 출처 |
|---|---|---|
| 1 | **세무 마감 기능 부재** | 사용자 리뷰 |
| 2 | **UI 디자인 올드** | 사용자 리뷰 |
| 3 | 심화 제조 기능 부족 | 한국ERP.com |

### 1.4 KNK 참조 포인트 (사실만)

**흡수할 패턴**:
- 무제한 사용자 모델 — 부서 확장·베트남 법인 포함 시 비용 확장성
- 올인원 통합 구조 — 판매-구매-재고 연결

**참고 주의**:
- 이카운트는 표준 제조업 패턴 기반 — 장비 제조업(검사기·자동화)의 복잡 BOM·ECO 대응 여부 미확인
- KNK 자체 구축 시 "무제한 사용자" 모델을 재현 가능 (라이선스 제약 없음)

## 2. 영림원 K-System Ace

**출처**: korea-erp.com, swimparkslog (실전 가이드 #12), 이노소프트기술

| 평가 | 고객 지원에 공을 들임 |

### 2.2 🟢 장점

- **방문 지원** 및 심화 컨설팅 제공 (경쟁사 대비 강점)

### 2.3 🔴 치명적 약점 (사용자 실전 가이드 인용)

| # | 실패 요인 | 출처 |
|---|---|---|

| 6 | 10인 미만 부적합 | 이전 §3.4 리서치 (사내 전산팀 필수) |

### 2.4 성공 조건 (공식 언급)

- 사용자 중심 교육 전략
- **오픈 후 4주 집중지원 체계**
- 불만 해소 소통 채널 + 실행력 있는 TF

### 2.5 KNK 참조 포인트

**흡수할 패턴**:
- 부서별·사용자별 맞춤 대시보드 (설문 각 부서 요구와 일치)
- 오픈 후 4주 집중지원 개념 (정착 중요)
- 실행력 있는 TF — 세션1이 구현 후 부서 적응 돕는 구조

**피할 함정**:
- 요구사항 정의 부족 → 설문 12부서 응답 기반 이미 KNK는 유리
- 교육·소통 부재 → 부서별 가이드 필수 (설문 가공팀 요구)

## 6. MRPeasy (자재 관점 보강)

**이전 §3.1에서 다룬 내용 자재·재고 관점 추가 보강**

### 6.1 자재 특화 기능 (팩트)

- BOM 계층 트리 (다중 레벨)
- MRP 제안 알고리즘 (수요 → 발주 자동)
- 작업지시서(Work Order) 상태 흐름
- 재고 로트 추적 (lot tracking)
- Make-to-order + Make-to-stock 둘 다

### 6.2 한계 (2026 Flowlity 분석)

- **고급 수요 예측 부족**
- **멀티-echelon 인벤토리 최적화 부족** (여러 공장·창고 동시 최적화)
- 단일 공장 이상 확장 시 한계

### 6.3 KNK 참조

- KNK는 본사 + 베트남 법인 = 멀티 사이트 → MRPeasy 단일 공장 한계와 불일치
- 본사 장비 제조 + 베트남 후공정 흐름 = 자체 구축이 나을 수 있음 (사실 관찰)

**출처**: NSOFT, SINOSYS, 한솔PNS, 한국네트웍스, Cello Square, BoxHero

| **SINOSYS** | WMS 전문 | 중견 |

| **한국네트웍스** | 스마트 물류 WMS | 대기업 중심 |

| **SAP EWM** | Extended Warehouse Management | 대기업 |
| **Oracle WMS** | 글로벌 SCM | 대기업 |

### 7.2 WMS 도입 효과 (한국 성공 사례)

- **유시스 사례 (NSOFT)**: 바코드 스캔으로 입·출고·수량 관리 → 휴먼 에러 감소 + 신뢰도 높은 재고 데이터 + 자재 정보 신속 조회
- **극동자동화 MES 구축 사례 (NSOFT)**: 현장 중심 업무 프로세스 개선

### 7.3 WMS 공통 패턴

- **바코드·RFID 스캔** — 숙련도 무관 재고 위치 빠른 확인
- 창고 작업 오류 크게 감소
- 실시간 재고 데이터

### 7.4 KNK 참조 포인트 (사실만)

**흡수할 패턴**:
- 바코드/QR 스캔 UI (모바일 앱)
- 창고 위치 관리 (locator)
- 입출고 실시간 반영

**피할 함정**:
- 대형 패키지 도입은 다음 §10 사례처럼 실패 위험
- SME용 BoxHero 같은 얇은 SaaS는 복잡 BOM 장비 제조업에 부족 가능

## 9. MRP 공통 실패 요인

**출처**: mes-korea, SAP Korea, Waveon, ispark

### 9.1 성공 핵심 요소 (공식)

1. **BOM 정확도** — "**1%라도 틀리면 시스템 전체 결과 왜곡**"
2. **재고 데이터 실시간성**
3. **조달 리드타임 정확성** (공급사와 협의된 실데이터)
4. **부서 간 실시간 데이터 공유** (생산계획 변경 시 구매·물류 즉시)
5. **안전 재고 적정 설정**

### 9.2 KNK 설문과 일치

- 설문 구매팀 "수불 ERP·엑셀 이중 입력" = 실시간성 부족 문제
- 설문 전장팀 "기구 변경 통보 1~3일 대기" = 부서 간 실시간 공유 부족
- 설문 제조2 "변경 누락 사고" = BOM 정확도 문제

### 9.3 KNK 참조 포인트

- MRP 도입 실패는 대부분 **데이터 품질**·**부서 소통**에서 발생
- 세션1이 이미 구축한 HAIST_WORKS 변경 Inform + 진행률 대시보드가 **정확히 이 문제 해결**

## 10. ⭐ 한국 H기업 WMS 실패→자체개발 전환 사례

**출처**: 학술 논문 ("제조기업의 통합 WMS 구축 방법 및 효과에 관한 연구"), scienceon.kisti.re.kr

### 10.1 사례 요약 (원문 기반)

**H기업**: 중견 제조 그룹사

**실패 과정**:
- "통합적인 시스템 구축 필요한 그룹사" → **솔루션 패키지 도입 시도**
- **1년 허비** + 상당한 비용 손실
- 프로젝트 실패 확정

**문제 원인**:
- "깊이 있는 분석과 신중한 판단 부족"
- "Customization이 필요한 복잡한 시스템 환경" — 패키지가 맞지 않음

### 10.2 전환 결과

- 자체 개발로 방향 전환
- **"제조그룹에 맞는 기능들을 충족시키는 통합적인 WMS 개발"**
- 성공

### 10.3 ⭐ KNK와의 정확한 일치

| 항목 | H기업 | KNK |
|---|---|---|
| 규모 | 중견 그룹사 | 중견 (본사 75 + 베트남 60 = 135명) |
| 업종 | 제조업 | 제조업 (검사기·자동화 장비) |
| 초기 시도 | 솔루션 패키지 도입 | **회피** (처음부터 자체 구축) |
| 결과 | 1년 허비 후 자체 개발 | ✅ **H기업 교훈을 미리 적용** |
| 방향 | 자체 개발 WMS | HAIST_WORKS (FastAPI) |

### 10.4 KNK 참조 포인트 (사실 기반)

**관찰**:
- H기업은 **실패 후 자체개발**로 전환해 성공
- KNK는 **처음부터 자체구축** 방향 = H기업이 얻은 교훈을 선제 적용
- 즉 KNK 방향은 **학술 논문에서 검증된 성공 경로**

- 외주 비용 없음 (대표 + 개발혁신팀 최 상무)
- API 비용만 월 발생

- 2024~2026: **중소기업까지 확산 + 클라우드·AI 통합**

### 13.8 성공 요인 (동종업계 확정 조건)

**출처**: ERP 실태조사 논문, 영림원 K-System 실전 가이드

1. **최고경영자(CEO) 지원** — KNK는 대표이사 직접 주도 ✅
2. **숙련된 인력 지원 가능성** — KNK는 개발혁신팀 최보현 상무 ✅
3. **교육·협조체제 조성** — 설문 12부서 기반이면 확보 ✅
4. **예산 확보** — 자체 구축이라 외부 구축비 없음 ✅

### 13.9 KNK 참조 포인트 (사실 기반)
