# Part A. 하이웍스 연결

<a id="a1-하이웍스-api-구조-개요"></a>
## A.1 하이웍스 API 구조 개요

**출처**: https://developers.hiworks.com/about, /docs, /support/article/3

### A.1.1 API 2종 분류 (공식 분류)

**새로워진 하이웍스 API** (REST API 기반)은 2종으로 나뉨:

| 분류 | 용도 | 관리 위치 |
|---|---|---|
| **서비스 API** | 하이웍스 서비스 기능 사용 (결재·메신저·메일·SMS·알림톡·세금계산서·인사) | 오피스 관리 페이지 (`오피스 관리 > 환경설정 > API 관리`) |
| **계정 연동 API** | 외부 사이트에 하이웍스 화면 iframe 삽입(SSO) + 인트라넷 조직 정보 ↔ 하이웍스 인사 계정 동기화 | 개발자센터 (애플리케이션 등록) |

### A.1.2 세부 제공 API 8종

| # | API | 카테고리 | 주 용도 |
|---|---|---|---|
| 1 | 전자결재 | 서비스 | 외부에서 기안 트리거 |
| 2 | 메신저 알림 | 서비스 | 하이웍스 메신저 푸시 발송 |
| 3 | 공용 문자 (SMS) | 서비스 | 단문·장문·포토문자 |
| 4 | 카카오 알림톡 | 서비스 | 정보성 메시지 |
| 5 | 세금계산서 | 서비스 | 전자세금계산서 발행·국세청 전송 |
| 6 | 인사관리 | 서비스 | 조직도·근태·직원정보 **조회** |
| 7 | 내 정보 조회 | 계정 연동 (애플리케이션) | 사용자 정보 |
| 8 | 메일 | 계정 연동 (애플리케이션) | 자동 이메일 발송·팝업 |

### A.1.3 기술 스펙

| 항목 | 내용 |
|---|---|
| 프로토콜 | **REST API (HTTP)** |
| 언어 | 언어 독립 — 모든 HTTP 지원 언어 (Python/FastAPI 호환) |
| 인증 | **오피스 토큰** (서비스 API) 또는 **OAuth 2.0** (계정 연동 API) |
| 호출 방식 | URI·파라미터만으로 호출 가능 |
| 공식 문서 | Postman 컬렉션 (`https://documenter.getpostman.com/view/6863253/S1TVWcri`) |
| 테스트 도구 | Postman 권장 (공식) |
| 신청 | 개발자센터 `My 애플리케이션 관리` |
| 가비아 계정 필요 | ✅ (개발자센터 로그인용). 단, API 문서 조회는 비로그인 가능 |

<a id="a2-사전-준비--토큰-발급-절차"></a>
## A.2 사전 준비 — 토큰 발급 절차

**출처**: developers.hiworks.com/support/article/3, /about

### A.2.1 서비스 API 토큰 발급 (전자결재·메신저 알림·공용 문자)

> **조건**: 하이웍스 오피스 정보가 있는 회원만 가능

**절차**:
1. 하이웍스 오피스에 로그인
2. `오피스 홈 → 오피스 관리 → 환경 설정 → API 관리` 이동
3. 필요한 API별로 설정 및 토큰 생성
4. 토큰을 HAIST_WORKS에 안전하게 저장 (`.env`)

### A.2.2 계정 연동 API (내 정보 조회·메일) — OAuth 2.0

**절차**:
1. 하이웍스 개발자센터 → `애플리케이션 등록`
2. 애플리케이션 정보 입력 (서비스명·리다이렉트 URI 등)
3. **사전 검수 요청** → 승인 대기
4. 승인 후 Client ID/Secret 발급
5. OAuth 2.0 Authorization Code → Access Token 발급 흐름

### A.2.3 중요 팩트

> **"검수가 완료되어 승인이 완료된 애플리케이션만 하이웍스 API를 이용할 수 있습니다."**

- 승인 없이는 Production 사용 불가
- 검수 기간·기준은 공식 명시 없음 — 가비아 문의 필요

### A.2.4 테스트 흐름 (공식 권장)

1. **Postman 다운로드** (postman.com)
2. 하이웍스 제공 Postman 컬렉션 import
3. HTTP 메서드 (GET·POST) 선택
4. 파라미터·헤더 (Token) 입력
5. 요청 실행 → 응답 확인
6. 정상 응답 확인 후 실제 코드로 구현

<a id="a3-api-카테고리별-상세"></a>
## A.3 API 카테고리별 상세

> ⚠️ **상세 엔드포인트·파라미터는 공식 Postman 컬렉션 필수 확인**:
> https://documenter.getpostman.com/view/6863253/S1TVWcri

### A.3.1 인사관리 API — 근태·조직도 조회

**확인된 기능 (팩트)**:
- 조직도 조회 (`조직 및 조직원 정보`)
- 조직원별 근태 현황 확인
- 이름·이메일 등 조직원 정보 파트너사 솔루션에서 조회
- 파트너사 솔루션의 출입 정보 ↔ 하이웍스 근태 양방향 연동 가능

**사용 흐름 (추정 — Postman 확인 필요)**:
```
GET /인사관리 엔드포인트
Headers: Authorization: Bearer {OfficeToken}
Params: date 등
Response: JSON (조직·직원·근태)
```

### A.3.2 전자결재 API — 외부에서 기안 생성

**확인된 기능 (공식 사례 기반)**:
- 외부 시스템에서 "기안하기" 버튼 클릭 시 하이웍스로 POST
- **기안자 자동 인식** + 본문 자동 작성
- 에스원·아이퀘스트·영림원 실제 구현

**사용 흐름**:
```
[외부 시스템 "기안하기" 버튼]
       ↓ POST
[하이웍스 API]
  - doc_type (결재 양식)
  - title
  - content (본문)
  - attachments
  - approval_line (결재선)
       ↓
[하이웍스 결재 페이지 열림]
  - 기안자·본문 자동 채워짐
  - 사용자는 확인·수정·제출만
```

### A.3.3 메신저 알림 API — 푸시 발송

**확인된 기능**:
- 하이웍스 메신저로 일방향 푸시
- 외부 이벤트 → 하이웍스 메신저 자동 알림

### A.3.4 메일 API (계정 연동 API)

**확인된 기능 (공식 분류)**:
- 자동 이메일 발송
- 팝업 메일

**주의**: 이는 OAuth 2.0 인증 필요한 계정 연동 API 범주

### A.3.5 SMS / 알림톡 API

**확인된 기능**:
- **공용 문자 (SMS)** — 단문·장문·포토문자 발송
- **카카오 알림톡** — 정보성 메시지 (기업→고객)

### A.3.6 세금계산서 API

**확인된 기능**:
- 전자세금계산서 발행
- **국세청 자동 전송**

## A.4 SSO 연동 (외부 사이트)

**출처**: customer.gabia.com/manual/hiworks/3403/16181 (가비아 공식 매뉴얼)

### A.4.1 SSO 연동 방식 (팩트)

**공식 매뉴얼 존재**: "하이웍스에서 외부 사이트로 SSO 연동하기"

**2가지 주 방식**:
1. **iframe 삽입** — 하이웍스 화면을 외부 사이트(인트라넷)에 iframe으로 포함
2. **조직 API 동기화** — 외부 인트라넷의 조직 정보 → 하이웍스 인사 계정으로 반영

### A.4.2 구체 매뉴얼 내용

가비아 고객센터 매뉴얼에 상세 절차가 있으나, 본 리서치는 매뉴얼 URL 접근 시 **UI 메타 정보만 반환** (실제 단계별 가이드 텍스트 추출 실패).

**확인 방법** (세션1이 직접 수행):
1. 가비아 고객센터 (customer.gabia.com) 로그인
2. 매뉴얼 검색: "SSO 연동"
3. 상세 페이지 접근 → URL 형식·토큰 생성·암호화 방식 확인

### A.4.3 에스원·아이퀘스트 사례에서 확인된 SSO 유사 UX

- 외부 시스템에서 "하이웍스 기안하기" 버튼 클릭
- 팝업/새 탭으로 **하이웍스 결재 페이지 열림**
- 이미 로그인된 세션이면 바로 진입 (단일 로그인)

## A.5 메일 외부 연결 옵션

**출처**: customer.gabia.com/manual/hiworks/118/1569, /128/12800, /113/17680

### A.5.1 옵션 A — POP3/SMTP 외부 클라이언트 (예: Gmail)

**절차**:
1. 하이웍스 로그인 → `메일 → 환경설정 → 기본 설정`
2. **POP3/SMTP "사용함"** 설정 (먼저 선행)
3. Gmail/Outlook 등에서 "다른 계정 메일 가져오기" → POP3 입력

### A.5.2 옵션 B — 메일 포워딩 (단방향)

- 하이웍스 수신 메일을 Gmail/외부로 자동 전달
- 가비아 공식 매뉴얼: `/manual/hiworks/2047/2067`

### A.5.3 옵션 C — 메일 이전 프로그램 (가비아 공식)

- 하이웍스 → 다른 서비스로 데이터 이전용
- `/manual/hiworks/113/17680`

### A.5.4 옵션 D — Google Workspace 이관

- Google Workspace 공식 데이터 이전 서비스
- IMAP 기반 웹메일 → Workspace
- MX 레코드 변경으로 도메인 전체 이전

### A.5.5 주의사항 (팩트)

| 항목 | 내용 |
|---|---|
| 회사 정책 제한 | POP3/SMTP 사용은 관리자가 막을 수 있음 |
| IMAP 공식 지원 여부 | **공식 매뉴얼에서 명확히 확인되지 않음** (POP3만 확정적) |
| Gmail 쪽 | IMAP·POP3·SMTP 모두 지원 |
| 공개 사례 | 한국 기업의 하이웍스↔Gmail 연결 구체 케이스 공개 자료 희박 |

## B.2 공식 API·Bot·Webhook 구조

**출처**: docs.kakaoi.ai/kakao_work/, kakaowork.gitbook.io, devtalk.kakao.com

### B.2.1 API 구조 개요

| 항목 | 내용 |
|---|---|
| **공식 문서** | https://docs.kakaoi.ai/kakao_work/ |
| **API 스타일** | **HTTP RPC (Remote Procedure Call)** — 순수 REST 아님 |
| **URL 형식** | `https://api.kakaowork.com/v1/{API 종류}.{API 이름}` |
| **유지·운영** | DK Techin Corp (카카오 자회사) |

### B.2.2 제공 Web API 카테고리

| API | 용도 |
|---|---|
| users | 사용자 조회·관리 |
| conversations | 채팅방 생성·관리 |
| messages | 메시지 전송 |
| reactive | 상호작용 (버튼 클릭 등) |
| organizations | 조직 데이터 |
| spaces | 워크스페이스 정보 |
| bots | Bot 설정 |

### B.2.3 Webhook (알림 일방향) vs Interactive Bot (대화형)

| 유형 | 특성 | 용도 |
|---|---|---|
| **Webhook형 Bot** | 외부 이벤트 → 지정된 채팅방에 **일방향 알림** | 단순 알림 (빌드 성공·주문 접수 등) |
| **알림형 Bot** | 일반 텍스트 또는 Block Kit 말풍선으로 알림. 상호작용 없음 | 시스템 이벤트 알림 |
| **Interactive Bot** | 사용자와 **양방향 대화**. 메시지·모달·버튼으로 응답 | 명령 처리·설문·조회 |

### B.2.4 Webhook 발급 방식

- 봇 등록 시 고유 **Webhook URL** 자동 발급
- 외부 시스템에서 해당 URL로 POST → 지정 채팅방 자동 포스팅
- 계정 연동 불필요 (사용자 동의 절차 없음)

### B.2.5 REST 엔드포인트 (일부 공개)

- `POST https://api.kakaowork.com/v1/users.info` — 사용자 정보 조회
- `POST https://api.kakaowork.com/v1/conversations.open` — 채팅방 생성
- `POST https://api.kakaowork.com/v1/messages.send` — 메시지 전송

(완전 목록은 공식 Web API 레퍼런스 참조)

### B.2.6 SSO 지원

- **Okta SSO 공식 지원** 문서 제공

### B.3.1 Bot 개발 라이프 사이클 (공식 5단계)

```
1. Bot 기획      (알림형/상호작용형 결정)
     ↓
2. Bot 등록 요청  (관리자 서비스에서)
     ↓

     ↓

     ↓
```

- 발주 승인 요청 → 구매팀 채팅방

**구현 방식**: `curl -X POST {webhook_url} -H "..." -d '{"text": "..."}'`

- 일일업무 등록 모달 팝업
- 설문 응답 수집 (Block Kit 모달)

### B.7.3 조직 정보 동기화

**기능적으로 가능**:
- 카카오워크 조직도 API로 사용자 목록 조회

- 부서 변경 시 자동 반영

### B.7.4 SSO (선택)

**기능적으로 가능**:

## 🔍 세션1 참조 포인트 (흡수할 패턴 / 피할 함정)

### 하이웍스 — 흡수할 패턴
- **"하이웍스 기안하기" 버튼 UX** (에스원·아이퀘스트) — 한 클릭 외부 → 결재
- **오피스 토큰 기반 서비스 API** — Postman으로 먼저 검증 후 구현
- **근태·조직도 조회만 활용** — 결재 본체는 하이웍스에 유지 (정책)

### 하이웍스 — 피할 함정
- 계정 연동 API는 **검수·승인 필요** (일정 여유)
- **IMAP 공식 지원 불명확** (POP3 기준 안전)
- 한국어 문서만 → 영문 팀 대응 한계

### 카카오워크 — 흡수할 패턴
- **Webhook URL만 발급하면 즉시 알림** — 가장 빠른 통합
- **Block Kit Builder**로 Rich 메시지 디자인
- **무료 플랜으로도 Webhook·기본 기능 가능**

### 카카오워크 — 피할 함정
- **서버 오류 빈발** — 재시도 큐·fallback 필수
- **메시지 삭제 불가** — 사전 검증 필수
- **모기업도 이탈(슬랙으로)** — 장기 유지 리스크 인지
- **Premium 이상부터 커스텀 봇 무제한** — 플랜 비용 고려
- **HTTP RPC 스타일**이라 순수 REST 클라이언트에서는 URL 패턴 주의
