# 사내 서버 테스트 배포 가이드 (Windows + LAN/VPN)

KNK 메신저를 사내 서버 PC에 올려 한국 사무실 직원 + VPN 접속 베트남 법인 직원이 함께 사용하는 단계별 안내.

대상: 정식 클라우드 운영 전 1~2주 베타 테스트.

---

## 0. 사전 준비 (사내 서버 PC에서 1회만)

| 항목 | 확인 방법 |
|---|---|
| Python 3.11+ 설치 | CMD에서 `py --version` → `Python 3.13.x` 같은 결과 |
| 회사 서버 LAN/VPN IP | CMD에서 `ipconfig` → IPv4 주소 (예: `192.168.10.50`) |
| 5050 포트 미사용 | CMD에서 `netstat -ano \| findstr :5050` → 결과 없으면 OK |

---

## 1. 폴더 복사

`10_KNK_Messenger` 폴더 전체를 사내 서버 PC의 원하는 위치(예: `C:\KNK_Messenger\`)로 복사.

> 주의: `data\` 폴더 안에 `messenger.db` 와 `vapid_private.pem` 파일은 **그대로 가져갈지** 결정. 그대로 가져가면 빅터가 만든 4개 시드 아이템·기술영업팀 채널이 같이 옴. 새로 시작하려면 `data\` 폴더만 지우고 복사.

## 2. Python 의존성 설치 (서버 PC에서 1회)

```cmd
cd C:\KNK_Messenger
py -m pip install -r requirements.txt
py -m pip install pywebpush Pillow openpyxl
```

## 3. 방화벽 인바운드 5050 허용 (관리자 권한 CMD)

```cmd
netsh advfirewall firewall add rule name="KNK Messenger" dir=in action=allow protocol=TCP localport=5050
```

해제 시:
```cmd
netsh advfirewall firewall delete rule name="KNK Messenger"
```

## 4. 서버 시작

`START.bat` 더블클릭. 검은 창에 다음과 같이 출력됨:

```
============================================
  KNK Messenger - 서버 시작
============================================
   PC(이 서버):     http://localhost:5050
   직원 접속 URL:   http://192.168.10.50:5050
   직원 접속 URL:   http://10.8.0.5:5050        (← VPN IP)
   포트:            5050  (방화벽 인바운드 허용 필요)
============================================
```

위 "직원 접속 URL" 을 직원에게 공지하시면 됩니다.

## 5. 직원 안내 (카톡·메일로 공지)

```
KNK 사내 메신저 베타 시범 — 1주 운영

URL: http://[위 IP]:5050
계정 ID: [영문 이니셜]   (예: kjr, hong, lhr ...)
비밀번호: knk1234        (첫 로그인 후 변경 권장)

기능: 카톡과 비슷하게 사용. 추가로
  - 아이템(고객사×모델)별 자동 정리
  - 메시지 -> 요청 등록(📌)으로 마감일 추적
  - 사진/파일 갤러리 + 한글 검색
  - 대시보드(/dashboard) 한눈에 보기

휴대폰: 같은 와이파이 또는 VPN 접속 후 같은 URL.
        브라우저 메뉴에서 "홈 화면에 추가" 권장.

문의: 김정락 대표
```

## 6. 24시간 운영 옵션

### A. 서버 PC를 항상 켜둠 (가장 단순)
- 윈도우 자동 잠금 끄기: 설정 → 시스템 → 전원 → 화면 끄기 "사용 안 함"
- 윈도우 자동 업데이트로 재부팅 방지: 활성 사용 시간 24시간 설정 권장

### B. 서비스/작업스케줄러 등록 (재부팅 시 자동 실행)

```powershell
# PowerShell 관리자 권한
$action = New-ScheduledTaskAction -Execute "C:\KNK_Messenger\START.bat"
$trigger = New-ScheduledTaskTrigger -AtStartup
$principal = New-ScheduledTaskPrincipal -UserId "SYSTEM" -RunLevel Highest
$settings = New-ScheduledTaskSettingsSet -StartWhenAvailable
Register-ScheduledTask -TaskName "KNK_Messenger" -Action $action -Trigger $trigger -Principal $principal -Settings $settings
```

### C. NSSM으로 Windows 서비스화 (권장)

```cmd
:: nssm.exe 다운로드 후 (https://nssm.cc/)
nssm install KNK_Messenger "C:\Users\xxx\AppData\Local\Python\pythoncore-3.14-64\python.exe" "C:\KNK_Messenger\app.py"
nssm set KNK_Messenger AppDirectory "C:\KNK_Messenger"
nssm set KNK_Messenger DisplayName "KNK 메신저"
nssm set KNK_Messenger Start SERVICE_AUTO_START
nssm start KNK_Messenger
```

이러면 PC 재부팅해도 자동 실행됨.

## 7. 자동 백업 (작업스케줄러)

이미 `backup.py` 와 `backup.bat` 있음. 매일 새벽 3시 자동 실행:

```powershell
$action = New-ScheduledTaskAction -Execute "C:\KNK_Messenger\backup.bat"
$trigger = New-ScheduledTaskTrigger -Daily -At 3am
Register-ScheduledTask -TaskName "KNK_Messenger_Backup" -Action $action -Trigger $trigger -RunLevel Highest
```

## 8. VPN 환경 체크리스트

| 항목 | 확인 |
|---|---|
| VPN 클라이언트(베트남 직원 PC)에서 사내 서버 IP 핑 | `ping 192.168.10.50` → 응답 |
| VPN 클라이언트에서 5050 포트 접근 | 브라우저로 URL 접속 → 로그인 화면 뜸 |
| Web Push 알림 | 안 됨 (HTTPS 미적용 상태). 정식 운영 시 클라우드+SSL로 전환 시 작동 |

## 9. 운영 중 주요 점검

| 무엇을 | 어디서 | 빈도 |
|---|---|---|
| 서버 살아있는지 | 검은 창 또는 서비스 상태 | 매일 1회 |
| DB 크기 | `data\messenger.db` 파일 크기 | 주 1회 |
| 업로드 폴더 크기 | `data\uploads\` | 월 1회 (커지면 정리) |
| 백업 잘 되는지 | `backups\` 폴더의 최신 파일 날짜 | 주 1회 |

## 알려진 제약 (사내 LAN+VPN 운영 시)

| 제약 | 영향 | 대응 |
|---|---|---|
| HTTPS 없음 | 비밀번호 평문, Web Push 작동 안 함 | 정식 운영 시 SSL로 해결 |
| 단일 PC 의존 | PC 다운 시 모두 멈춤 | NSSM 서비스 + 자동 재시작 |
| SQLite 동시성 | 50+ 동시 접속 시 락 발생 가능 | 베타 (10~30명) 까진 무리 없음 |
| 베트남 응답 속도 | VPN 거치므로 ~50-150ms | 채팅엔 무리 없음. 정식 운영 시 SG 클라우드로 단축 |

---

문제 발생 시:
1. 검은 창의 마지막 줄 에러 메시지 캡처
2. 서버 재시작 → 같은 증상이면 빅터에게 알림
