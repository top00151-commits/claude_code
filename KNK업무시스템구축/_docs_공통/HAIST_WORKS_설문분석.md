#### C. 업무 흐름 5단계

**[검사기 부문] 자재 발주 및 입고처리**
1. **시작**: 영업팀 제작요청서 접수 + 제조팀 카카오톡 자재요청 수신 + 설계팀에서 가공팀&가공업체로 보내는 메일의 도면경로 확인 후 도면 확인
2. **첫행동**: 검사기 반출리스트_2026.xlsx + 도면 열어 구매품 리스트·핀블럭 POGO PIN 수량 확인 → 리스트에 기록 → ERP 재고확인 → 부족품목 협력사 이메일 발주 → 수불부·ERP 업데이트
3. **막히는 순간**: 도면내 품명(형번) 오류 / 협력사 납기확인 회신 대기 (평균 2~8시간) / 특정부품 납기지연으로 제조팀 일정 충돌
4. **완료기준**: 협력사 이메일 발주완료 + 2026년 수불부_검사기파트.xlsx & ERP 업데이트 + 제조팀에 카카오톡·유선 납기일 공유
5. **다음전달**: 제조팀에 입고예정일 공유 → 입고품 현품 검수·거래명세서 확인 서명 → 수불부·ERP 업데이트 → 관리팀에 월별 구매실적 공유

**[자동화 설비] 자재 발주**
1. **시작**: 영업팀 제작요청서 + 설계팀·전장팀 BOM 접수
2. **첫행동**: BOM 취합 → 협력사 견적요청(단가·납기) → 사내 재고 확인·단가 협의 → 발주서 이메일 발송 → 수불부·ERP 업데이트
3. **막히는 순간**: 도면내 형번 오류 / 협력사 납기회신 대기 (0.5~2일) / 장납기 부품 지연 → 영업·제조 충돌
4. **완료기준**: 발주완료 + 수불부·ERP 업데이트 + 제조·영업·KNKVINA 납기 공유
5. **다음전달**: 입고 검수 → ERP 업데이트 → 관리팀 월별 구매실적 공유

**지연 원인**:
- 원인1: 협력사 납기지연 — 월 1~2회, 평균 2~7일
- 원인2: 재고수량 불일치 확인작업 — 월 2회, 평균 1~2시간
- 원인3: 글로벌 이슈로 인한 납기지연

**부재시 대체**: 있음 / **인수인계문서**: 기록관리 파일 있음

#### D. 팀간 연계

- 제조기술1·2팀 ← 자재 요청 및 불출 / 기안·카톡·유선 / 일 20~40건
- 설계팀·전장팀 → BOM 수신 / 이메일·카톡 / 프로젝트별
- 검사기팀 ← PCB 및 필요자재 구매 요청 / 하이웍스 / 주 4~5회
- 가공팀 ← POGO PIN 요청 / 카톡·유선 / 주 4~5회
- 관리팀 ← 매월 5일 정기결제 대금 확인 요청 / 기안 합의 / 월 1회
- 관리팀 → 월별 구매실적(구매합계표) 작성 요청 / 서버·이메일 / 월 1회
- 기술영업팀 → 원가확인 / 이메일·카톡 / 견적건마다
- 기술영업팀 → 구매 요청 / 이메일·카톡 / 수주(예정분 포함) 건
- 기술영업팀 → 매출마감시 NET 재료비 입력 / 월 1회
- 기술영업팀 → 프로젝트별 수출자재 입출고 / 카톡·유선 / 주 1~2회
- **베트남법인 → 수입자재 수출서류 처리** / 법인 발주물량 별도관리 / 월 2~3회 / 불편한점 없음
- 베트남법인 → 프로젝트별 누락·추가품목 요청 / 카톡 / 발생시

#### E. 개선 우선순위

**없애고 싶은 비효율 TOP3**:
1. **수불 관리** — ERP 및 수불부(엑셀) 중복 입력으로 업무 LOSS 발생
2. **발주 주체 이원화** — 가공품(설계팀·구매팀), PCB(검사기팀·구매팀) 발주 주체 혼선으로 **발주/입고/마감 누락** 발생
   - NET재료비 입력 부정확 (매출은 계약금 30%로 잡지만 실제 금액 불명, BOM 입력 전 재료비 입력)
   - BOM 작성시 **정확한 형번 기입** 필요 (부정확시 견적요청·발주시 시간낭비·오발주)
3. 문서 정리 — 사용하지 않는 문서 정리

**매일 아침 확인 희망**:
1. 진행중인 프로젝트의 부서별 진행상황 및 이슈사항
2. 회사일정, 휴가중인 인원 현황, 점심메뉴

**시스템 도입 의견**: 찬성. **진행예정 프로젝트 현황, 진행중 프로젝트의 부서별 업무진행현황, 부서간 요청사항**이 공유되었으면 함.

2. 첫행동: 요구사항 분석 → 개발계획 수립 → 팀원 업무배분 → 개발 착수
3. **막히는 순간**: **타부서 일정이 초기 계획과 맞지 않음** (예: 기구·PCB)
4. 완료기준: 내부 QC·고객사 검수 완료 + 인도
5. 다음전달: 기술영업팀에 진행현황 공유 (구두·납품 요청)

**지연 원인**:
- 원인1: 초기 계획 일정과 다르게 **기구·전장·PCB 일정 문제 발생**
- 원인2: **유지보수 업무 최우선 처리**
- 원인3: **대부분 신규 개발 건으로 기술 검토 부족 → 재개발 상황 발생**

#### D. 팀간 연계

- 기술영업팀·고객사 ← 유지보수 요청 / 구두·이메일
- 기술영업팀 ← 계약확정·착수요청 / 구두·이메일
- 검사기팀 ↔ 기술협업 / 구두·이메일

#### E. 개선 우선순위

**1순위**: **모든 프로젝트의 공정별(구매·기구·전장·…) 진행 상황을 실시간으로 또는 예측 가능한 수준으로 알고 싶다**

**매일 아침 확인 희망**: (동일 — 공정별 진행 상황)

## HAIST WORKS 기능 우선순위

### 🔴 1순위 (2주 내) — 10개 팀 공통 요구

**① 프로젝트 공정 진행률 대시보드**
- 관리코드 × 공정 (수주 → 컨셉 → 설계 → 전장 → SW → 가공 → 구매 → 조립 → 검수 → 출하 → Set-up → KNKVN 이관)
- 단계별 상태 (예정·진행·완료·지연) + 담당자 + 예정일
- 지연 알림 (기준일 초과시)

**② 변경사항 Inform 시스템**
- 기구 설계 변경 → 자동 알림 (전장·SW·제조·가공·구매)
- Concept 변경 → 하류 부서 자동 공지
- **변경 전/후 part 구분 표시** (제조2 실제 사고 예방)
- 변경 이력 DB (언제·누가·무엇을·왜)

### 🟠 2순위 (4~6주)

**③ 요청 티켓 시스템** (카톡 보조)
- 자재요청·긴급가공·MODIFY·검수요청·AS만 티켓화
- 요청자·수신자·상태·소요시간 자동 기록
- 카톡은 그대로 두고, 중요 요청은 링크 클릭으로 티켓 생성

**④ 개인 PC 파일 → 공용 저장소 이전**
- 제조2 (출하성적서·체크리스트·특이사항)
- 품질 (출하이슈·CHECK SHEET)
- 영업 (수주현황·일정파일)
- 제조1 (작업일정표)
- 자동 sync 또는 공용 서버 강제

**⑤ 전사 근태 조회**
- 오늘 휴가·재택·출장 실시간 확인
- 하이웍스 근태 → HAIST WORKS 자동 연동

### 🟡 3순위 (로드맵)

**⑥ BOM 통합 마스터** — 설계 + 전장 + 수불 단일 출처
**⑦ 이슈·AS 이력 DB** — 품질·검사기·SW 공용, 고객사·모델·증상·처리·재발방지 검색
**⑧ 도면 버전 관리** — 최신 도면만 노출, 구버전 자동 아카이브, 30일 link 만료 해결
**⑨ 표준 Library 저장소** — SW팀 비젼·I/O·축제어 코드 재사용
**⑩ 증빙 제출 포털** — 관리팀 법인카드·지출·출장 증빙 업로드 → ERP 연동
**⑪ KNKVN 통합 뷰** — 본사+베트남 수주·발주·Set Up 현황 (영업·구매·설계·SW 공통 요구)
**⑫ 4·5단계 입고·재고** — 기존 로드맵, 티켓 시스템과 자연 결합

### ⚠️ 반드시 지킬 설계 원칙

1. **이중 입력 금지** — 기존 ERP·엑셀에 있는 데이터는 **읽기만**, 쓰기는 시스템에서
2. **기존 도구 연동 우선** — Git·Altium·SolidWorks·WEHAGO는 대체하지 말고 **메타데이터만 수집**
3. **카톡 대체 아닌 보조** — 중요 요청만 티켓화
4. **읽기 자동, 쓰기 최소** — 대시보드 자동 집계 + 상태 변경만 1~2 클릭
