## 4. ⚠️ 직원들이 직접 제안한 설계 원칙 (반드시 지킬 것)

1. **"일을 위한 일" 최소화** — SW팀 (이한중)
2. **데이터 구조 통합·업무 기준 강제화 후 반복 업무만 자동화 → 차츰 확대** — 관리팀 (박지은)
3. **현업 부담 최소화 방향 설계** — 전장팀 (김형렬)
4. **시스템의 정체를 이해할 수 있게 설명 필요** — 가공팀 (윤영조)

### 이를 반영한 설계 4대 원칙

1. **이중 입력 금지** — 기존 ERP·엑셀에 있는 데이터는 **읽기만**, 쓰기는 시스템에서
2. **기존 도구 연동 우선** — Git · Altium · SolidWorks · WEHAGO · 카톡 = 대체하지 말고 **메타데이터만 수집**
3. **카톡 대체 아닌 보조** — 중요 요청만 티켓화, 일상 잡담은 그대로 카톡
4. **읽기 자동, 쓰기 최소** — 대시보드는 자동 집계 + 상태 변경만 1~2 클릭

## 6. 🎯 기능 우선순위 (구현 로드맵)

### 🔴 1순위 — 2주 내 (10팀 공통 요구)

#### ① 프로젝트 공정 진행률 대시보드
**핵심 데이터**:
- 관리코드 × 12공정 (수주 → 컨셉 → 설계 → 전장 → SW → 가공 → 구매 → 조립 → 검수 → 출하 → Set-up → KNKVN 이관)
- 단계별 상태: 예정 / 진행 / 완료 / 지연
- 단계별 담당자 + 예정일 + 실제일

**필수 기능**:
- 지연 알림 (기준일 초과시 빨간색 + 알림)
- 모델별·고객사별·사업부별 필터
- 매일 아침 8시 자동 요약 메일/대시보드
- 클릭 시 상세 (해당 공정의 누가 무엇을 하고 있나)

**왜 1순위?**: 8팀이 직접 언급 + 매일 아침 요청 정보 1위 + 김정락 대표 본인의 가장 큰 관심사일 가능성

#### ② 변경사항 Inform 시스템
**핵심 데이터**:
- 변경 객체 (기구·전기·SW·BOM·도면)
- 변경 내용 (전/후 비교)
- 영향 받는 하류 부서 (자동 판별)

**필수 기능**:
- 기구 설계 변경 → 자동 알림 (전장·SW·제조·가공·구매)
- Concept 변경 → 하류 부서 자동 공지
- **변경 전/후 part 구분 표시** (제조2 실제 사고 예방)
- 변경 이력 DB (언제·누가·무엇을·왜)

**왜 1순위?**: 6팀 공통 + 실제 사고 사례 (제조2 — 30일 link 만료, 가공 — 변경 미통보로 잘못된 도면 가공) → 즉시 해결 필요

### 🟠 2순위 — 4~6주

#### ③ 요청 티켓 시스템 (카톡 보조)
- 자재요청·긴급가공·MODIFY·검수요청·AS만 티켓화
- 요청자·수신자·상태·소요시간 자동 기록
- 카톡은 그대로 두고, 중요 요청은 카톡 메시지에 링크 클릭으로 티켓 생성

#### ④ 개인 PC 파일 → 공용 저장소 이전
- 제조2 (출하성적서·체크리스트·특이사항)
- 품질 (출하이슈·CHECK SHEET)
- 영업 (수주현황·일정파일)
- 제조1 (작업일정표)
- 가공 (가공일정·자재현황)
- 자동 sync 또는 공용 서버 강제

#### ⑤ 전사 근태 조회
- 오늘 휴가·재택·출장 실시간 확인
- 하이웍스 근태 → HAIST WORKS 자동 연동

## 7. 부서별 핵심 한 줄 요약

| 부서 | 가장 큰 고통 | 가장 원하는 기능 |
|---|---|---|
| **구매** (정성진) | 수불 ERP·엑셀 이중 입력, 카톡 출고 요청 누락 | 진행현황·부서간 요청 공유 |
| **관리** (박지은) | 부서별 증빙 제출 지연, 카드 사용자 확인 불가 | 증빙 제출 포털 + ERP 연동 |
| **영업** (이해림·오경환) | 모델별 진척율 가시화 안됨, 일정 미팅 매일 | **모델별 진척율 % 표시** + Set Up 진행현황 |
| **전장** (김형렬) | 기구 설계 변경 통보 대기 (1~3일) | 변경사항 자동 inform |
| **설계** (윤경호) | 본사-베트남 진행 공유 부재 | 본사-법인 이슈·진행 공유 |
| **검사기** (이한빈) | 의사소통 단편화, 이력 분산 | 의사소통 원활화 + 이력 통합 |
| **가공** (윤영조) | 긴급 가공 요청 우선순위 기준 없음, 도면 버전 혼선 | 도면 버전 관리 + 진행 가시화 |
| **품질** (김정록) | 파트별 진행상태 파악 불가, 이슈 종이 기록 | 파트별 진행 + 이슈 전산 기록 |
| **제조1** (노충일) | 작업일정표 개인 PC | 진행 가시화 |
| **제조2** (임택훈) | 도면 30일 link 만료, 변경 통보 누락으로 사고 | 변경 inform + 도면 버전 관리 |
| **SW** (이한중) | 표준 라이브러리 없어 매번 새로 작성, 일정 변경 통보 늦음 | 표준 Library + 일정 통합 |
| **개발혁신** (최보현) | 공정별 진행 실시간 파악 불가 | 공정별 진행 실시간 가시화 |
